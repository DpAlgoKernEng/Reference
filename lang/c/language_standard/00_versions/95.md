# C95

## 1. 概述

**C95**（正式名称 **ISO/IEC 9899/AMD1:1995**）是 C 语言标准的一个重要修订版本，也称为 **C90 修正案 1**（C90 Amendment 1）。它是 C 语言标准化进程中的第一个正式修正案，在 C89/C90 标准基础上增加了国际化支持和少量语言特性。

C95 的核心目标是增强 C 语言的国际字符处理能力，为多语言环境提供更好的支持。它在 C 语言发展历程中起到了承上启下的作用，连接了早期的 C90 标准和后续的 C99 标准。

## 2. 来源与演变

### 历史背景

C95 的诞生源于 C 语言国际化的需求：

- **1990 年**：ISO/IEC 9899:1990（C90）发布，成为 C 语言的第一个国际标准
- **1995 年**：ISO/IEC 9899/AMD1:1995 发布，作为 C90 的修正案 1
- **1999 年**：C99 标准发布，吸收了 C95 的特性并进一步扩展

### 设计动机

C95 主要解决以下问题：

1. **国际化字符处理**：C90 标准主要面向 ASCII 字符集，对宽字符和多字节字符支持有限
2. **编程便利性**：提供更直观的操作符替代符号（双字符组）
3. **标准版本识别**：引入宏让代码能够检测编译器遵循的标准版本

### 版本演进

```
C89/C90 → C95 (AMD1) → C99 → C11 → C17 → C23
           ↑
           增加宽字符、双字符组、版本宏
```

## 3. 语法与参数

### 3.1 新语言特性

#### 双字符组（Digraphs）

双字符组是替代某些操作符和标点的两字符序列，主要用于键盘缺乏相应符号的环境：

| 双字符组 | 替代符号 | 说明 |
|----------|----------|------|
| `<%` | `{` | 左花括号 |
| `%>` | `}` | 右花括号 |
| `<:` | `[` | 左方括号 |
| `:>` | `]` | 右方括号 |
| `%:` | `#` | 预处理符号 |
| `%:%:` | `##` | 预处理连接符号 |

**注意**：双字符组在 C99 中得到保留，但实际使用较少。C99 还引入了三字符组（Trigraphs），但三字符组在 C17 中被标记为可选特性。

#### `__STDC_VERSION__` 宏

```c
__STDC_VERSION__
```

**功能**：标识编译器遵循的 C 标准版本

**取值**：
| 标准版本 | 宏值（十进制格式） | 说明 |
|----------|-------------------|------|
| C90 | 未定义 | C90 标准未定义此宏 |
| C95 | `199409L` | 1994 年 9 月通过 |
| C99 | `199901L` | 1999 年 1 月通过 |
| C11 | `201112L` | 2011 年 12 月通过 |
| C17 | `201710L` | 2017 年 10 月通过 |
| C23 | `202311L` | 2023 年 11 月通过 |

**使用场景**：条件编译，根据不同标准版本启用不同代码路径

### 3.2 新增头文件

C95 新增三个标准库头文件，提供宽字符和多字节字符处理能力：

#### `<iso646.h>`

提供操作符的宏替代形式，适用于键盘缺乏特定符号的环境：

| 宏名称 | 替代操作符 | 说明 |
|--------|-----------|------|
| `and` | `&&` | 逻辑与 |
| `or` | `||` | 逻辑或 |
| `not` | `!` | 逻辑非 |
| `bitand` | `&` | 按位与 |
| `bitor` | `|` | 按位或 |
| `xor` | `^` | 按位异或 |
| `compl` | `~` | 按位取反 |
| `and_eq` | `&=` | 按位与赋值 |
| `or_eq` | `|=` | 按位或赋值 |
| `xor_eq` | `^=` | 按位异或赋值 |
| `not_eq` | `!=` | 不等于 |

#### `<wchar.h>`

宽字符处理头文件，提供：

- 宽字符类型 `wchar_t` 的输入输出函数
- 宽字符串处理函数（如 `wcscpy`、`wcslen`）
- 宽字符分类和映射函数
- 宽字符格式化函数

#### `<wctype.h>`

宽字符分类和映射头文件，提供：

- 宽字符分类函数（如 `iswalpha`、`iswdigit`）
- 宽字符大小写转换函数（如 `towupper`、`towlower`）
- 宽字符分类和映射类型定义

## 4. 底层原理

### 4.1 双字符组处理机制

双字符组在**词法分析阶段**被编译器识别并转换为对应的单字符或操作符。处理流程：

```
源代码文本
    ↓
预处理阶段（处理 %: 和 %:%:）
    ↓
词法分析阶段（处理其他双字符组）
    ↓
标记（Token）生成
```

**关键点**：
- 双字符组**不是**宏替换，而是词法级的转换
- 双字符组可以在字符串字面量外使用
- 不影响字符串内的字符序列

### 4.2 宽字符实现机制

C95 的宽字符支持采用以下设计：

**字符编码抽象**：
- `wchar_t` 类型表示宽字符
- 宽字符的宽度（字节数）由实现定义
- 通常为 2 字节（UTF-16）或 4 字节（UTF-32）

**多字节字符与宽字符转换**：
```
多字节字符（外部编码）←→ 宽字符（内部处理）
         ↑                        ↑
    文件/网络 I/O            内存操作
```

**性能特征**：
- 宽字符占用更多内存，但处理效率更高
- 多字节字符节省存储空间，但处理需要解码

### 4.3 标准版本宏的实现

`__STDC_VERSION__` 宏由编译器预定义：

```c
// 编译器内部定义（伪代码）
#if __STDC_VERSION_C95__
#define __STDC_VERSION__ 199409L
#elif __STDC_VERSION_C99__
#define __STDC_VERSION__ 199901L
// ...
#endif
```

## 5. 使用场景

### 5.1 双字符组应用场景

| 场景 | 是否推荐 | 说明 |
|------|---------|------|
| 现代开发环境 | ❌ | 使用标准符号更清晰 |
| 遗留代码维护 | ⚠️ | 可能遇到，需理解但不推荐新增 |
| 受限键盘环境 | ✅ | 某些嵌入式设备或特殊终端 |
| 代码混淆 | ⚠️ | 不推荐用于此目的 |

### 5.2 版本宏应用场景

**跨标准版本兼容**：

```c
#if defined(__STDC_VERSION__)
    #if __STDC_VERSION__ >= 201112L
        /* C11 及以上特性 */
    #elif __STDC_VERSION__ >= 199901L
        /* C99 特性 */
    #else
        /* C90/C95 兼容代码 */
    #endif
#else
    /* C89 或非标准编译器 */
#endif
```

### 5.3 宽字符应用场景

| 场景 | 说明 |
|------|------|
| 国际化软件 | 处理多语言文本（中文、日文、韩文等） |
| 文本处理工具 | 需要处理 Unicode 字符的编辑器、解析器 |
| 系统编程 | Windows API 使用宽字符（UTF-16） |
| 文件系统操作 | 处理非 ASCII 文件名 |

### 5.4 最佳实践

**宽字符使用建议**：

```c
// ✅ 推荐：明确使用宽字符处理非 ASCII 文本
#include <wchar.h>
#include <locale.h>

int main(void) {
    setlocale(LC_ALL, "");  // 设置本地化环境
    wchar_t chinese[] = L"中文示例";
    wprintf(L"%ls\n", chinese);
    return 0;
}

// ⚠️ 注意：宽字符的宽度是平台相关的
// Windows: 2 字节（UTF-16LE）
// Linux/macOS: 4 字节（UTF-32）
```

## 6. 代码示例

### 6.1 检测 C 标准版本

```c
#include <stdio.h>

int main(void) {
    printf("C 标准版本检测：\n");

#if defined(__STDC__)
    printf("  符合 ISO C 标准\n");

    #if defined(__STDC_VERSION__)
        #if __STDC_VERSION__ >= 202311L
            printf("  版本：C23\n");
        #elif __STDC_VERSION__ >= 201710L
            printf("  版本：C17\n");
        #elif __STDC_VERSION__ >= 201112L
            printf("  版本：C11\n");
        #elif __STDC_VERSION__ >= 199901L
            printf("  版本：C99\n");
        #elif __STDC_VERSION__ >= 199409L
            printf("  版本：C95（ISO/IEC 9899/AMD1:1995）\n");
        #else
            printf("  版本：未知（__STDC_VERSION__ = %ldL）\n", __STDC_VERSION__);
        #endif
    #else
        printf("  版本：C90（无 __STDC_VERSION__ 宏）\n");
    #endif
#else
    printf("  非 ISO C 标准编译器\n");
#endif

    return 0;
}
```

**编译运行**：

```bash
# 使用 C95 模式编译
gcc -std=c95 version_check.c -o version_check

# 使用 C99 模式编译
gcc -std=c99 version_check.c -o version_check

# 使用 C11 模式编译
gcc -std=c11 version_check.c -o version_check
```

### 6.2 双字符组示例

```c
#include <stdio.h>

/* 使用双字符组的等价代码 */
int main(void)
<%
    int arr<:5:> = <%1, 2, 3, 4, 5%>;
    int i;

    for (i = 0; i < 5; i++)
        printf("arr<:%d:> = %d\n", i, arr<:i:>);

    return 0;
%>

/* 上述代码等价于：
int main(void)
{
    int arr[5] = {1, 2, 3, 4, 5};
    int i;

    for (i = 0; i < 5; i++)
        printf("arr[%d] = %d\n", i, arr[i]);

    return 0;
}
*/
```

**编译**：

```bash
gcc -std=c95 digraph_example.c -o digraph_example
```

**注意**：双字符组在现代代码中**不推荐使用**，示例仅用于理解遗留代码。

### 6.3 iso646.h 使用示例

```c
#include <stdio.h>
#include <iso646.h>  /* C95 新增 */

int main(void) {
    int a = 5, b = 3;

    /* 使用 iso646.h 宏 */
    if (a > b and b > 0) {
        printf("a > b && b > 0\n");
    }

    if (a > 0 or b < 0) {
        printf("a > 0 || b < 0\n");
    }

    int result = (a not_eq b);  /* 等价于 a != b */
    printf("a != b: %d\n", result);

    return 0;
}
```

**编译**：

```bash
gcc -std=c95 iso646_example.c -o iso646_example
```

### 6.4 宽字符处理示例

```c
#include <stdio.h>
#include <wchar.h>    /* C95 新增 */
#include <locale.h>   /* 设置本地化环境 */

int main(void) {
    /* 设置本地化环境以支持宽字符输出 */
    setlocale(LC_ALL, "");

    /* 宽字符和宽字符串字面量 */
    wchar_t wc = L'中';
    wchar_t ws[] = L"中文示例";

    /* 宽字符输出 */
    wprintf(L"宽字符：%lc\n", wc);
    wprintf(L"宽字符串：%ls\n", ws);
    wprintf(L"宽字符串长度：%zu\n", wcslen(ws));

    /* 宽字符与窄字符对比 */
    char narrow[] = "ASCII";
    wchar_t wide[] = L"ASCII";

    printf("窄字符 '%s' 大小：%zu 字节\n", narrow, sizeof(narrow));
    wprintf(L"宽字符 '%ls' 大小：%zu 字节\n", wide, sizeof(wide) * sizeof(wchar_t));

    return 0;
}
```

**编译运行**：

```bash
gcc -std=c95 wchar_example.c -o wchar_example
./wchar_example
```

**输出示例**：

```
宽字符：中
宽字符串：中文示例
宽字符串长度：4
窄字符 'ASCII' 大小：6 字节
宽字符 'ASCII' 大小：10 字节
```

### 6.5 宽字符分类示例

```c
#include <stdio.h>
#include <wctype.h>   /* C95 新增 */
#include <wchar.h>
#include <locale.h>

int main(void) {
    setlocale(LC_ALL, "");

    wchar_t wc;

    wprintf(L"输入一个字符：");
    wc = getwc(stdin);

    /* 宽字符分类 */
    if (iswalpha(wc)) {
        wprintf(L"'%lc' 是字母\n", wc);
    }
    if (iswdigit(wc)) {
        wprintf(L"'%lc' 是数字\n", wc);
    }
    if (iswlower(wc)) {
        wprintf(L"'%lc' 是小写字母\n", wc);
    }
    if (iswupper(wc)) {
        wprintf(L"'%lc' 是大写字母\n", wc);
    }
    if (iswspace(wc)) {
        wprintf(L"'%lc' 是空白字符\n", wc);
    }

    /* 大小写转换 */
    wprintf(L"大写：%lc\n", towupper(wc));
    wprintf(L"小写：%lc\n", towlower(wc));

    return 0;
}
```

### 6.6 常见错误及修正

```c
/* ❌ 错误：未设置本地化环境 */
#include <stdio.h>
#include <wchar.h>

int main(void) {
    wchar_t ws[] = L"中文";
    wprintf(L"%ls\n", ws);  /* 可能输出乱码或无输出 */
    return 0;
}

/* ✅ 正确：设置本地化环境 */
#include <stdio.h>
#include <wchar.h>
#include <locale.h>

int main(void) {
    setlocale(LC_ALL, "");  /* 设置本地化环境 */
    wchar_t ws[] = L"中文";
    wprintf(L"%ls\n", ws);  /* 正确输出 */
    return 0;
}

/* ❌ 错误：混用窄字符和宽字符函数 */
#include <stdio.h>
#include <wchar.h>

int main(void) {
    wchar_t ws[] = L"中文";
    printf("%ls\n", ws);  /* 错误：应使用 wprintf */
    return 0;
}

/* ✅ 正确：使用对应的宽字符函数 */
#include <stdio.h>
#include <wchar.h>

int main(void) {
    wchar_t ws[] = L"中文";
    wprintf(L"%ls\n", ws);  /* 正确 */
    return 0;
}
```

## 7. 总结

### 7.1 核心要点回顾

C95 标准是 C 语言发展史上一个重要的里程碑，其主要贡献：

| 特性 | 价值 | 现状 |
|------|------|------|
| `<wchar.h>` | 提供宽字符处理能力 | 仍广泛使用 |
| `<wctype.h>` | 提供宽字符分类功能 | 仍广泛使用 |
| `<iso646.h>` | 提供操作符宏替代 | 较少使用 |
| 双字符组 | 兼容受限环境 | 几乎不再使用 |
| `__STDC_VERSION__` | 标准版本检测 | 广泛使用 |

### 7.2 相关标准对比

| 标准 | 发布年份 | 主要特性 |
|------|----------|----------|
| C89/C90 | 1990 | 标准化基础特性 |
| **C95** | **1995** | **宽字符、双字符组、版本宏** |
| C99 | 1999 | 变长数组、复合字面量、inline |
| C11 | 2011 | 多线程、原子操作、泛型选择 |
| C17 | 2017 | 修正和澄清 |
| C23 | 2023 | 位精度整数、nullptr、constexpr |

### 7.3 学习建议

1. **重点掌握**：
   - 宽字符处理（`<wchar.h>`、`<wctype.h>`）
   - `__STDC_VERSION__` 宏的使用
   - 国际化编程基础

2. **了解即可**：
   - 双字符组（理解遗留代码用）
   - `<iso646.h>` 宏（实际很少使用）

3. **进阶方向**：
   - 学习 C99、C11 等后续标准的演进
   - 掌握 Unicode 编码（UTF-8、UTF-16、UTF-32）
   - 了解平台相关的宽字符实现差异
