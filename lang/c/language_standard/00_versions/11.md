# C11 语言标准

## 1. 概述

C11（正式名称 ISO/IEC 9899:2011）是 C 语言的第三个官方标准，于 2011 年由 ISO/IEC 发布。作为 C99 的继任者，C11 引入了多线程支持、原子操作、增强的对齐支持、Unicode 字符支持等重要特性，标志着 C 语言正式进入多核时代。

C11 在 C 语言发展历程中具有重要地位：
- 首次在语言层面提供原生多线程支持
- 引入泛型选择表达式，增强类型安全性
- 将部分 C99 特性改为可选，提高可移植性
- 新增边界检查函数，提升安全性

## 2. 来源与演变

### 发布背景

C11 标准于 2011 年 8 月正式发布，旨在解决现代计算环境中的新挑战：

| 时间节点 | 里程碑事件 |
|---------|-----------|
| 2007 年 | ISO/IEC JTC1/SC22/WG14 开始 C11 标准制定工作 |
| 2011 年 | C11 标准（ISO/IEC 9899:2011）正式发布 |

### 设计动机

1. **多核处理器普及**：需要在语言层面支持多线程编程
2. **安全性需求提升**：需要边界检查等安全机制
3. **Unicode 普及**：需要原生支持 UTF-16/UTF-32
4. **可移植性考虑**：将部分难以实现的特性改为可选

### 与前代标准的关系

| 特性 | C99 | C11 |
|------|-----|-----|
| 变长数组（VLA） | 强制要求 | 可选特性 |
| 复数类型 | 强制要求 | 可选特性 |
| 多线程支持 | 无 | 新增 |
| 原子操作 | 无 | 新增 |
| `gets()` 函数 | 已弃用 | 移除 |

## 3. 语法与参数

### 移除的特性

C11 正式移除了以下危险特性：

```c
// ❌ 已移除：gets() 函数
// 该函数无法检查缓冲区大小，存在缓冲区溢出风险
char buf[100];
gets(buf);  // C11 中已移除

// ✅ 替代方案：使用 fgets()
fgets(buf, sizeof(buf), stdin);
```

### 新增语言特性

#### 多线程支持

| 特性关键字 | 说明 |
|-----------|------|
| `_Atomic` | 原子类型修饰符 |
| `_Thread_local` | 线程本地存储 |

```c
// 原子变量声明
_Atomic int counter = 0;

// 线程本地存储
_Thread_local int thread_data;
```

#### 对齐支持

| 关键字 | 说明 |
|-------|------|
| `_Alignof` | 查询类型对齐要求 |
| `_Alignas` | 指定对齐要求 |

```c
// 查询对齐要求
size_t align = _Alignof(double);  // 通常为 8

// 指定对齐要求
_Alignas(16) int aligned_var;
```

#### Unicode 支持

| 前缀 | 类型 | 说明 |
|------|------|------|
| `u` | char16_t | UTF-16 字符常量 |
| `U` | char32_t | UTF-32 字符常量 |
| `u8` | char | UTF-8 字符串字面量 |
| `u` | char16_t[] | UTF-16 字符串字面量 |
| `U` | char32_t[] | UTF-32 字符串字面量 |

```c
char16_t c16 = u'好';        // UTF-16 字符
char32_t c32 = U'好';        // UTF-32 字符
char* u8s = u8"你好";        // UTF-8 字符串
char16_t* u16s = u"你好";    // UTF-16 字符串
char32_t* u32s = U"你好";    // UTF-32 字符串
```

#### 泛型选择表达式

```c
// 语法：_Generic (expression, type1: result1, type2: result2, ..., default: resultN)

#define type_name(x) _Generic((x), \
    int: "int", \
    float: "float", \
    double: "double", \
    default: "unknown")

printf("%s\n", type_name(42));      // 输出: int
printf("%s\n", type_name(3.14));    // 输出: double
```

#### 非返回函数

```c
#include <stdnoreturn.h>

// 标记不返回的函数
noreturn void fatal_error(const char* msg) {
    fprintf(stderr, "Fatal: %s\n", msg);
    exit(1);
}
```

#### 静态断言

```c
// 编译期断言
_Static_assert(sizeof(int) == 4, "int must be 4 bytes");
_Static_assert(INT_MAX > 100000, "INT_MAX too small");
```

#### 匿名结构体和联合体成员

```c
struct Person {
    char name[50];
    union {
        int age;
        int year_of_birth;
    };  // 匿名联合体
};

struct Person p;
p.age = 25;  // 直接访问匿名成员
```

### 可选特性测试宏

| 宏名称 | 说明 |
|--------|------|
| `__STDC_ANALYZABLE__` | 支持可分析性 |
| `__STDC_LIB_EXT1__` | 支持边界检查函数 |
| `__STDC_NO_ATOMICS__` | 不支持原子操作 |
| `__STDC_NO_COMPLEX__` | 不支持复数类型（C99 强制） |
| `__STDC_NO_THREADS__` | 不支持线程 |
| `__STDC_NO_VLA__` | 不支持变长数组（C99 强制） |

```c
// 检测特性可用性
#ifndef __STDC_NO_ATOMICS__
    #include <stdatomic.h>
    _Atomic int counter = 0;
#endif

#ifdef __STDC_LIB_EXT1__
    // 使用边界检查函数
    set_constraint_handler_s(abort_handler_s);
#endif
```

## 4. 底层原理

### 原子操作实现机制

原子操作（Atomic Operations）通过硬件级的原子指令实现，保证操作的不可分割性：

| 操作类型 | 实现机制 | 性能特征 |
|---------|---------|---------|
| 原子加载 | 内存屏障 + 读指令 | O(1) |
| 原子存储 | 内存屏障 + 写指令 | O(1) |
| 比较交换 | CPU CAS 指令 | O(1)，可能重试 |
| 原子算术 | CPU 原子算术指令 | O(1) |

```c
#include <stdatomic.h>

atomic_int counter = ATOMIC_VAR_INIT(0);

// 原子递增
atomic_fetch_add(&counter, 1);

// 比较交换（CAS）
int expected = 10;
bool success = atomic_compare_exchange_strong(&counter, &expected, 20);
```

### 线程本地存储机制

线程本地存储（Thread Local Storage, TLS）为每个线程维护独立的变量副本：

```
线程1: [TLS 区域] → thread_var 副本1
线程2: [TLS 区域] → thread_var 副本2
线程3: [TLS 区域] → thread_var 副本3
```

### 对齐要求原理

内存对齐影响 CPU 访问效率和某些指令的正确性：

```
地址:    0x00  0x01  0x02  0x03  0x04  0x05  0x06  0x07
         [---------- int (对齐) ----------]
         [----- 未对齐 int (性能差) -----]

对齐访问：单次内存周期完成
未对齐访问：可能需要多次内存周期，某些架构会产生异常
```

## 5. 使用场景

### 多线程编程

C11 的多线程支持适用于：

| 场景 | 推荐特性 | 说明 |
|------|---------|------|
| 计数器统计 | `_Atomic` | 线程安全的计数 |
| 线程专属数据 | `_Thread_local` | 避免锁竞争 |
| 状态标志 | `atomic_flag` | 自旋锁实现 |
| 同步操作 | `threads.h` | 线程创建与管理 |

### 跨平台开发

使用特性测试宏实现可移植代码：

```c
// 可移植的原子操作实现
#ifdef __STDC_NO_ATOMICS__
    // 回退到互斥锁实现
    pthread_mutex_t lock;
    int counter;
#else
    #include <stdatomic.h>
    atomic_int counter;
#endif
```

### 安全性要求高的场景

边界检查函数（` Annex K`）适用于：

- 输入验证
- 字符串处理
- 内存拷贝操作

```c
#ifdef __STDC_LIB_EXT1__
    errno_t err = strcpy_s(dest, dest_size, src);
    if (err != 0) {
        // 处理错误
    }
#else
    // 回退到传统函数
    strncpy(dest, src, dest_size - 1);
    dest[dest_size - 1] = '\0';
#endif
```

### 最佳实践

1. **优先使用标准特性**：使用 `_Atomic` 而非编译器扩展 `__sync_*`
2. **检测特性可用性**：使用特性测试宏保证可移植性
3. **避免 VLA 依赖**：变长数组在 C11 中为可选特性
4. **使用静态断言**：编译期捕获错误

### 常见陷阱

| 陷阱 | 说明 | 解决方案 |
|------|------|---------|
| 假设 VLA 可用 | C11 中 VLA 为可选 | 使用固定大小数组或动态分配 |
| 忽略特性检测 | 代码在某些平台无法编译 | 使用特性测试宏 |
| 原子操作语义误解 | 误以为原子操作即线程安全 | 理解内存顺序语义 |
| 边界检查函数不可用 | Annex K 实现不一致 | 提供回退实现 |

## 6. 代码示例

### 基础用法：多线程计数器

```c
#include <stdio.h>
#include <stdatomic.h>
#include <threads.h>

atomic_int counter = ATOMIC_VAR_INIT(0);

int thread_func(void* arg) {
    for (int i = 0; i < 10000; i++) {
        atomic_fetch_add(&counter, 1);
    }
    return 0;
}

int main(void) {
    thrd_t threads[4];

    // 创建 4 个线程
    for (int i = 0; i < 4; i++) {
        thrd_create(&threads[i], thread_func, NULL);
    }

    // 等待所有线程完成
    for (int i = 0; i < 4; i++) {
        thrd_join(threads[i], NULL);
    }

    printf("Final counter: %d\n", counter);  // 输出: 40000
    return 0;
}
```

### 高级用法：泛型打印函数

```c
#include <stdio.h>
#include <stdbool.h>

// 泛型打印函数
void print_value(void) {
    #define print(x) _Generic((x), \
        int: print_int, \
        float: print_float, \
        double: print_double, \
        char*: print_string, \
        const char*: print_string, \
        default: print_unknown \
    )(x)

    // 辅助函数
    void print_int(int x) { printf("int: %d\n", x); }
    void print_float(float x) { printf("float: %f\n", x); }
    void print_double(double x) { printf("double: %f\n", x); }
    void print_string(const char* x) { printf("string: %s\n", x); }
    void print_unknown(void) { printf("unknown type\n"); }
}

int main(void) {
    print(42);
    print(3.14);
    print("Hello, C11!");
    return 0;
}
```

### 高级用法：线程本地存储

```c
#include <stdio.h>
#include <threads.h>

_Thread_local int thread_id = 0;

int thread_func(void* arg) {
    thread_id = *(int*)arg;  // 每个线程有独立的副本
    printf("Thread %d: thread_id = %d\n", *(int*)arg, thread_id);
    return 0;
}

int main(void) {
    thrd_t threads[3];
    int ids[] = {1, 2, 3};

    for (int i = 0; i < 3; i++) {
        thrd_create(&threads[i], thread_func, &ids[i]);
    }

    for (int i = 0; i < 3; i++) {
        thrd_join(threads[i], NULL);
    }

    return 0;
}
```

### 常见错误与修正

```c
// ❌ 错误：假设变长数组可用
void process(int n) {
    int buffer[n];  // C99 强制，C11 可选
    // ...
}

// ✅ 正确：动态分配或固定大小
void process(int n) {
    int* buffer = malloc(n * sizeof(int));
    if (buffer) {
        // ...
        free(buffer);
    }
}

// ❌ 错误：忽略原子操作的内存顺序
atomic_int flag = ATOMIC_VAR_INIT(0);
// 线程 1
flag = 1;  // 非原子写入
// 线程 2
if (flag) { ... }  // 非原子读取

// ✅ 正确：使用原子操作
atomic_store(&flag, 1);  // 原子存储
if (atomic_load(&flag)) { ... }  // 原子加载
```

## 7. 总结

### 核心要点回顾

| 特性类别 | 主要特性 | 版本要求 |
|---------|---------|---------|
| 多线程支持 | `_Atomic`, `_Thread_local`, `<threads.h>` | C11 |
| 对齐支持 | `_Alignof`, `_Alignas` | C11 |
| Unicode 支持 | `u`, `U`, `u8` 字面量 | C11 |
| 泛型编程 | `_Generic` | C11 |
| 编译期检查 | `_Static_assert` | C11 |
| 安全性 | 边界检查函数 | C11 可选 |

### 与 C99 的主要区别

| 方面 | C99 | C11 |
|------|-----|-----|
| 变长数组（VLA） | 强制要求 | 可选特性 |
| 复数类型 | 强制要求 | 可选特性 |
| 多线程支持 | 无 | 原生支持 |
| `gets()` 函数 | 弃用 | 移除 |
| 泛型选择 | 无 | 新增 |

### 学习建议

1. **掌握核心特性**：重点学习 `_Atomic`、`_Generic`、`_Static_assert`
2. **理解可选性**：C11 将部分特性改为可选，编写可移植代码需检测特性可用性
3. **实践多线程**：使用 `<threads.h>` 进行多线程编程实践
4. **安全编程**：了解边界检查函数的使用场景
