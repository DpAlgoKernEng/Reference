# C23 语言标准

## 1. 概述

C23（正式名称 ISO/IEC 9899:2024）是 C 语言的最新标准版本，于 2024 年正式发布。作为 C 语言标准的重大更新，C23 引入了多项现代编程特性，同时移除了部分过时的语言特性，使 C 语言更加安全、现代化。

C23 将预定义宏 `__STDC_VERSION__` 的值更新为 `202311L`，用于标识编译器对 C23 标准的支持。

### 核心定位

- **现代化**: 引入属性、二进制字面量、数字分隔符等现代特性
- **类型安全**: 新增 `nullptr` 常量、位精确整数类型等类型安全机制
- **兼容性提升**: 与 C++ 特性对齐，提高代码可移植性
- **性能优化**: 支持十进制浮点类型、改进的浮点运算

## 2. 来源与演变

### 历史背景

C23 是继 C11（2011）、C17/C18（2018）之后的重要标准更新。其设计动机主要包括：

1. **弥补历史缺陷**: 移除不安全的语言特性，消除未定义行为
2. **与现代硬件对齐**: 支持十进制浮点运算、位精确整数
3. **C++ 兼容性**: 引入与 C++ 兼容的属性语法、`nullptr` 等

### 主要版本演进

| 版本 | 发布年份 | `__STDC_VERSION__` | 主要特性 |
|------|----------|-------------------|----------|
| C89/C90 | 1989/1990 | 未定义 | 基础语言标准 |
| C99 | 1999 | 199901L | 变长数组、`long long`、复数类型 |
| C11 | 2011 | 201112L | 多线程、原子操作、泛型选择 |
| C17/C18 | 2017/2018 | 201710L | 缺陷修复、小改进 |
| C23 | 2024 | 202311L | 属性、位精确整数、十进制浮点 |

## 3. 语法与参数

### 3.1 移除的特性

以下特性在 C23 中被完全移除：

| 移除特性 | 说明 |
|----------|------|
| 旧式函数声明和定义 | K&R 风格函数声明不再支持 |
| 非二进制补码有符号整数 | 仅支持二进制补码表示 |
| `u`/`U` 前缀字符常量非 UTF-16/32 | 必须为 UTF-16/32 编码 |
| 混合宽字符串拼接 | 不同宽度的字符串字面量不可拼接 |
| `realloc(ptr, 0)` 定义行为 | 行为变为未定义 |
| `__alignof_is_defined`、`__alignas_is_defined` 宏 | 移除对齐相关宏定义 |
| `<assert.h>` 中的 `static_assert` 宏 | 成为关键字 |

### 3.2 弃用的特性

以下特性被标记为弃用，将在未来版本移除：

| 弃用特性 | 替代方案 |
|----------|----------|
| `<stdnoreturn.h>` 头文件 | 使用 `[[noreturn]]` 属性 |
| `__STDC_IEC_559__` 宏 | 使用 `__STDC_IEC_60559_BFP__` |
| `__STDC_IEC_559_COMPLEX__` 宏 | 使用 `__STDC_IEC_60559_COMPLEX__` |
| `_Noreturn` 函数说明符 | 使用 `[[noreturn]]` 属性 |
| `asctime()`、`ctime()` | 使用 `strftime()` |
| `DECIMAL_DIG` 宏 | 使用类型特定宏如 `FLT_DECIMAL_DIG` |
| `__bool_true_false_are_defined` | `true`/`false` 已成为关键字 |

### 3.3 新增语言特性语法

#### 属性语法

```c
[[deprecated]]           // 弃用属性
[[deprecated("use func_v2")]]  // 带消息的弃用属性
[[fallthrough]]          // switch 穿透提示
[[maybe_unused]]         // 抑制未使用警告
[[nodiscard]]            // 必须使用返回值
[[nodiscard("必须检查错误码")]]  // 带消息
[[noreturn]]             // 函数不返回
[[unsequenced]]         // 纯函数（无副作用）
[[reproducible]]        // 可复现函数
```

#### 位精确整数类型

```c
_BitInt(N)   // N 位有符号整数
unsigned _BitInt(N)  // N 位无符号整数
```

#### 十进制浮点类型

```c
_Decimal32   // 32 位十进制浮点
_Decimal64   // 64 位十进制浮点
_Decimal128  // 128 位十进制浮点
```

#### 空指针常量

```c
nullptr      // 类型安全的空指针常量
nullptr_t    // nullptr 的类型
```

#### 新增预处理器指令

```c
#elifdef     // 等价于 #elif defined
#elifndef    // 等价于 #elif !defined
#warning     // 生成编译警告
#embed       // 嵌入二进制资源
```

### 3.4 新增头文件

| 头文件 | 功能 |
|--------|------|
| `<stdbit.h>` | 位操作函数 |
| `<stdckdint.h>` | 整数溢出检查 |

### 3.5 可选特性测试宏

| 宏名称 | 说明 |
|--------|------|
| `__STDC_IEC_60559_BFP__` | IEEE-754 二进制浮点支持（替代 `__STDC_IEC_559__`） |
| `__STDC_IEC_60559_DFP__` | IEEE-754 十进制浮点支持 |
| `__STDC_IEC_60559_COMPLEX__` | IEEE-754 复数运算支持（替代 `__STDC_IEC_559_COMPLEX__`） |

## 4. 底层原理

### 4.1 二进制补码唯一性

C23 移除了对非二进制补码有符号整数表示的支持，这意味着：

- **简化实现**: 编译器无需处理多种整数表示方式
- **优化机会**: 编译器可针对二进制补码进行优化
- **可预测行为**: 整数运算行为更加确定

### 4.2 属性实现机制

属性通过双括号语法 `[[attribute]]` 实现：

1. **解析阶段**: 编译器在语法分析阶段识别属性
2. **语义标注**: 属性附加到声明或语句的 AST 节点
3. **代码生成**: 根据属性生成相应的代码或元数据

### 4.3 位精确整数的实现

`_BitInt(N)` 类型：

- **内存布局**: 按照机器字长对齐，使用最小可用存储
- **运算规则**: 遵循整数运算规则，支持所有整数运算
- **限制条件**: N 必须至少为 1，最大值由实现定义

### 4.4 十进制浮点运算

十进制浮点类型基于 IEEE 754-2008 标准：

| 类型 | 有效位数 | 指数范围 | 用途 |
|------|----------|----------|------|
| `_Decimal32` | 7 位 | -95 ~ +96 | 存储优化 |
| `_Decimal64` | 16 位 | -383 ~ +384 | 通用计算 |
| `_Decimal128` | 34 位 | -6143 ~ +6144 | 高精度计算 |

优势：避免二进制浮点的精度损失，适用于金融计算。

## 5. 使用场景

### 5.1 适用场景

| 场景 | C23 特性 | 优势 |
|------|----------|------|
| 金融计算 | 十进制浮点类型 | 避免浮点精度损失 |
| 嵌入式开发 | 位精确整数 `_BitInt(N)` | 精确控制内存使用 |
| 安全关键代码 | `[[nodiscard]]`、`[[noreturn]]` | 编译时错误检测 |
| 代码维护 | `[[deprecated]]` | 平滑 API 迁移 |
| 资源嵌入 | `#embed` | 简化资源文件处理 |
| 跨平台代码 | `nullptr` | 类型安全的空指针 |

### 5.2 最佳实践

1. **使用属性替代旧式宏**
   ```c
   // 推荐
   [[noreturn]] void panic(const char *msg);
   
   // 不推荐（已弃用）
   #include <stdnoreturn.h>
   noreturn void panic(const char *msg);
   ```

2. **使用 `nullptr` 替代 `NULL` 或 `0`**
   ```c
   // 推荐：类型安全
   int *ptr = nullptr;
   
   // 不推荐
   int *ptr = NULL;  // 宏定义，类型不安全
   int *ptr = 0;     // 整数隐式转换
   ```

3. **使用数字分隔符提高可读性**
   ```c
   int million = 1'000'000;
   int binary = 0b1010'1100;
   int hex = 0x1234'5678;
   ```

4. **使用二进制字面量**
   ```c
   int flags = 0b1110'0101;  // C23 二进制字面量
   ```

### 5.3 常见陷阱

1. **旧式函数声明迁移**
   ```c
   // C23 不再支持
   void old_func();  // 无原型声明
   
   // 正确写法
   void old_func(void);  // 明确无参数
   ```

2. **`realloc` 零大小行为变更**
   ```c
   // C23 之前：实现定义行为
   // C23：未定义行为
   void *ptr = realloc(old_ptr, 0);  // 危险！
   ```

3. **属性位置**
   ```c
   // 正确：属性在声明符之前
   [[nodiscard]] int get_value(void);
   
   // 错误：属性不能在声明符之后
   int get_value(void) [[nodiscard]];  // 语法错误
   ```

## 6. 代码示例

### 6.1 基础特性示例

#### 属性使用

```c
#include <stdio.h>

// 弃用函数警告
[[deprecated("请使用 process_v2()")]]
void process(int data) {
    printf("处理数据: %d\n", data);
}

// 必须检查返回值
[[nodiscard]] int get_status(void) {
    return 0;
}

// 不返回的函数
[[noreturn]] void fatal_error(const char *msg) {
    fprintf(stderr, "致命错误: %s\n", msg);
    exit(1);
}

// switch 穿透提示
void handle_option(int opt) {
    switch (opt) {
        case 1:
            printf("选项 1\n");
            [[fallthrough]];  // 明确穿透意图
        case 2:
            printf("选项 2\n");
            break;
    }
}

int main(void) {
    process(42);         // 编译警告：函数已弃用
    get_status();        // 编译警告：未使用返回值
    return 0;
}
```

#### 位精确整数

```c
#include <stdio.h>

int main(void) {
    // 定义精确位数的整数
    _BitInt(8) small = 127;
    unsigned _BitInt(4) nibble = 15;
    _BitInt(128) huge = 0;
    
    printf("8 位整数: %d\n", (int)small);
    printf("4 位无符号: %u\n", (unsigned)nibble);
    
    return 0;
}
```

#### 二进制字面量与数字分隔符

```c
#include <stdio.h>

int main(void) {
    // 二进制字面量
    int binary = 0b1010'1100;
    
    // 数字分隔符提高可读性
    long credit_card = 1234'5678'9012'3456L;
    int hex_value = 0xDEAD'BEEF;
    
    printf("二进制: %d\n", binary);
    printf("信用卡号格式: %ld\n", credit_card);
    printf("十六进制: 0x%X\n", hex_value);
    
    return 0;
}
```

### 6.2 高级特性示例

#### nullptr 与类型安全

```c
#include <stdio.h>
#include <stddef.h>  // nullptr_t 定义

int main(void) {
    // C23 类型安全的空指针
    int *ptr = nullptr;
    
    // nullptr_t 类型
    nullptr_t null_val = nullptr;
    
    if (ptr == nullptr) {
        printf("指针为空\n");
    }
    
    // nullptr 可隐式转换为任意指针类型
    char *str = nullptr;
    double *nums = nullptr;
    
    return 0;
}
```

#### 空初始化器

```c
#include <stdio.h>

struct Point {
    int x, y, z;
};

int main(void) {
    // 空初始化器：所有成员初始化为零
    int arr[10] = {};
    struct Point p = {};
    int value = {};  // 等价于 0
    
    printf("数组首元素: %d\n", arr[0]);
    printf("结构体 x: %d\n", p.x);
    printf("值: %d\n", value);
    
    return 0;
}
```

#### 十进制浮点运算

```c
#include <stdio.h>
#include <decimal.h>  // 十进制浮点支持

int main(void) {
    // 十进制浮点：避免精度损失
    _Decimal64 price = 0.10DD;  // 精确表示 0.10
    _Decimal64 quantity = 3.0DD;
    _Decimal64 total = price * quantity;
    
    printf("总价: %Df\n", (double)total);  // 精确的 0.30
    
    // 对比二进制浮点
    double bin_price = 0.10;
    double bin_quantity = 3.0;
    double bin_total = bin_price * bin_quantity;
    
    printf("二进制浮点总价: %.17f\n", bin_total);  // 可能有精度误差
    
    return 0;
}
```

#### 新增预处理器指令

```c
#include <stdio.h>

#define DEBUG 1
#define RELEASE 0

#define BUILD DEBUG

int main(void) {
#if BUILD == DEBUG
    printf("调试模式\n");
#elifdef RELEASE
    printf("发布模式\n");
#elifndef TEST
    printf("未知模式\n");
#endif

#warning "此代码需要进一步测试"

    return 0;
}
```

### 6.3 常见错误及修正

#### 错误：旧式函数声明

```c
// ❌ 错误：C23 不再支持
void func();  // 无原型声明

// ✅ 正确：明确参数列表
void func(void);  // 无参数
void func(int x, int y);  // 有参数
```

#### 错误：realloc 零大小

```c
#include <stdlib.h>

// ❌ 错误：C23 中零大小 realloc 为未定义行为
void *ptr = realloc(old_ptr, 0);

// ✅ 正确：使用 free 显式释放
if (new_size == 0) {
    free(old_ptr);
    ptr = NULL;
} else {
    ptr = realloc(old_ptr, new_size);
}
```

#### 错误：混用过时宏

```c
#include <stdbool.h>

// ❌ 错误：C23 中不再需要此检查
#ifndef __bool_true_false_are_defined
// ...
#endif

// ✅ 正确：直接使用 true/false（现为关键字）
bool flag = true;
if (flag == false) { /* ... */ }
```

## 7. 总结

### 7.1 核心要点回顾

| 类别 | 关键变更 |
|------|----------|
| 语言现代化 | 属性语法、二进制字面量、数字分隔符 |
| 类型安全 | `nullptr`、位精确整数、十进制浮点 |
| 移除特性 | 旧式函数声明、非二进制补码整数 |
| 弃用特性 | `<stdnoreturn.h>`、`_Noreturn`、`asctime()`/`ctime()` |
| 新增头文件 | `<stdbit.h>`、`<stdckdint.h>` |
| 预处理器 | `#elifdef`、`#elifndef`、`#warning`、`#embed` |

### 7.2 编译器支持现状

| 编译器 | 最低支持版本 | 备注 |
|--------|--------------|------|
| GCC | 13+ | 大部分特性支持 |
| Clang | 16+ | 大部分特性支持 |
| MSVC | 19.39+ | 部分特性支持 |
| Apple Clang | 13.1.6+ | 部分特性支持 |

### 7.3 迁移建议

1. **启用 C23 标准**
   ```bash
   gcc -std=c23 source.c
   clang -std=c23 source.c
   ```

2. **检查代码兼容性**
   - 移除所有旧式函数声明
   - 替换 `_Noreturn` 为 `[[noreturn]]`
   - 更新 `realloc(ptr, 0)` 调用

3. **利用新特性优化代码**
   - 使用属性增强代码可读性和安全性
   - 使用 `nullptr` 提高类型安全
   - 使用十进制浮点处理精确计算

### 7.4 相关标准对比

| 特性 | C17 | C23 | C++17 | C++20 |
|------|-----|-----|-------|-------|
| 属性语法 | 无 | `[[...]]` | `[[...]]` | `[[...]]` |
| `nullptr` | 无 | 有 | 有 | 有 |
| 二进制字面量 | 无 | 有 | 有 | 有 |
| 数字分隔符 | 无 | `'` | `'` | `'` |
| `constexpr` | 无 | 有 | 有 | 有 |

### 7.5 学习建议

1. **优先掌握**: 属性语法、`nullptr`、数字分隔符
2. **按需学习**: 位精确整数、十进制浮点（特定领域）
3. **逐步迁移**: 从弃用特性迁移到新特性
4. **关注编译器**: 持续关注编译器对新特性的支持进度
