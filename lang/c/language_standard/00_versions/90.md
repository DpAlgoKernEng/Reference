# C89/C90 标准

## 1. 概述

**C89/C90** 是 C 编程语言的第一个国际标准，正式名称为 **ISO/IEC 9899:1990**。该标准最初由 ANSI（美国国家标准协会）开发，称为 **ANSI X3.159-1989**（通常称为 ANSI C 或 C89），后来被 ISO（国际标准化组织）以极少的修改采纳为国际标准。

### 核心定位

- C 语言的第一个正式标准化版本
- 奠定了现代 C 语言的基础
- 所有 C 编译器都支持的基准标准

### 标准命名

| 名称 | 来源 | 说明 |
|------|------|------|
| ANSI C | ANSI X3.159-1989 | ANSI 于 1989 年发布 |
| C89 | ANSI X3.159-1989 | 同 ANSI C，强调年份 |
| C90 | ISO/IEC 9899:1990 | ISO 于 1990 年采纳 |
| ISO C | ISO/IEC 9899:1990 | 国际标准称呼 |

**注意**：ANSI 版本和 ISO 版本在技术上完全相同，仅存在格式和章节编号的差异。

## 2. 来源与演变

### 历史背景

C 语言的标准化工作始于 **1983 年**，由 ANSI X3J11 委员会负责。在此之前，C 语言主要依据 Brian Kernighan 和 Dennis Ritchie 合著的《The C Programming Language》（K&R）第一版作为事实标准。

### 发展时间线

| 年份 | 事件 |
|------|------|
| 1983 | ANSI X3J11 委员会成立，开始标准化工作 |
| 1989 | ANSI 批准 X3.159-1989 标准（ANSI C/C89） |
| 1990 | ISO 采纳为 ISO/IEC 9899:1990（C90） |

### 设计动机

- **统一方言**：K&R C 存在多种编译器扩展和方言分歧
- **提高可移植性**：确保相同代码在不同平台行为一致
- **明确语义**：消除模糊和未定义行为
- **标准化库**：提供统一的标准库接口

## 3. 语法与参数

### 新增关键字

C89/C90 标准引入了以下新关键字：

| 关键字 | 用途 | 说明 |
|--------|------|------|
| `const` | 常量限定符 | 声明不可修改的对象 |
| `volatile` | 易变限定符 | 声明可能被意外修改的对象 |
| `signed` | 有符号修饰符 | 显式声明有符号整数类型 |

### 函数原型（Function Prototype）

C89 引入了完整的函数原型语法，取代了旧的 K&R 风格声明：

```c
// ✅ 新风格（函数原型）
int func(int x, double y);

// ⚠️ 旧风格（K&R，已废弃但仍支持）
int func(x, y)
    int x;
    double y;
```

**函数原型的优势**：
- 编译时参数类型检查
- 参数类型不匹配时的警告或错误
- 自动参数类型转换

### 预处理器增强

新增预处理器指令：

| 指令 | 用途 |
|------|------|
| `#elif` | 条件编译的 else-if 分支 |
| `#error` | 生成编译错误消息 |
| `#pragma` | 编译器特定的指令 |

标准预定义宏：

| 宏 | 说明 |
|----|------|
| `__FILE__` | 当前源文件名 |
| `__LINE__` | 当前行号 |
| `__DATE__` | 编译日期 |
| `__TIME__` | 编译时间 |
| `__STDC__` | 编译器是否符合 ANSI C 标准 |

### 其他语言特性

| 特性 | 说明 |
|------|------|
| `void` 类型 | 表示无类型或无返回值 |
| `void *` 指针 | 通用指针类型 |
| `wchar_t` | 宽字符类型 |
| `enum` | 枚举类型 |
| `long double` | 扩展精度浮点类型 |
| 结构体赋值 | 支持结构体整体赋值和作为函数参数传递 |
| 十六进制转义序列 | 字符串字面量中的 `\x` 转义 |

## 4. 底层原理

### 标准库架构

C89 标准定义了 15 个标准头文件，构成了 C 标准库的核心：

| 头文件 | 功能领域 | 主要内容 |
|--------|----------|----------|
| `<assert.h>` | 诊断 | 断言宏 `assert()` |
| `<ctype.h>` | 字符处理 | 字符分类和转换函数 |
| `<errno.h>` | 错误处理 | 错误码 `errno` |
| `<float.h>` | 浮点特性 | 浮点类型限制常量 |
| `<limits.h>` | 实现限制 | 整数类型限制常量 |
| `<locale.h>` | 本地化 | 区域设置函数 |
| `<math.h>` | 数学运算 | 数学函数库 |
| `<setjmp.h>` | 非局部跳转 | `setjmp()`/`longjmp()` |
| `<signal.h>` | 信号处理 | 信号处理函数 |
| `<stdarg.h>` | 可变参数 | 可变参数函数支持 |
| `<stddef.h>` | 标准定义 | 常用类型和宏 |
| `<stdio.h>` | 输入输出 | 标准输入输出函数 |
| `<stdlib.h>` | 通用工具 | 内存分配、程序控制等 |
| `<string.h>` | 字符串处理 | 字符串操作函数 |
| `<time.h>` | 日期时间 | 时间处理函数 |

### 翻译限制

C89 标准规定了符合标准的实现必须支持的最低限制：

| 限制项 | 最小值 |
|--------|--------|
| 复合语句、循环、选择语句嵌套层数 | 15 |
| 条件编译嵌套层数 | 15 |
| 指针、数组、函数声明符组合嵌套层数 | 12 |
| 括号表达式嵌套层数 | 15 |
| 函数调用参数个数 | 127 |
| 函数定义参数个数 | 127 |
| 结构体或联合体定义嵌套层数 | 15 |
| 单个结构体或联合体成员数 | 63 |
| 单个枚举中的枚举常量数 | 31 |

## 5. 使用场景

### 适用场景

1. **最大兼容性需求**
   - 需要在所有 C 编译器上编译的代码
   - 嵌入式系统开发（资源受限环境）
   - 跨平台库的核心实现

2. **遗留代码维护**
   - 维护和扩展现有 C89 代码库
   - 遗留系统移植

3. **教学目的**
   - 学习 C 语言基础概念
   - 理解 C 语言演进历史

### 编译器支持

C89/C90 是所有 C 编译器必须支持的基础标准：

| 编译器 | 支持方式 |
|--------|----------|
| GCC | `-std=c90` 或 `-ansi` |
| Clang | `-std=c90` |
| MSVC | 默认支持 |
| Intel C++ | 完全支持 |

### 最佳实践

```c
/* 使用函数原型进行类型检查 */
int calculate(int a, double b);  /* 推荐 */

/* 使用 const 保护只读数据 */
size_t my_strlen(const char *s);

/* 使用 volatile 标记硬件寄存器 */
volatile int *hardware_register;

/* 使用预定义宏进行调试 */
#ifdef DEBUG
    printf("Debug: %s:%d\n", __FILE__, __LINE__);
#endif
```

### 常见陷阱

1. **K&R 函数声明**：旧风格声明不会进行参数检查
2. **隐式 int**：省略类型默认为 int（在 C99 中已移除）
3. **未声明函数**：隐式声明函数返回 int（在 C99 中已移除）

## 6. 代码示例

### 基础用法

#### 函数原型与声明

```c
#include <stdio.h>
#include <stdlib.h>

/* 函数原型 - 提供类型检查 */
int add(int a, int b);
void print_message(const char *msg);

int main(void) {
    int result = add(10, 20);
    printf("Result: %d\n", result);
    print_message("Hello, C89!");
    return 0;
}

int add(int a, int b) {
    return a + b;
}

void print_message(const char *msg) {
    printf("Message: %s\n", msg);
}
```

#### const 和 volatile 使用

```c
#include <stdio.h>

/* const 参数 - 防止修改 */
size_t safe_strlen(const char *str) {
    size_t len = 0;
    while (str[len] != '\0') {
        len++;
    }
    return len;
}

/* volatile 变量 - 可能被外部修改 */
volatile int interrupt_flag = 0;

int main(void) {
    const int max_value = 100;  /* 常量 */
    volatile int *port = (volatile int *)0xFF00;  /* 硬件端口 */
    
    printf("Length: %lu\n", (unsigned long)safe_strlen("Hello"));
    printf("Max: %d\n", max_value);
    
    return 0;
}
```

#### 预处理器使用

```c
#include <stdio.h>

/* 条件编译 */
#ifdef DEBUG
    #define LOG(msg) printf("[DEBUG] %s:%d: %s\n", __FILE__, __LINE__, msg)
#else
    #define LOG(msg) ((void)0)
#endif

/* 编译时错误检查 */
#ifndef BUFFER_SIZE
    #error "BUFFER_SIZE must be defined"
#endif

/* 平台特定配置 */
#if defined(_WIN32)
    #define PLATFORM_NAME "Windows"
#elif defined(__linux__)
    #define PLATFORM_NAME "Linux"
#elif defined(__APPLE__)
    #define PLATFORM_NAME "macOS"
#else
    #define PLATFORM_NAME "Unknown"
#endif

int main(void) {
    LOG("Application started");
    printf("Platform: %s\n", PLATFORM_NAME);
    printf("Compiled on %s at %s\n", __DATE__, __TIME__);
    
    if (__STDC__) {
        printf("Compiler conforms to ANSI C standard\n");
    }
    
    return 0;
}
```

#### 标准库使用

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <time.h>

int main(void) {
    /* 内存分配 */
    char *buffer = (char *)malloc(256);
    if (buffer == NULL) {
        fprintf(stderr, "Memory allocation failed\n");
        return 1;
    }
    
    /* 字符串操作 */
    strcpy(buffer, "Hello");
    strcat(buffer, ", C89!");
    printf("%s (length: %lu)\n", buffer, (unsigned long)strlen(buffer));
    
    /* 时间处理 */
    time_t now = time(NULL);
    printf("Current time: %s", ctime(&now));
    
    /* 清理 */
    free(buffer);
    
    return 0;
}
```

### 高级用法

#### 可变参数函数

```c
#include <stdio.h>
#include <stdarg.h>

/* 可变参数函数 */
int sum(int count, ...) {
    va_list args;
    int total = 0;
    
    va_start(args, count);
    for (int i = 0; i < count; i++) {
        total += va_arg(args, int);
    }
    va_end(args);
    
    return total;
}

/* 格式化日志 */
void log_format(const char *level, const char *format, ...) {
    va_list args;
    printf("[%s] ", level);
    va_start(args, format);
    vprintf(format, args);
    va_end(args);
    printf("\n");
}

int main(void) {
    printf("Sum: %d\n", sum(4, 10, 20, 30, 40));
    log_format("INFO", "Value: %d, Name: %s", 42, "Test");
    return 0;
}
```

#### 结构体与枚举

```c
#include <stdio.h>
#include <string.h>

/* 枚举类型 */
typedef enum {
    STATUS_OK = 0,
    STATUS_ERROR = -1,
    STATUS_PENDING = 1,
    STATUS_CANCELLED = 2
} Status;

/* 结构体 */
typedef struct {
    char name[64];
    int age;
    double salary;
    Status status;
} Employee;

/* 结构体作为参数 */
void print_employee(const Employee *emp) {
    printf("Name: %s\n", emp->name);
    printf("Age: %d\n", emp->age);
    printf("Salary: %.2f\n", emp->salary);
    printf("Status: %d\n", emp->status);
}

int main(void) {
    Employee emp1;
    
    /* 结构体初始化 */
    strcpy(emp1.name, "John Doe");
    emp1.age = 30;
    emp1.salary = 50000.0;
    emp1.status = STATUS_OK;
    
    /* 结构体赋值 */
    Employee emp2 = emp1;
    strcpy(emp2.name, "Jane Doe");
    
    print_employee(&emp1);
    print_employee(&emp2);
    
    return 0;
}
```

### 常见错误及修正

```c
#include <stdio.h>

/* ❌ 错误：使用旧式 K&R 声明 */
int old_func(x, y)
    int x;
    double y;
{
    return x + (int)y;
}

/* ✅ 正确：使用函数原型 */
int new_func(int x, double y) {
    return x + (int)y;
}

int main(void) {
    /* ❌ 错误：隐式 int 声明（C89 允许但不推荐） */
    /* result = add(1, 2); */  /* 未声明函数 */
    
    /* ✅ 正确：先声明后使用 */
    int result = new_func(10, 20.5);
    printf("Result: %d\n", result);
    
    return 0;
}
```

## 7. 总结

### 核心要点回顾

| 要点 | 说明 |
|------|------|
| 标准地位 | C 语言的第一个国际标准，所有编译器的基准 |
| 核心特性 | 函数原型、const/volatile/signed 关键字、void 类型、enum |
| 标准库 | 15 个标准头文件，定义了核心输入输出、内存管理、字符串处理等功能 |
| 编译支持 | 所有 C 编译器均完全支持 |

### 与后续标准的对比

| 特性 | C89/C90 | C99 | C11 |
|------|---------|-----|-----|
| 变量声明位置 | 块起始 | 任意位置 | 任意位置 |
| 变长数组 (VLA) | 不支持 | 支持 | 可选 |
| `inline` 关键字 | 不支持 | 支持 | 支持 |
| `_Bool` 类型 | 不支持 | 支持 | 支持 |
| `_Generic` | 不支持 | 不支持 | 支持 |
| 线程支持 | 不支持 | 不支持 | 支持 |

### 学习建议

1. **作为 C 语言学习的起点**
   - C89 是最简洁的 C 标准，适合入门学习
   - 理解函数原型、类型系统等核心概念

2. **逐步学习现代特性**
   - 在掌握 C89 基础后，学习 C99/C11 的新特性
   - 了解 C99 的变长数组、指定初始化器等

3. **实践建议**
   - 使用 `-std=c90 -Wall -Wextra` 编译选项验证代码兼容性
   - 阅读经典 C 语言教材（如 K&R 第二版）
