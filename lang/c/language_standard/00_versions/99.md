# C99 语言标准

## 1. 概述

**C99**（ISO/IEC 9899:1999）是 C 语言的正式标准版本，由国际标准化组织（ISO）和国际电工委员会（IEC）于 1999 年发布。C99 是 C90（C89）标准的重要后续版本，引入了大量新特性和改进，显著增强了 C 语言的表达能力和可移植性。

C99 在保持 C 语言简洁高效核心理念的同时，添加了布尔类型、变长数组、复数支持、内联函数等现代编程特性，使其更适合科学计算、系统编程和嵌入式开发等领域。

## 2. 来源与演变

### 历史背景

C99 是 C 语言标准化进程中的第二个主要版本：

| 版本 | 发布年份 | 说明 |
|------|----------|------|
| C89/C90 | 1989/1990 | 首个 ANSI/ISO 标准 |
| C99 | 1999 | 本文档所述版本 |
| C11 | 2011 | 引入多线程、原子操作等 |
| C17/C18 | 2017/2018 | C11 的技术修正版 |
| C23 | 2024 | 最新标准版本 |

### 设计目标

C99 的主要设计目标包括：

- 提高编程效率和代码可读性
- 增强数值计算能力（复数、浮点环境）
- 改善与 C++ 的兼容性
- 解决 C90 标准中的缺陷和模糊之处

### 废弃与移除的特性

C99 移除或废弃了部分 C90 特性：

| 状态 | 特性 | 说明 |
|------|------|------|
| **移除** | 隐式 int 声明 | 函数声明必须明确返回类型 |
| **移除** | 隐式函数声明 | 使用函数前必须声明或定义 |
| **废弃** | `gets()` 函数 | 存在缓冲区溢出风险，C11 正式移除 |

## 3. 语法与参数

### 3.1 新增语言特性

#### 布尔类型

```c
#include <stdbool.h>

bool flag = true;        // 布尔变量
flag = false;            // 布尔值
```

| 类型 | 宏定义 | 说明 |
|------|--------|------|
| `_Bool` | 原生类型 | C99 原生布尔类型 |
| `bool` | 宏 | 展开为 `_Bool` |
| `true` | 宏 | 展开为 `1` |
| `false` | 宏 | 展开为 `0` |

#### 长整型类型

```c
long long ll = 9223372036854775807LL;
unsigned long long ull = 18446744073709551615ULL;
```

| 类型 | 说明 | 最小范围 |
|------|------|----------|
| `long long` | 有符号长整型 | -2^63 到 2^63-1 |
| `unsigned long long` | 无符号长整型 | 0 到 2^64-1 |

#### 复数类型

```c
#include <complex.h>

double complex z1 = 1.0 + 2.0 * I;
double complex z2 = CMPLX(3.0, 4.0);

double real_part = creal(z1);    // 获取实部
double imag_part = cimag(z1);    // 获取虚部
```

| 类型 | 说明 |
|------|------|
| `float _Complex` | 单精度复数 |
| `double _Complex` | 双精度复数 |
| `long double _Complex` | 扩展精度复数 |
| `float _Imaginary` | 单精度虚数（可选） |
| `double _Imaginary` | 双精度虚数（可选） |

#### 变长数组（VLA）

```c
void process(int n) {
    int arr[n];                    // 变长数组
    int matrix[n][n];              // 二维变长数组
}
```

**注意**：VLA 在 C11 中变为可选特性。

#### 灵活数组成员

```c
struct Header {
    int length;
    char data[];    // 灵活数组成员
};

struct Header *h = malloc(sizeof(struct Header) + 100);
h->length = 100;
```

#### 指定初始化器

```c
struct Point { int x, y, z; };

struct Point p1 = { .x = 1, .z = 3 };        // 指定成员初始化
int arr[10] = { [0] = 1, [5] = 2, [9] = 3 }; // 指定索引初始化
```

#### 复合字面量

```c
struct Point { int x, y; };

void print_point(struct Point p);

// 直接构造临时对象
print_point((struct Point){ .x = 10, .y = 20 });

int *arr = (int[]){ 1, 2, 3, 4, 5 };    // 数组复合字面量
```

#### 内联函数

```c
static inline int square(int x) {
    return x * x;
}
```

#### restrict 关键字

```c
void copy(int *restrict dest, const int *restrict src, size_t n) {
    // restrict 保证 dest 和 src 不重叠
    for (size_t i = 0; i < n; i++) {
        dest[i] = src[i];
    }
}
```

#### 变参宏

```c
#define debug(fmt, ...) fprintf(stderr, fmt, __VA_ARGS__)
#define log_msg(...) printf(__VA_ARGS__)

debug("Error: %s, line %d\n", __FILE__, __LINE__);
```

#### for 循环初始化语句

```c
// C99 允许在 for 循环中声明变量
for (int i = 0; i < 10; i++) {
    printf("%d\n", i);
}
// i 在此处超出作用域
```

#### 混合声明与代码

```c
void process(int n) {
    printf("Processing %d items\n", n);
    
    int arr[n];           // 变长数组声明
    int sum = 0;          // 变量声明可以出现在代码块任意位置
    
    for (int i = 0; i < n; i++) {
        arr[i] = i * 2;
    }
}
```

#### // 单行注释

```c
// 这是 C99 引入的单行注释风格
int x = 10; // 行尾注释
```

#### 十六进制浮点常量

```c
double a = 0x1.5p2;      // 等于 5.25 (1.3125 * 2^2)
double b = 0x1.fp4;      // 等于 31.0 (1.9375 * 2^4)
```

#### __func__ 预定义标识符

```c
void log_function() {
    printf("Current function: %s\n", __func__);
    // 输出: Current function: log_function
}
```

#### 数组参数限定符

```c
void process(int arr[static 10]);   // 数组至少有 10 个元素
void copy(char dest[restrict], const char src[restrict]); // 指针不重叠
void read_only(const int arr[10]);  // 数组大小提示
```

### 3.2 特性测试宏

C99 定义了以下特性测试宏用于检测可选特性：

| 宏 | 说明 |
|------|------|
| `__STDC_IEC_559__` | 支持 IEEE-754 二进制浮点运算 |
| `__STDC_IEC_559_COMPLEX__` | 支持 IEEE-754 复数运算 |
| `__STDC_HOSTED__` | 宿主环境实现（非独立实现） |
| `__STDC_ISO_10646__` | 宽字符使用 Unicode 编码 |
| `__STDC_MB_MIGHT_NEQ_WC__` | 某些基本字符在普通/宽字符编码中值不同 |

## 4. 底层原理

### 4.1 整数除法截断

C99 明确要求有符号整数除法向零截断：

```c
int a = -7 / 3;    // C99 明确结果为 -2（向零截断）
int b = -7 % 3;    // C99 明确结果为 -1（与除法一致）
```

### 4.2 main 函数隐式返回

C99 规定 `main` 函数如果执行到末尾没有返回语句，隐式返回 0：

```c
int main(void) {
    printf("Hello, World!\n");
    // 隐式返回 0（C99）
}
```

### 4.3 restrict 指针优化

`restrict` 关键字告诉编译器指针是访问对象的唯一方式，允许更激进的优化：

```c
void vector_add(double *restrict c,
                const double *restrict a,
                const double *restrict b,
                int n) {
    for (int i = 0; i < n; i++) {
        c[i] = a[i] + b[i];  // 编译器可以安全向量化
    }
}
```

### 4.4 浮点环境

C99 引入了浮点环境访问能力：

```c
#include <fenv.h>

#pragma STDC FENV_ACCESS ON

void compute() {
    feclearexcept(FE_ALL_EXCEPT);    // 清除异常标志
    
    double result = some_calculation();
    
    if (fetestexcept(FE_DIVBYZERO)) {
        printf("Division by zero occurred\n");
    }
}
```

| 标准 pragma | 说明 |
|-------------|------|
| `FENV_ACCESS` | 允许访问浮点环境 |
| `FP_CONTRACT` | 允许浮点表达式收缩（如 FMA） |
| `CX_LIMITED_RANGE` | 允许复数乘除简化 |

```c
                 const void *restrict src,
```

### 5.3 常见陷阱

#### 变长数组的栈溢出风险

```c
// ❌ 危险：可能导致栈溢出
void process_large(int n) {
    int arr[n];  // 如果 n 很大，会溢出栈
}

// ✅ 正确：使用动态内存分配
void process_large(int n) {
    int *arr = malloc(n * sizeof(int));
    if (arr) {
        // 使用 arr
        free(arr);
    }
}
```

#### restrict 违反契约

```c
// ❌ 错误：违反 restrict 契约
void bad_example() {
    int arr[10];
    int *p1 = arr;
    int *p2 = arr + 5;  // 与 p1 重叠
    
    // 如果函数声明了 restrict 参数，这是未定义行为
    process_both(p1, p2);
}
```

### 6.2 高级用法

#### 复数运算

```c
#include <stdio.h>
#include <complex.h>
#include <math.h>

int main(void) {
    double complex z1 = 3.0 + 4.0 * I;
    double complex z2 = CMPLX(1.0, 1.0);
    
    // 基本运算
    double complex sum = z1 + z2;
    double complex prod = z1 * z2;
    double complex conj_z1 = conj(z1);  // 共轭
    
    // 模和辐角
    double magnitude = cabs(z1);        // |z1| = 5.0
    double phase = carg(z1);            // arg(z1) ≈ 0.927 rad
    
    printf("z1 = %.1f + %.1fi\n", creal(z1), cimag(z1));
    printf("|z1| = %.1f\n", magnitude);
    printf("arg(z1) = %.3f rad\n", phase);
    
    // 指数和对数
    double complex exp_z = cexp(z2);    // e^z2
    double complex log_z = clog(z1);    // ln(z1)
    
    // 三角函数
    double complex sin_z = csin(z1);
    double complex cos_z = ccos(z1);
    
    return 0;
}
```

#### 浮点环境控制

```c
#include <stdio.h>
#include <fenv.h>
#include <math.h>

void check_fp_exceptions() {

    printf("向上舍入: %.1f\n", 1.2);  // 2.0
    
    fesetround(FE_DOWNWARD);
    printf("向下舍入: %.1f\n", 1.8);  // 1.0
    
    fesetround(FE_TONEAREST);
    printf("就近舍入: %.1f\n", 1.5);  // 2.0
}

int main(void) {
    check_fp_exceptions();
    rounding_demo();
    return 0;
```

### 6.3 常见错误及修正

#### 错误使用变长数组

```c
// ❌ 错误：VLA 大小可能过大
int get_input() {
    int n;
    scanf("%d", &n);
    int arr[n];  // 潜在的栈溢出
    return 0;
}

// ✅ 正确：检查大小并使用堆分配
int get_input() {
    int n;
    if (scanf("%d", &n) != 1 || n <= 0 || n > 1000000) {
        return -1;
    }
    int *arr = malloc(n * sizeof(int));
    if (!arr) return -1;
    
    // 使用 arr
    
    free(arr);
    return 0;
}
```

#### restrict 指针误用

```c
// ❌ 错误：违反 restrict 契约
void add_arrays(int n, int *restrict a, int *restrict b) {
    for (int i = 0; i < n; i++) {
        a[i] = a[i] + b[i];
    }
}

int main() {
    int arr[10];
    add_arrays(10, arr, arr);  // 未定义行为！
}

// ✅ 正确：确保指针不重叠
int main() {
    int a[10], b[10];
    add_arrays(10, a, b);  // 安全
}
```

## 7. 总结

### 核心要点

| 类别 | 主要特性 |
|------|----------|
| **类型增强** | `_Bool`、`long long`、`_Complex`、`_Imaginary` |
| **数组改进** | 变长数组（VLA）、灵活数组成员、指定初始化器 |
| **函数改进** | 内联函数、`restrict` 指针、数组参数限定符 |
| **预处理器** | 变参宏、`_Pragma` 运算符、`__func__` 标识符 |
| **库扩展** | `<stdint.h>`、`<stdbool.h>`、`<complex.h>`、`<fenv.h>` |

### 与其他版本对比

| 特性 | C90 | C99 | C11 |
|------|-----|-----|-----|
| 单行注释 `//` | ❌ | ✅ | ✅ |
| 变长数组 | ❌ | ✅ | 可选 |
| 布尔类型 | ❌ | ✅ | ✅ |
| 复数类型 | ❌ | ✅ | ✅ |
| `_Generic` | ❌ | ❌ | ✅ |
| 原子操作 | ❌ | ❌ | ✅ |
| 线程支持 | ❌ | ❌ | ✅ |

### 学习建议

1. **掌握核心新特性**：重点学习布尔类型、变长数组、指定初始化器和内联函数
2. **理解 restrict 语义**：正确使用 `restrict` 可以显著提升性能
3. **注意可移植性**：VLA 在 C11 中变为可选，建议在关键场景使用动态内存分配
4. **利用新库头文件**：`<stdint.h>` 和 `<stdbool.h>` 提高代码可移植性
5. **浮点环境**：科学计算场景充分利用 `<complex.h>` 和 `<fenv.h>`
