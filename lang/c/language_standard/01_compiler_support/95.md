# C95 编译器支持

## 1. 概述

C95（ISO/IEC 9899:1990/AMD1:1995）是 C90 标准的规范性修正案（Normative Amendment），于 1995 年发布。它作为 C90 的补充，修复了若干缺陷并添加了少量新特性。

C95 是一个相对较小的更新，主要关注国际化支持和代码可移植性改进。所有现代 C 编译器都完全支持 C95，无需特殊标志即可使用其特性。

## 2. 来源与演变

### 史背景

C95 的全称是 **C90 技术勘误与修正案 1**（C90 Technical Corrigenda and Amendment 1），是国际标准化组织（ISO）对 C90 标准的官方修订。

| 时间节点 | 事件 |
|---------|------|
| 1990 | ISO/IEC 9899:1990（C90）发布 |
| 1995 | ISO/IEC 9899:1990/AMD1:1995（C95）发布 |
| 1999 | ISO/IEC 9899:1999（C99）取代 C95 |

### 设计动机

C95 的主要目标包括：

1. **国际化支持**：引入宽字符和多字节字符处理能力
2. **代码可移植性**：提供双字符组（digraphs）作为某些字符的替代
3. **缺陷修复**：修正 C90 标准中的已知问题
4. **标准版本标识**：引入 `__STDC_VERSION__` 宏用于版本检测

### 与其他标准的关系

```
C90 → C95（修正案）→ C99 → C11 → C17 → C23
```

C95 不替代 C90，而是作为其补充。C99 标准发布后，C95 的所有特性被完整继承。

## 3. 语法与参数

### 核心语言特性

C95 新增的核心语言特性如下：

| 特性 | 说明 | 示例 |
|------|------|------|
| 双字符组（Digraphs） | 提供某些符号的可替代写法 | `<%` 替代 `{` |
| `__STDC_VERSION__` 宏 | 编译时版本检测宏 | 值为 `199409L` |

#### 双字符组

双字符组是为了解决某些环境（如 ISO 646 字符集）中特定字符不可用的问题：

| 双字符组 | 等价符号 | 说明 |
|---------|---------|------|
| `<%` | `{` | 左花括号 |
| `%>` | `}` | 右花括号 |
| `<:` | `[` | 左方括号 |
| `:>` | `]` | 右方括号 |
| `%:` | `#` | 预处理符号 |

#### `__STDC_VERSION__` 宏

```c
#if defined(__STDC_VERSION__)
    #if __STDC_VERSION__ >= 199409L
        /* C95 或更高版本 */
    #endif
#endif
```

| 标准版本 | 宏值 |
|---------|------|
| C90 | 未定义 |
| C95 | `199409L` |
| C99 | `199901L` |
| C11 | `201112L` |
| C17 | `201710L` |

### 标准库特性

C95 新增了三个标准头文件：

| 头文件 | 功能 |
|--------|------|
| `<iso646.h>` | 替代标记（alternative tokens） |
| `<wchar.h>` | 宽字符处理工具 |
| `<wctype.h>` | 宽字符分类与映射 |

#### `<iso646.h>` 替代标记

为解决某些字符集中位运算符不可用的问题：

| 宏定义 | 替代符号 |
|--------|---------|
| `and` | `&&` |
| `or` | `\|\|` |
| `not` | `!` |
| `bitand` | `&` |
| `bitor` | `\|` |
| `xor` | `^` |
| `compl` | `~` |
| `and_eq` | `&=` |
| `or_eq` | `\|=` |
| `xor_eq` | `^=` |
| `not_eq` | `!=` |

#### `<wchar.h>` 宽字符函数

提供宽字符输入输出和字符串处理：

```c
#include <wchar.h>

wchar_t wc = L'A';              // 宽字符字面量
wchar_t ws[] = L"宽字符串";      // 宽字符串
size_t len = wcslen(ws);        // 宽字符串长度
```

#### `<wctype.h>` 宽字符分类

宽字符版本的字符分类函数：

| 函数 | 说明 |
|------|------|
| `iswalpha()` | 是否为字母 |
| `iswdigit()` | 是否为数字 |
| `iswspace()` | 是否为空白 |
| `iswlower()` | 是否为小写 |
| `iswupper()` | 是否为大写 |
| `towlower()` | 转换为小写 |
| `towupper()` | 转换为大写 |

### 扩展多字节/宽字符函数

C95 扩展了多字节和宽字符之间的转换能力：

```c
#include <wchar.h>
#include <stdlib.h>

// 多字节转宽字符
size_t mbstowcs(wchar_t *pwcs, const char *s, size_t n);

// 宽字符转多字节
size_t wcstombs(char *s, const wchar_t *pwcs, size_t n);

// 单字符转换
int wctomb(char *s, wchar_t wc);
int mbtowc(wchar_t *pwc, const char *s, size_t n);
```

## 4. 底层原理

### 双字符组的处理机制

双字符组在**词法分析阶段**被转换为对应的单字符。编译器在预处理之前就将这些双字符组替换，因此它们可以用于任何需要对应符号的地方。

处理流程：

```
源代码（含双字符组）→ 词法分析 → 符号替换 → 预处理 → 编译
```

### 版本宏的实现

`__STDC_VERSION__` 宏由编译器预定义，其值表示编译器支持的 C 标准版本：

- 宏在 C90 中未定义
- C95 中定义为 `199409L`（1994 年 9 月）
- 后续标准中更新为对应版本号

### 宽字符支持机制

C95 的宽字符支持基于以下设计：

1. **编码独立性**：`wchar_t` 类型大小由实现定义，可适应不同编码
2. **流支持**：宽字符流（如 `stdin`）与字节流分离
3. **本地化支持**：宽字符函数受 `setlocale()` 影响

```
字节字符 ←→ 多字节字符 ←→ 宽字符
   (char)        (char[])      (wchar_t)
```

## 5. 使用场景

### 适用场景

| 场景 | 推荐使用的特性 |
|------|---------------|
| 国际化程序 | `<wchar.h>`, `<wctype.h>` |
| 可移植代码 | 双字符组, `<iso646.h>` |
| 版本条件编译 | `__STDC_VERSION__` |
| 跨平台开发 | 所有 C95 特性 |

### 编译器支持情况

C95 得到所有主流编译器的普遍支持：

#### 核心语言特性支持

| C95 特性 | GCC | Clang | MSVC | Apple Clang | Intel C++ |
|---------|-----|-------|------|-------------|-----------|
| 双字符组 | Yes | Yes | Yes | Yes | Yes |
| `__STDC_VERSION__` 宏 | 2.95+ | Yes | Yes | Yes | Yes |

#### 标准库特性支持

| C95 特性 | GCC | Clang | MSVC | Apple Clang | Intel C++ |
|---------|-----|-------|------|-------------|-----------|
| `<iso646.h>` 头文件 | Yes | Yes | Yes | Yes | Yes |
| `<wchar.h>` 头文件 | Yes | Yes | Yes | Yes | Yes |
| `<wctype.h>` 头文件 | Yes | Yes | Yes | Yes | Yes |
| 扩展多字节/宽字符函数 | Yes | Yes | Yes | Yes | Yes |

### 编译器标志

| 编译器 | 最低版本 | 标志 |
|--------|---------|------|
| GCC | 2.95+ | `-std=c95` 或 `-std=iso9899:199409` |
| Clang | 所有版本 | `-std=c95` |
| MSVC | 所有版本 | 默认支持 |
| Intel C++ | 所有版本 | 默认支持 |

### 最佳实践

#### 版本检测

使用 `__STDC_VERSION__` 进行条件编译：

```c
#if defined(__STDC_VERSION__) && __STDC_VERSION__ >= 199409L
    /* 使用 C95 特性 */
    #include <wchar.h>
#else
    /* 降级处理 */
#endif
```

#### 国际化程序设计

对于需要支持多语言的程序：

```c
#include <wchar.h>
#include <locale.h>

int main(void) {
    setlocale(LC_ALL, "");  // 设置本地化环境

    wchar_t greeting[] = L"你好，世界";
    wprintf(L"%ls\n", greeting);

    return 0;
}
```

### 注意事项

1. **向后兼容性**：C95 完全向后兼容 C90，所有 C90 代码可在 C95 模式下编译
2. **过渡性质**：C95 是一个过渡标准，现代项目建议直接使用 C99 或更高版本
3. **宽字符大小**：`wchar_t` 的大小在不同平台不同（Windows 上 2 字节，Linux/macOS 上 4 字节）
4. **双字符组使用**：现代代码中极少使用双字符组，建议直接使用标准符号

### 常见陷阱

| 陷阱 | 说明 | 解决方案 |
|------|------|---------|
| 宽字符移植性 | `wchar_t` 大小不一致 | 使用 C11 的 `char16_t`/`char32_t` |
| 版本宏缺失 | C90 编译器无版本宏 | 同时检查 `__STDC__` |
| 本地化未设置 | 宽字符函数依赖本地化设置 | 使用前调用 `setlocale()` |

## 6. 代码示例

### 基础用法：版本检测

```c
#include <stdio.h>

int main(void) {
#if defined(__STDC_VERSION__)
    #if __STDC_VERSION__ >= 201710L
        printf("C17 或更高版本\n");
    #elif __STDC_VERSION__ >= 201112L
        printf("C11\n");
    #elif __STDC_VERSION__ >= 199901L
        printf("C99\n");
    #elif __STDC_VERSION__ >= 199409L
        printf("C95\n");
    #else
        printf("未知标准版本\n");
    #endif
#else
    printf("C90 或更早版本\n");
#endif

    return 0;
}
```

### 宽字符处理

```c
#include <stdio.h>
#include <wchar.h>
#include <locale.h>
#include <wctype.h>

int main(void) {
    // 设置本地化环境以支持宽字符输出
    setlocale(LC_ALL, "");

    // 宽字符字面量
    wchar_t wc = L'A';
    wchar_t ws[] = L"Hello 世界";

    // 宽字符字符串长度
    size_t len = wcslen(ws);
    wprintf(L"字符串: %ls, 长度: %zu\n", ws, len);

    // 宽字符分类
    if (iswalpha(wc)) {
        wprintf(L"'%lc' 是字母\n", wc);
    }

    // 宽字符大小写转换
    wchar_t upper = towupper(L'a');
    wprintf(L"大写: %lc\n", upper);

    return 0;
}
```

### 多字节与宽字符转换

```c
#include <stdio.h>
#include <stdlib.h>
#include <wchar.h>
#include <locale.h>

int main(void) {
    setlocale(LC_ALL, "");

    // 多字节字符串
    char mbs[] = "Hello 世界";

    // 计算所需宽字符数量
    size_t wlen = mbstowcs(NULL, mbs, 0);
    wchar_t *wcs = malloc((wlen + 1) * sizeof(wchar_t));

    // 转换
    mbstowcs(wcs, mbs, wlen + 1);
    wprintf(L"宽字符串: %ls\n", wcs);

    // 转回多字节
    char *buf = malloc(MB_CUR_MAX * wlen + 1);
    wcstombs(buf, wcs, MB_CUR_MAX * wlen + 1);
    printf("多字节字符串: %s\n", buf);

    free(wcs);
    free(buf);

    return 0;
}
```

### 替代标记示例

```c
#include <stdio.h>
#include <iso646.h>

int main(void) {
    int a = 5, b = 10;

    // 使用 iso646.h 中的替代标记
    if (a > 0 and b > 0) {
        printf("两个数都为正\n");
    }

    if (a < 0 or b < 0) {
        printf("至少一个数为负\n");
    }

    if (not (a == b)) {
        printf("a 不等于 b\n");
    }

    // 位运算替代
    int c = a bitand b;  // 等价于 a & b
    int d = a bitor b;   // 等价于 a | b
    int e = a xor b;     // 等价于 a ^ b

    printf("位运算结果: %d, %d, %d\n", c, d, e);

    return 0;
}
```

### 双字符组示例（不推荐使用）

```c
/* 仅作演示，现代代码中不推荐使用双字符组 */
<%  /* 等价于 { */
    int arr<::> = <:1, 2, 3:>;  /* 等价于 int arr[] = {1, 2, 3}; */
%>   /* 等价于 } */
```

## 7. 总结

### 核心要点

| 要点 | 说明 |
|------|------|
| 定义 | C95 是 C90 的规范性修正案，非独立标准 |
| 主要贡献 | 引入宽字符支持、双字符组、版本宏 |
| 兼容性 | 完全向后兼容 C90 |
| 现代意义 | 作为 C 标准演进的里程碑，特性已被后续标准继承 |

### 与其他 C 标准对比

| 特性 | C90 | C95 | C99 | C11 |
|------|-----|-----|-----|-----|
| 双字符组 | 无 | 有 | 有 | 有 |
| `__STDC_VERSION__` | 无 | 有 | 有 | 有 |
| `<wchar.h>` | 无 | 有 | 有 | 有 |
| 变长数组 | 无 | 无 | 有 | 可选 |
| 内联函数 | 无 | 无 | 有 | 有 |
| 泛型选择 | 无 | 无 | 无 | 有 |

### 学习建议

1. **了解历史**：理解 C95 有助于把握 C 语言标准演进脉络
2. **关注特性**：宽字符支持是国际化编程的基础
3. **版本检测**：`__STDC_VERSION__` 是可移植代码的重要工具
4. **适度使用**：双字符组和 `<iso646.h>` 在现代代码中几乎不使用
