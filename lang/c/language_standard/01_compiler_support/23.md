# C23 编译器支持

## 1. 概述

C23（原称 C2x）是 C 语言的最新标准版本，正式名称为 ISO/IEC 9899:2024。本文档详细记录了主流编译器对 C23 核心语言特性和库特性的支持情况，帮助开发者了解各编译器的兼容性状态。

### 主要内容

- **核心语言特性**：新增语法、属性、类型系统等编译器支持矩阵
- **库特性**：标准库新特性的支持状态

### 支持的编译器

| 编译器 | 说明 |
|--------|------|
| GCC | GNU Compiler Collection |
| Clang | LLVM 编译器前端 |
| MSVC | Microsoft Visual C++ |
| Apple Clang | Apple 版 Clang |
| EDG eccp | Edison Design Group 编译器 |
| Intel C++ | Intel C++ 编译器 |
| Nvidia HPC C++ | 高性能计算编译器（原 PGI） |
| Nvidia nvcc | Nvidia CUDA 编译器 |
| Cray | Cray 编译器 |

## 2. 来源与演变

### 标准发展历程

| 时间节点 | 事件 |
|----------|------|
| 2018 年 | C2x 标准化工作启动 |
| 2023 年 | C23 标准正式发布（ISO/IEC 9899:2024） |
| 持续更新 | 各编译器逐步实现新特性 |

### 设计动机

C23 标准的主要改进方向：

1. **与现代 C++ 对齐**：引入属性（attribute）、`constexpr` 等现代特性
2. **提高类型安全性**：引入 `nullptr`、改进枚举类型
3. **增强可移植性**：统一 Unicode 标识符、改进位精确整数
4. **简化编码**：新增语法糖、预处理器增强

## 5. 使用场景

### 选择合适的编译器版本

根据所需 C23 特性，选择支持该特性的编译器版本：

```bash
# GCC 编译 C23 代码
gcc -std=c23 -Wall -Wextra source.c

# Clang 编译 C23 代码
clang -std=c23 -Wall -Wextra source.c

# 检查特性支持（预处理器）
#if __has_c_attribute(nodiscard)
[[nodiscard]] int get_value(void);
#endif
```

### 条件编译最佳实践

使用 `__has_c_attribute` 和 `__has_include` 进行条件编译：

```c
// 检查属性支持
#if defined(__has_c_attribute)
#  if __has_c_attribute(fallthrough)
#    define FALLTHROUGH [[fallthrough]]
#  else
#    define FALLTHROUGH ((void)0)
#  endif
#endif

// 检查头文件支持
#if defined(__has_include)
#  if __has_include(<stdbit.h>)
#    include <stdbit.h>
#  endif
#endif
```

### 移植性注意事项

1. **MSVC 支持有限**：Microsoft Visual C++ 对 C23 的支持相对滞后，部分特性不可用
2. **Apple Clang 差异**：Apple Clang 版本号与上游 Clang 不同，需查阅对应文档
3. **部分支持**：某些特性标注为"部分"支持，表示功能不完整或有差异

## 6. 代码示例

### 属性使用示例

```c
// C23 属性示例

// nodiscard: 必须使用返回值
[[nodiscard]] int get_status(void);

// nodiscard 带消息
[[nodiscard("忽略返回值可能导致资源泄漏")]]
int allocate_resource(void);

// maybe_unused: 抑制未使用警告
[[maybe_unused]] static void debug_print(void) {
    // 调试代码
}

// deprecated: 标记废弃
[[deprecated("使用 new_func 替代")]]
void old_func(void);

// fallthrough: 显式贯穿
switch (value) {
    case 1:
        process_one();
        [[fallthrough]];
    case 2:
        process_two();
        break;
}

// noreturn: 函数不返回
[[noreturn]] void fatal_error(const char *msg) {
    fprintf(stderr, "Fatal: %s\n", msg);
    exit(1);
}
```

### nullptr 和布尔类型

```c
// C23 之前
#define NULL ((void*)0)

// C23
#include <stddef.h>

void* ptr = nullptr;  // 类型安全的空指针

// 布尔类型内置
bool flag = true;   // 无需包含 <stdbool.h>
bool done = false;
```

### 类型推断和 typeof

```c
// 类型推断（C23）
auto x = 42;           // x 为 int
auto y = 3.14;         // y 为 double
auto ptr = &x;         // ptr 为 int*

// typeof 运算符
int a = 10;
typeof(a) b = 20;      // b 为 int
typeof_unqual(a) c;    // 去除限定符
```

### 二进制字面量和数字分隔符

```c
// 二进制字面量
int binary = 0b101010;        // 42
int flags = 0b11110000;       // 位标志

// 数字分隔符（提高可读性）
int million = 1'000'000;
long credit_card = 1234'5678'9012'3456;
double pi = 3.141'592'653'589;
int binary_grouped = 0b1010'1100'1111;
```

### constexpr 对象

```c
// C23 constexpr
constexpr int MAX_SIZE = 100;
constexpr double PI = 3.141592653589793;

// 编译期计算
constexpr int factorial(int n) {
    return n <= 1 ? 1 : n * factorial(n - 1);
}
```

### #embed 指令

```c
// C23: 嵌入二进制资源
const unsigned char icon[] = {
#embed "icon.png"
};

const char config[] = {
#embed "config.json"
, '\0'  // 添加终止符
};
```

### 枚举固定底层类型

```c
// C23: 指定枚举底层类型
enum SmallFlags : unsigned char {
    FLAG_A = 1,
    FLAG_B = 2,
    FLAG_C = 4
};

enum LargeValues : long long {
    BIG_VALUE = 0x7FFFFFFFFFFFFFFFLL
};
```

## 7. 总结

### 核心要点

1. **GCC 和 Clang 领先**：GCC 13+ 和 Clang 15+ 已支持大部分 C23 核心特性
2. **MSVC 支持有限**：Microsoft 编译器对 C23 支持较慢，建议关注更新
3. **属性是重点**：`[[nodiscard]]`、`[[maybe_unused]]`、`[[deprecated]]` 等属性提高代码表达力
4. **类型安全增强**：`nullptr`、`constexpr`、类型推断等特性提升类型安全性

### 编译器版本建议

| 需求 | 推荐编译器版本 |
|------|----------------|
| 基本属性支持 | GCC 10+ / Clang 9+ |
| 大部分 C23 特性 | GCC 13+ / Clang 16+ |
| 完整 C23 支持 | GCC 14+ / Clang 19+ |
| 生产环境 | GCC 13+ / Clang 18+ |

### 相关技术对比

| 特性 | C23 | C11 | C++17 | C++20 |
|------|-----|-----|-------|-------|
| 属性 | ✓ | ✗ | ✓ | ✓ |
| `nullptr` | ✓ | ✗ | ✓ | ✓ |
| `constexpr` | ✓ (对象) | ✗ | ✓ | ✓ |
| 类型推断 | ✓ (`auto`) | ✗ | ✓ | ✓ |
| 二进制字面量 | ✓ | ✗ | ✓ | ✓ |

### 学习建议

1. **循序渐进**：从属性和 `nullptr` 等基础特性开始
2. **查阅文档**：使用 `__has_c_attribute` 等宏检查编译器支持
3. **关注更新**：编译器支持状态持续更新，关注官方文档
4. **实践验证**：重要特性在实际项目中验证后再大规模使用
