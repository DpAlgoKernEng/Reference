# C90 编译器支持

## 1. 概述

C90（ISO/IEC 9899:1990），又称 C89 或 ANSI C，是 C 语言的第一个国际标准。所有现代 C 编译器都将 C90 作为基准支持，这是最广泛支持的 C 语言版本，也是跨平台可移植性的基准标准。

C90 确立了 C 语言的核心语法特性、类型系统和标准库框架，为后续 C99、C11、C17、C23 标准奠定了坚实基础。预定义宏（predefined macro）`__STDC__` 在符合标准的编译器中定义为 1。

## 2. 来源与演变

### 标准化历程

| 时间节点 | 事件 | 说明 |
|----------|------|------|
| 1978 | K&R C | Brian Kernighan 与 Dennis Ritchie 合著《The C Programming Language》 |
| 1983 | ANSI 成立 X3J11 委员会 | 开始 C 语言标准化工作 |
| 1989 | ANSI X3.159-1989 | 美国国家标准协会发布 ANSI C |
| 1990 | ISO/IEC 9899:1990 | 国际标准化组织采纳，成为国际标准 |

### C90 与 K&R C 的差异

| 特性 | K&R C | C90 |
|------|-------|-----|
| 函数声明（function declaration） | 仅旧式风格 | 函数原型（prototype）+ 旧式（已弃用） |
| `void` 关键字 | 不可用 | 可用 |
| `const`/`volatile` 限定符 | 不可用 | 可用 |
| 标准库 | 有限 | 15 个标准头文件 |
| 预处理器（preprocessor） | 有限 | 增强 `#elif`、`#error`、`#pragma` |
| 结构体操作 | 不支持赋值 | 支持赋值和传参 |

### 版本演进关系

```
K&R C (1978)
    ↓
C90 (ISO/IEC 9899:1990) ← 基准标准
    ↓
C95 (ISO/IEC 9899:1990/AMD1:1995)
    ↓
C99 (ISO/IEC 9899:1999)
    ↓
C11 (ISO/IEC 9899:2011)
    ↓
C17 (ISO/IEC 9899:2018)
    ↓
C23 (ISO/IEC 9899:2024)
```

## 4. 底层原理

### 翻译限制（Translation Limits）

标准规定了编译器必须支持的最小翻译限制：

| 限制项 | 最小值 | 说明 |
|--------|--------|------|
| 复合语句嵌套层数 | 15 | `{}` 块嵌套深度 |
| 条件编译嵌套层数 | 8 | `#if`/`#ifdef` 嵌套 |
| 指针/数组/函数声明符嵌套 | 12 | 类型声明复杂度 |
| 括号表达式嵌套层数 | 15 | 表达式复杂度 |
| 函数调用参数个数 | 31 | 实参数量 |
| 函数定义参数个数 | 31 | 形参数量 |
| 结构体/联合体定义嵌套层数 | 15 | 成员定义复杂度 |
| 单个结构体/联合体成员数 | 127 | 成员数量上限 |
| 单个枚举常量数 | 127 | 枚举值数量 |
| 外部标识符有效字符数 | 6 | 链接器限制 |
| 内部标识符有效字符数 | 31 | 编译器内部限制 |
| 单个编译单元外部标识符数 | 511 | 全局符号数量 |
| 单个块内块作用域标识符数 | 127 | 局部变量数量 |

### 预定义宏机制

| 宏 | 值/格式 | 描述 |
|----|---------|------|
| `__FILE__` | 字符串字面量 | 当前源文件名 |
| `__LINE__` | 整数常量 | 当前行号 |
| `__DATE__` | `"Mmm dd yyyy"` | 编译日期 |
| `__TIME__` | `"hh:mm:ss"` | 编译时间 |
| `__STDC__` | `1` | 符合 ISO C 标准 |

### 函数原型实现原理

函数原型（function prototype）是 C90 的核心改进：

```
编译阶段：
1. 解析函数声明 → 记录参数类型信息
2. 遇到函数调用 → 检查参数类型匹配
3. 类型不匹配 → 编译错误/隐式转换
4. 生成目标代码 → 参数按声明类型传递
```

与 K&R C 相比，函数原型提供了：
- **编译时类型检查**：捕获类型不匹配错误
- **参数类型转换**：自动进行隐式类型转换
- **代码文档化**：函数签名即文档

## 5. 使用场景

### 适用场景

| 场景 | 推荐理由 |
|------|----------|
| 嵌入式开发 | 编译器支持广泛，资源占用最小 |
| 跨平台库开发 | 最广泛兼容性，任何平台均可编译 |
| 遗留代码维护 | 大量 K&R C 代码需要迁移 |
| 安全关键系统 | 经过长期验证，行为确定性强 |
| 教学入门 | 语言特性精简，适合学习基础 |

### 最佳实践

```c
/* ✅ 推荐：使用函数原型 */
int add(int a, int b);
void process(const char *str);
int get_value(void);  /* 明确表示无参数 */

/* ❌ 避免：K&R 风格函数声明 */
int add();  /* 无参数信息，类型不安全 */

/* ✅ 推荐：使用 const 保护只读数据 */
size_t str_len(const char *str);
void print_array(const int *arr, size_t n);

/* ✅ 推荐：使用 void* 实现泛型接口 */
void swap(void *a, void *b, size_t size);
```

### 注意事项

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| 外部标识符长度 | 仅保证前 6 个字符有效 | 使用前缀区分，避免仅后几位不同 |
| 整数提升规则 | `char`/`short` 自动提升为 `int` | 显式转换为所需类型 |
| 无原型函数调用 | 参数类型不检查 | 始终提供函数原型 |
| 隐式 int 声明 | C90 允许省略返回类型 | 显式声明返回类型 |

## 6. 代码示例

### 基础特性演示

```c
/* c90_features.c - C90 核心特性演示 */
#include <stdio.h>
#include <stdlib.h>

/* 函数原型（function prototype） */
int factorial(int n);
void print_info(const char *msg);

/* 枚举类型 */
enum status {
    STATUS_OK = 0,
    STATUS_ERROR = -1
};

/* 结构体定义 */
struct point {
    int x;
    int y;
};

int main(void) {
    /* void 指针 */
    void *generic_ptr;
    int value = 42;
    generic_ptr = &value;
    
    /* const 限定符 */
    const int max_size = 100;
    
    /* volatile 限定符（硬件寄存器模拟） */
    volatile int hardware_reg = 0;
    
    /* 结构体赋值 */
    struct point p1 = {10, 20};
    struct point p2 = p1;  /* 结构体复制 */
    
    /* 宽字符 */
    wchar_t wc = L'A';
    
    printf("factorial(5) = %d\n", factorial(5));
    printf("point: (%d, %d)\n", p2.x, p2.y);
    
    /* 预定义宏 */
    printf("File: %s, Line: %d\n", __FILE__, __LINE__);
    printf("Compiled: %s %s\n", __DATE__, __TIME__);
    
    return STATUS_OK;
}

int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}
```

### 预处理器使用

```c
/* c90_preprocessor.c - 预处理器特性演示 */
#include <stdio.h>

/* #pragma 指令 */
#pragma warning(push)
#pragma warning(disable: 4996)  /* MSVC 禁用警告 */

/* #error 指令 - 编译时检查 */
#ifndef __STDC__
#error "This code requires an ISO C compiler"
#endif

/* 条件编译 */
#ifdef DEBUG
    #define LOG(msg) printf("[DEBUG] %s\n", msg)
#else
    #define LOG(msg) ((void)0)
#endif

/* 平台检测 */
#if defined(_WIN32)
    #define PLATFORM "Windows"
#elif defined(__linux__)
    #define PLATFORM "Linux"
#elif defined(__APPLE__)
    #define PLATFORM "macOS"
#else
    #define PLATFORM "Unknown"
#endif

int main(void) {
    /* #elif 演示 */
    #if __STDC_VERSION__ >= 199901L
        printf("C99 or later\n");
    #elif __STDC__ == 1
        printf("C90 (ANSI C)\n");
    #else
        printf("Pre-ANSI C\n");
    #endif
    
    printf("Platform: %s\n", PLATFORM);
    LOG("Application started");
    
    return 0;
}
```

### 编译配置示例

```makefile
# Makefile - C90 编译配置
CC = gcc
CFLAGS = -std=c90 -Wall -Wextra -pedantic
LDFLAGS = 

SRCS = main.c utils.c
OBJS = $(SRCS:.c=.o)
TARGET = program

all: $(TARGET)

$(TARGET): $(OBJS)
	$(CC) $(LDFLAGS) -o $@ $^

%.o: %.c
	$(CC) $(CFLAGS) -c -o $@ $<

clean:
	rm -f $(OBJS) $(TARGET)

# 严格 C90 模式
strict: CFLAGS += -pedantic-errors -Werror
strict: clean all
```

```cmake
# CMakeLists.txt - C90 项目配置
cmake_minimum_required(VERSION 3.10)
project(C90Project C)

# 设置 C 标准
set(CMAKE_C_STANDARD 90)
set(CMAKE_C_STANDARD_REQUIRED ON)

# 编译选项
add_compile_options(-Wall -Wextra -pedantic)

# 可执行文件
add_executable(program main.c utils.c)
```

## 7. 总结

### 核心要点

| 类别 | 要点 |
|------|------|
| 定位 | 第一个 C 语言国际标准，所有编译器的基准 |
| 兼容性 | 最广泛支持，任何平台均可编译 |
| 主要改进 | 函数原型、类型限定符、标准库、预处理器增强 |
| 后续标准 | C99/C11/C17/C23 均向后兼容 C90 |

### 编译器支持汇总

| 编译器 | 支持状态 | 推荐版本 |
|--------|----------|----------|
| GCC | ✅ 完全支持 | 所有版本 |
| Clang | ✅ 完全支持 | 所有版本 |
| MSVC | ✅ 完全支持 | 所有版本 |
| Intel C++ | ✅ 完全支持 | 所有版本 |
| ARM GCC | ✅ 完全支持 | 所有版本 |

### 迁移建议

| 从 | 到 | 建议 |
|----|----|------|
| K&R C | C90 | 添加函数原型，使用 `const`/`void` |
| C90 | C99 | 可选使用新特性，代码无需修改 |
| C90 | C11 | 可选使用线程/原子操作，代码无需修改 |
