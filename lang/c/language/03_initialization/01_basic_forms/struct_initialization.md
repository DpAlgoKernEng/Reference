# 结构体与联合体初始化 (Struct and Union Initialization)

## 1. 概述

结构体（struct）和联合体（union）的初始化是 C 语言中用于为聚合类型（aggregate type）对象设置初始值的重要机制。初始化时使用花括号包围的初始化列表，列表中的初始化器按顺序或通过指示符指定对应的成员。

初始化列表提供了一种简洁、类型安全的方式来设置结构体和联合体成员的初始值，是 C 语言聚合类型初始化的核心特性。

## 2. 来源与演变

### C89/C90 标准

在早期的 C 语言标准中，结构体和联合体的初始化只能使用顺序初始化列表：

```c
struct point {double x, y, z;} p = {1.2, 1.3};
```

此时：
- 初始化列表必须是非空的
- 初始化器必须按成员声明顺序提供
- 所有表达式必须是常量表达式（对于静态存储期的聚合类型）
- 不支持指定初始化器

### C99 标准

C99 标准引入了重大改进：

1. **指定初始化器（Designated Initializers）**：允许通过成员名或数组索引指定要初始化的成员
2. **非常量表达式**：对于自动存储期的对象，初始化表达式可以是运行时表达式
3. **灵活的初始化顺序**：使用指示符可以按任意顺序初始化成员

```c
struct tm my_time = { .tm_year=2012-1900, .tm_mon=9, .tm_mday=9 };
```

### C11 标准

C11 标准扩展了初始化规则：
- 线程局部存储期（thread-local storage duration）的对象也需要常量表达式初始化
- 保持了 C99 的指定初始化器特性

### C23 标准

C23 标准进一步放宽了限制：
- 允许空初始化列表 `{}`
- 未显式初始化的成员自动进行空初始化

```c
struct {int n;} s = {};  // C23: s.n 初始化为 0
```

## 3. 语法与参数

### 基本语法

#### C89 语法

```c
= { expression, ... }
```

#### C99 及以后语法

```c
= { designator(可选) expression, ... }
= { }
```

其中指示符（designator）是以下形式的序列：

| 指示符类型 | 语法 | 说明 |
|-----------|------|------|
| 成员指示符 | `.member` | 指定结构体成员 |
| 数组指示符 | `[index]` | 指定数组元素 |

指示符可以嵌套使用，如 `.in_u.a8[0]`。

### 参数说明

| 参数 | 说明 |
|------|------|
| `expression` | 初始化表达式，用于设置成员的值 |
| `designator` | 指示符，指定要初始化的成员或数组元素 |
| `...` | 可重复的初始化项 |

### 初始化规则

1. **顺序初始化**：第一个初始化器初始化第一个声明的成员（除非使用指示符）
2. **指示符初始化**：指示符后的初始化器初始化指定的成员
3. **隐式初始化**：未显式初始化的成员自动进行空初始化
4. **联合体限制**：联合体的初始化列表只能有一个成员，除非使用指定初始化器

## 4. 底层原理

### 当前对象机制

初始化过程中维护一个"当前对象"（current object）的概念：

1. 每个左花括号 `{` 建立一个新的当前对象
2. 当前对象的成员按其自然顺序初始化（除非使用指示符）
3. 数组元素按下标顺序初始化
4. 结构体成员按声明顺序初始化
5. 联合体只初始化第一个声明的成员（除非使用指示符）

### 嵌套初始化处理

当成员是数组、结构体或联合体时：

**带花括号的嵌套初始化**：整个嵌套初始化器用于初始化对应的成员对象。

```c
struct example {
    struct addr_t { uint32_t port; } addr;
    union { uint8_t a8[4]; uint16_t a16[2]; } in_u;
};

struct example ex = {
    { 80 },           // 初始化 ex.addr
    { 127, 0, 0, 1 }  // 初始化 ex.in_u.a8
};
```

**省略花括号的嵌套初始化**：初始化器按顺序填充成员。

```c
struct example ex = {80, 127, 0, 0, 1};
// 80 初始化 ex.addr.port
// 127, 0, 0, 1 初始化 ex.in_u.a8[0..3]
```

### 初始化顺序

初始化器中子表达式的求值顺序是不确定顺序的（indeterminately sequenced）：

```c
int n = 1;
struct {int x, y;} p = {n++, n++};
// n 被递增两次，但顺序不确定
// p.x 和 p.y 可能是 {1, 2} 或 {2, 1}
```

### 重复初始化处理

如果同一子对象被多次显式初始化（使用指示符时可能发生），后出现的初始化器生效：

```c
struct {int n;} s = {
    printf("a\n"),    // 可能被跳过
    .n = printf("b\n")  // 总是执行
};
```

## 5. 使用场景

### 适合使用指定初始化器的场景

| 场景 | 说明 |
|------|------|
| 成员顺序不确定的结构体 | 如 `struct tm`，不同实现可能有不同的成员顺序 |
| 只初始化部分成员 | 只设置需要的成员，其他自动初始化为 0 |
| 提高代码可读性 | 明确显示每个值对应的成员名 |
| 大型结构体初始化 | 避免记忆成员顺序，减少错误 |

### 不适合的场景

| 场景 | 说明 |
|------|------|
| C89 兼容代码 | 指定初始化器需要 C99 及以上 |
| 简单小型结构体 | 直接顺序初始化更简洁 |
| 需要精确控制初始化顺序 | 指示符可能改变初始化顺序 |

### 最佳实践

1. **优先使用指定初始化器**：提高代码可读性和可维护性
2. **避免重复初始化**：同一成员不要多次初始化
3. **注意嵌套结构**：复杂的嵌套结构建议使用完整花括号
4. **使用尾部逗号**：方便扩展和版本控制

```c
// 推荐：使用指定初始化器
struct tm time = {
    .tm_year = 2024 - 1900,
    .tm_mon = 0,
    .tm_mday = 1,
};

// 推荐：使用尾部逗号
struct point p = {
    1.0,
    2.0,  // 尾部逗号 OK
};
```

### 注意事项

1. **初始化器数量限制**：不能提供超过成员数量的初始化器
2. **空结构体**：C 语言中结构体不能为空
3. **静态存储期限制**：静态和线程局部存储期的对象初始化需要常量表达式
4. **联合体特性**：联合体一次只有一个成员活跃

## 6. 代码示例

### 基础用法

```c
#include <stdio.h>

int main(void) {
    // 顺序初始化
    struct point { double x, y, z; } p = {1.2, 1.3};
    // p.x = 1.2, p.y = 1.3, p.z = 0.0 (空初始化)

    printf("point: (%.1f, %.1f, %.1f)\n", p.x, p.y, p.z);

    // 指定初始化器（C99）
    struct div_t { int quot, rem; } answer = {
        .quot = 2,
        .rem = -1
    };

    printf("div_t: quot=%d, rem=%d\n", answer.quot, answer.rem);

    return 0;
}
```

### 指定初始化器示例

```c
#include <stdio.h>
#include <time.h>

int main(void) {
    char buff[70];

    // 使用指定初始化器初始化 tm 结构
    // 不需要关心成员的声明顺序
    struct tm my_time = {
        .tm_year = 2012 - 1900,
        .tm_mon  = 9,
        .tm_mday = 9,
        .tm_hour = 8,
        .tm_min  = 10,
        .tm_sec  = 20
    };

    strftime(buff, sizeof(buff), "%A %c", &my_time);
    puts(buff);

    return 0;
}
// 输出: Sunday Sun Oct  9 08:10:20 2012
```

### 联合体初始化

```c
#include <stdio.h>

int main(void) {
    // 联合体初始化：默认初始化第一个成员
    union { int x; char c[4]; } u = {1};
    // u.x 处于活跃状态，值为 1

    printf("u.x = %d\n", u.x);

    // 使用指定初始化器初始化其他成员
    union { int x; char c[4]; } u2 = { .c = {'\1'} };
    // u2.c 处于活跃状态，值为 {'\1', '\0', '\0', '\0'}

    printf("u2.c = {%d, %d, %d, %d}\n",
           u2.c[0], u2.c[1], u2.c[2], u2.c[3]);

    return 0;
}
```

### 嵌套结构初始化

```c
#include <stdio.h>
#include <stdint.h>

struct example {
    struct addr_t {
        uint32_t port;
    } addr;
    union {
        uint8_t a8[4];
        uint16_t a16[2];
    } in_u;
};

int main(void) {
    // 完整花括号的嵌套初始化
    struct example ex1 = {
        { 80 },           // 初始化 addr.port
        { 127, 0, 0, 1 }  // 初始化 in_u.a8
    };

    printf("ex1.addr.port = %u\n", ex1.addr.port);
    printf("ex1.in_u.a8 = %d.%d.%d.%d\n",
           ex1.in_u.a8[0], ex1.in_u.a8[1],
           ex1.in_u.a8[2], ex1.in_u.a8[3]);

    // 省略花括号的嵌套初始化
    struct example ex2 = {80, 127, 0, 0, 1};

    printf("ex2.addr.port = %u\n", ex2.addr.port);

    return 0;
}
```

### 指定初始化器的高级用法（C99）

```c
#include <stdio.h>

struct datetime {
    int sec, min, hour, day, mon, year;
};

int main(void) {
    // 混合使用指示符和顺序初始化
    struct datetime z = {
        .day = 31,    // 初始化 day
        12,           // 初始化下一个成员 mon
        2014,         // 初始化下一个成员 year
        .sec = 30,    // 初始化 sec
        15,           // 初始化下一个成员 min
        17            // 初始化下一个成员 hour
    };
    // 最终 z = {30, 15, 17, 31, 12, 2014}

    printf("datetime: %04d-%02d-%02d %02d:%02d:%02d\n",
           z.year, z.mon, z.day, z.hour, z.min, z.sec);

    return 0;
}
```

### 常见错误及修正

#### 错误 1：初始化器过多

```c
// 错误：初始化器数量超过成员数量
struct { int x, y; } p = {1, 2, 3};  // 编译错误

// 修正：只提供足够的初始化器
struct { int x, y; } p = {1, 2};  // 正确
```

#### 错误 2：C89 使用指定初始化器

```c
// 错误：C89 不支持指定初始化器
struct { int a, b; } s = { .b = 2, .a = 1 };  // C89: 编译错误

// 修正：升级到 C99 或使用顺序初始化
struct { int a, b; } s = { 1, 2 };  // 通用写法
// 或编译时使用 -std=c99 或更高版本
```

#### 错误 3：空初始化列表（C23 之前）

```c
// 错误：C23 之前不允许空初始化列表
struct { int n; } s = {};  // C23 之前: 编译错误

// 修正：使用 0 或显式初始化
struct { int n; } s = {0};  // 正确

// C23 可使用空初始化列表
struct { int n; } s = {};  // C23: 正确，s.n = 0
```

#### 错误 4：静态存储期使用非常量表达式

```c
// 错误：静态存储期对象需要常量表达式初始化
static struct { char *p; } s = {malloc(1)};  // 编译错误

// 修正：使用常量表达式或在函数内初始化
static struct { char *p; } s = {NULL};  // 正确
// 在程序运行时再分配
```

#### 错误 5：重复初始化同一成员

```c
#include <stdio.h>

int main(void) {
    // 注意：后出现的初始化器生效
    struct { int k; int l; int a[2]; } x = {
        .l = 43,
        .k = 42,
        .a[1] = 19,
        .a[0] = 18
    };
    // x = {42, 43, {18, 19}}

    printf("x.k=%d, x.l=%d, x.a={%d,%d}\n",
           x.k, x.l, x.a[0], x.a[1]);

    // 潜在陷阱：后续指示符可能覆盖之前的初始化
    struct { int i; struct { int k; int l; } t; } l = {
        1,
        .t = x,        // 初始化 l.t 为 {42, 43, {18, 19}}
        .t.l = 41,     // 修改 l.t.l 为 41，l.t 现在是 {42, 41, {18, 19}}
        .t.k = 0       // 这会将 l.t.k 设为 0，并可能影响其他成员！
    };

    return 0;
}
```

## 7. 总结

结构体和联合体初始化是 C 语言的基础特性，从 C89 到 C23 经历了多次增强：

### 核心要点

| 特性 | C89 | C99 | C23 |
|------|-----|-----|-----|
| 顺序初始化 | 支持 | 支持 | 支持 |
| 指定初始化器 | 不支持 | 支持 | 支持 |
| 空初始化列表 | 不支持 | 不支持 | 支持 |
| 嵌套指示符 | 不支持 | 支持 | 支持 |
| 非常量表达式（自动存储期） | 不支持 | 支持 | 支持 |

### 使用建议

1. **优先使用指定初始化器**：提高代码可读性，避免成员顺序错误
2. **使用完整花括号**：嵌套结构建议使用完整的花括号，便于理解和维护
3. **注意版本兼容性**：指定初始化器需要 C99，空初始化列表需要 C23
4. **避免重复初始化**：同一成员多次初始化会导致不可预期的结果
5. **静态对象用常量**：静态和线程局部存储期对象的初始化器必须是常量表达式

### 与 C++ 的区别

- C++ 不完全支持 C99 的指定初始化器语法（C++20 引入了受限支持）
- C++ 允许空初始化列表，C 直到 C23 才支持
- C++ 初始化顺序是确定性的，C 中初始化表达式求值顺序是不确定顺序的
