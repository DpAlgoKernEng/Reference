# C 语言发展历史

## 1. 概述

C 语言是一种通用的、过程式的编程语言，由丹尼斯·里奇（Dennis M. Ritchie）于 1972 年在贝尔实验室创建。C 语言是现代编程语言的基石之一，对后续众多编程语言（如 C++、Java、C#、JavaScript 等）产生了深远影响。

C 语言的设计目标是提供一种既能进行系统级编程（如操作系统开发），又能保持高级语言可移植性的工具。其核心特点包括：

- **低级访问能力**：提供指针操作，可直接访问内存地址
- **高效执行**：编译后的代码接近汇编语言的执行效率
- **可移植性**：源代码可在不同平台间移植
- **简洁语法**：关键字数量少，语法清晰

C 语言的标准由 ANSI（美国国家标准协会）和 ISO（国际标准化组织）共同维护，经历了从 C89/C90 到 C23 的多个版本演进。

## 2. 来源与演变

### 早期 C 语言（1969-1978）

#### 1969 年：B 语言的诞生

B 语言由肯·汤普逊（Ken Thompson）创建，基于 BCPL 语言，旨在替代 PDP-7 汇编语言成为 Unix 的系统编程语言。

**主要特点**：
- 引入了 `++`、`--` 运算符和复合赋值运算符
- 仍然是无类型语言（typeless language），类似 BCPL

#### 1971 年：NB（New B）语言

在将 B 语言移植到 PDP-11 时，发展出 NB 语言。

**重要变化**：
- 引入类型系统（int、char、数组和指针）
- 实现数组到指针的转换
- 直接编译为机器代码

#### 1972 年：语言更名为 C

**新增特性**：
- `struct` 结构体
- `&&` 和 `||` 逻辑运算符
- 预处理器（preprocessor）
- 可移植的 I/O 系统

#### 1973 年：Unix 用 C 重写

这一里程碑事件证明了 C 语言可用于系统级编程。

**新增特性**：
- `unsigned` 无符号类型
- `long` 长整型
- `union` 联合体
- 枚举类型（enumerations）
- 增强的类型安全

#### 1978 年：《The C Programming Language》第一版出版

布莱恩·柯林汉（Brian Kernighan）和丹尼斯·里奇合著的这本书成为 C 语言的事实标准，被称为 "K&R C"。

### 标准化 C 语言

#### C89/C90（1989-1990）

| 时间节点 | 事件 |
|---------|------|
| 1983 年 | ANSI 成立 X3J11 委员会 |
| 1988 年 | 《The C Programming Language》第二版出版 |
| 1989 年 | **C89**，ANSI C 标准发布 |
| 1990 年 | **C90**，ANSI C 被 ISO 接受为 ISO/IEC 9899:1990 |

**C89/C90 新特性**：
1. 整合现有实践，形成统一标准
2. 新增关键字：`volatile`、`enum`、`signed`、`void`
3. 引入本地化支持（locales）
4. 从 C++ 借鉴：`const` 限定符、函数原型（function prototypes）

#### C95（1995）

ISO/IEC 9899:1990/Amd.1:1995

**主要变化**：
1. 大幅扩展宽字符和多字节字符支持
   - 新增头文件：`<wctype.h>`、`<wchar.h>`
   - 流 I/O 相关扩展
2. 引入双字符组（digraphs）
3. 新增 `<iso646.h>` 头文件

**技术勘误**：
- 1994 年：TC1（44 处小修改）
- 1996 年：TC2（24 处小修改）

#### C99（1999）

ISO/IEC 9899:1999

**全新特性**：
| 特性 | 说明 |
|------|------|
| `bool` | 布尔类型（`<stdbool.h>`） |
| `long long` | 64 位整数类型 |
| `<stdint.h>`、`<inttypes.h>` | 固定宽度整数类型 |
| `restrict` | 指针别名优化限定符 |
| 复合字面量（Compound literals） | 匿名数组/结构体对象 |
| 变长数组（VLA） | 运行时确定数组大小 |
| 灵活数组成员 | 结构体末尾的未指定大小数组 |
| 指定初始化器（Designated initializers） | 按名称或索引初始化 |
| `<fenv.h>` | 浮点环境访问 |
| 变参宏（Variadic macros） | 支持可变参数的宏 |
| 复数类型 | `_Complex`、`_Imaginary` |
| `__func__` | 当前函数名预定义标识符 |
| 十六进制浮点格式 | `%a` 格式说明符 |

**从 C++ 借鉴的特性**：
- `inline` 关键字
- 混合声明和代码
- `for` 循环初始化子句中的声明
- `//` 单行注释
- 源码中的通用字符名（Universal character names）

**移除的特性**：
- 隐式函数声明（implicit functions）
- 隐式 int（implicit int）

**技术勘误**：
- 2001 年：TC1（11 处缺陷修复）
- 2004 年：TC2
- 2007 年：TC3（弃用 `gets` 函数）

#### C11（2011）

ISO/IEC 9899:2011

**主要新特性**：
| 特性 | 说明 |
|------|------|
| 线程感知内存模型 | 多线程支持基础 |
| `<stdatomic.h>` | 原子操作库 |
| `<threads.h>` | 线程支持库 |
| 类型泛型函数 | `_Generic` 选择表达式 |
| `alignas`/`alignof` | 对齐指定符 |
| `noreturn` | 函数不返回属性 |
| `static_assert` | 静态断言 |
| 匿名结构体和联合体 | 简化嵌套访问 |
| 独占文件打开模式 | `x` 模式标志 |
| `quick_exit` | 快速退出函数 |

**移除特性**：
- `gets` 函数（被 `gets_s` 替代）

**从技术报告（TR）引入**：
- 边界检查接口（Bounds-checking interfaces）
- `char16_t`、`char32_t` 和 `<uchar.h>`（Unicode 支持）

#### C17（2018）

ISO/IEC 9899:2018

C17 主要是一个缺陷修复版本，未引入重大新特性。

## 7. 总结

### C 语言版本演进时间线

| 版本 | 年份 | ISO 标准 | 重要性 |
|------|------|---------|--------|
| K&R C | 1978 | - | 事实标准，奠定语言基础 |
| C89/C90 | 1989/1990 | ISO/IEC 9899:1990 | 首个官方标准 |
| C95 | 1995 | ISO/IEC 9899:1990/Amd.1 | 扩展国际化支持 |
| C99 | 1999 | ISO/IEC 9899:1999 | 重大更新，现代化改进 |
| C11 | 2011 | ISO/IEC 9899:2011 | 多线程和安全性增强 |
| C17 | 2018 | ISO/IEC 9899:2018 | 缺陷修复版本 |
| C23 | 2023 | ISO/IEC 9899:2024 | 当前最新标准 |

### 关键里程碑

1. **1969-1972**：从 B 语言演变为 C 语言
2. **1973**：Unix 用 C 重写，证明系统编程能力
3. **1978**：《The C Programming Language》出版
4. **1989**：ANSI 标准化，语言走向成熟
5. **1999**：C99 引入大量现代特性
6. **2011**：C11 加入多线程支持
7. **2023**：C23 成为当前标准

### 版本选择建议

| 场景 | 推荐版本 | 原因 |
|------|---------|------|
| 嵌入式开发 | C99/C11 | 广泛的编译器支持，丰富的类型支持 |
| 系统编程 | C11 | 多线程和原子操作支持 |
| 跨平台项目 | C99 | 最佳兼容性和现代特性平衡 |
| 遗留代码维护 | C89/C90 | 兼容旧代码库 |
| 最新特性需求 | C23 | 获取最新语言改进 |
