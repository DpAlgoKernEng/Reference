# 字符常量 (Character Constant)

## 1. 概述 (Overview)

字符常量是 C 语言中表示单个字符值的字面量。它用单引号括起一个或多个字符，在编译时转换为对应的整数值。字符常量是 C 语言基础文本处理的核心构造。

**核心特点**：
- 用单引号 `'` 包围字符内容
- 编译后转换为整数值
- 支持多种字符编码（ASCII、UTF-8、UTF-16、UTF-32）
- 支持转义序列表示特殊字符

**技术定位**：
- 属于词法元素中的常量类型
- 与字符串字面量（双引号）不同，字符常量表示单个整数值
- 在表达式中作为整数值参与运算

## 2. 来源与演变 (Origin and Evolution)

### 历史背景

字符常量最早出现在 B 语言中，C 语言继承了这一设计。最初的设计目标是提供一种直观表示单个字符值的方式。

### 版本演进

| 版本 | 变更内容 |
|------|----------|
| C89/C90 | 定义基本的单字节字符常量和宽字符常量 `L'...'` |
| C99 | 新增通用字符名 `\u...` 和 `\U...` 支持 |
| C11 | 新增 `u'...'` (char16_t) 和 `U'...'` (char32_t) 类型 |
| C23 | 新增 `u8'...'` (char8_t) 类型；移除多字符的 u/U 常量 |

### 设计动机

- 提供与实现无关的字符表示方式
- 支持国际化字符集（Unicode）
- 解决不同平台字符编码差异问题

## 3. 语法与参数 (Syntax and Parameters)

### 基本语法

```c
// 单字符常量
'c-char'           // (1) 单字节字符常量，类型为 int
u8'c-char'         // (2) UTF-8 字符常量 (C23)，类型为 char8_t
u'c-char'          // (3) UTF-16 字符常量 (C11)，类型为 char16_t
U'c-char'          // (4) UTF-32 字符常量 (C11)，类型为 char32_t
L'c-char'          // (5) 宽字符常量，类型为 wchar_t

// 多字符常量（实现定义）
'c-char-sequence'  // (6) 多字符常量，类型为 int
L'c-char-sequence' // (7) 宽多字符常量，类型为 wchar_t
```

### c-char 定义

c-char 可以是以下之一：

1. **基本源字符集字符**：除单引号 `'`、反斜杠 `\` 和换行符外的任意字符
2. **转义序列**：
   - 特殊字符转义：`\'` `\"` `\?` `\\` `\a` `\b` `\f` `\n` `\r` `\t` `\v`
   - 十六进制转义：`\x...`
   - 八进制转义：`\...`
3. **通用字符名** (C99+)：`\u....` 或 `\U........`

### 类型与值

| 前缀 | 类型 | 说明 | 版本 |
|------|------|------|------|
| 无 | `int` | 执行字符集中的单字节值 | C89+ |
| `u8` | `char8_t` | UTF-8 单码单元值 (0x00-0x7F) | C23 |
| `u` | `char16_t` | UTF-16 单码单元值 (0x0-0xD7FF 或 0xE000-0xFFFF) | C11+ |
| `U` | `char32_t` | UTF-32 单码单元值 (0x0-0x10FFFF) | C11+ |
| `L` | `wchar_t` | 执行宽字符集中的值 | C89+ |

### 转义序列参考

| 转义序列 | 含义 | ASCII 值 |
|----------|------|----------|
| `\'` | 单引号 | 0x27 |
| `\"` | 双引号 | 0x22 |
| `\?` | 问号 | 0x3F |
| `\\` | 反斜杠 | 0x5C |
| `\a` | 响铃 | 0x07 |
| `\b` | 退格 | 0x08 |
| `\f` | 换页 | 0x0C |
| `\n` | 换行 | 0x0A |
| `\r` | 回车 | 0x0D |
| `\t` | 水平制表 | 0x09 |
| `\v` | 垂直制表 | 0x0B |
| `\xHH` | 十六进制值 | HH |
| `\OOO` | 八进制值 | OOO |

## 4. 底层原理 (Underlying Principles)

### 编译期转换机制

字符常量在编译时被转换为整数值：

```
源代码 'A' → 编译器 → 整数值 65 (ASCII)
```

### 字符集映射

1. **普通字符常量**：从源字符集映射到执行字符集
2. **宽字符常量**：通过 `mbtowc` 机制转换
3. **UTF 字符常量**：直接使用 ISO 10646 码点值

### 多字符常量的存储

多字符常量（如 `'AB'`）按大端序、零填充、右对齐方式存储：

```
'\1'        → 0x00000001
'\1\2\3\4'  → 0x01020304
'AB'        → 0x00004142  // 'A'=0x41, 'B'=0x42
```

**注意**：MSVC 不遵循此约定，多字符常量行为可能不同。

### 符号扩展

当 `char` 为有符号类型时，字符常量可能产生负值：

```c
// 若 char 为有符号类型
'\xFF'  // 值为 -1 (int 类型)
```

## 5. 使用场景 (Use Cases)

### 适用场景

| 场景 | 推荐用法 | 示例 |
|------|----------|------|
| ASCII 字符处理 | 普通字符常量 | `'a'`, `'\n'` |
| Unicode 字符 | UTF-32 常量 | `U'猫'`, `U'中'` |
| 宽字符环境 | 宽字符常量 | `L'字'` |
| 四字符标记 | 多字符常量 | `'PNG'` |

### 最佳实践

1. **优先使用单字符常量**：多字符常量的行为是实现定义的
2. **Unicode 字符使用 U 前缀**：保证跨平台一致性
3. **避免依赖实现定义行为**：如非 BMP 字符在 16 位 wchar_t 环境下的处理

### 常见陷阱

| 陷阱 | 问题 | 解决方案 |
|------|------|----------|
| `u'🍌'` 在 C11 中 | Emoji 需要 UTF-16 代理对 | 使用 `U'🍌'` 或 `u'\U0001f34c'` |
| `'AB'` 的值 | 实现定义，不可移植 | 避免使用多字符常量 |
| `'\xFF'` 的符号 | 取决于 char 是否有符号 | 使用显式类型转换 |
| `#if` 中的字符常量 | 字符集由实现定义 | 避免在预处理指令中使用 |

## 6. 代码示例 (Examples)

### 基础用法

```c
#include <stdio.h>

int main(void) {
    // 普通字符常量
    int newline = '\n';
    int tab = '\t';
    int quote = '\'';

    printf("换行符值: %d\n", newline);   // 输出: 10
    printf("制表符值: %d\n", tab);       // 输出: 9
    printf("单引号值: %d\n", quote);     // 输出: 39

    return 0;
}
```

### Unicode 字符常量

```c
#include <stdio.h>
#include <uchar.h>

int main(void) {
    // UTF-8 字符常量 (C23)
    // char8_t c8 = u8'A';

    // UTF-16 字符常量
    char16_t c16 = u'猫';
    printf("UTF-16 '猫': 0x%04x\n", c16);  // 输出: 0x732b

    // UTF-32 字符常量 - 支持 Emoji
    char32_t c32 = U'🍌';
    printf("UTF-32 '🍌': 0x%06x\n", c32);  // 输出: 0x01f34c

    return 0;
}
```

### 宽字符常量

```c
#include <stdio.h>
#include <wchar.h>
#include <locale.h>

int main(void) {
    setlocale(LC_ALL, "");

    wchar_t wc = L'中';
    wprintf(L"宽字符: %lc\n", wc);

    return 0;
}
```

### 转义序列示例

```c
#include <stdio.h>

int main(void) {
    printf("响铃字符: %d\n", '\a');        // 7
    printf("十六进制 \\x41: %c\n", '\x41'); // A
    printf("八进制 \\101: %c\n", '\101');   // A
    printf("通用字符 \\u4e2d: %lc\n", L'\u4e2d'); // 中

    return 0;
}
```

### 常见错误示例

```c
#include <stdio.h>

int main(void) {
    // 错误：多字符常量值是实现定义的
    int mc = 'AB';
    printf("'AB' = 0x%08x (实现定义)\n", mc);

    // 错误：C11 中 u'🍌' 需要代理对，值是实现定义的
    // char16_t emoji = u'🍌';  // 不推荐

    // 正确：使用 UTF-32 或通用字符名
    char32_t emoji = U'\U0001f34c';  // 正确

    // 错误：在 #if 中使用字符常量
    // #if 'A' == 65  // 字符集可能不同
    // ...

    return 0;
}
```

### 完整示例：字符常量类型对比

```c
#include <stdio.h>
#include <stddef.h>
#include <uchar.h>

int main(void) {
    printf("字符常量类型对比\n");
    printf("================\n\n");

    // 普通字符常量
    int c = 'A';
    printf("'A'      类型: int, 值: %d (0x%02x)\n", c, c);

    // 宽字符常量
    wchar_t wc = L'A';
    printf("L'A'     类型: wchar_t, 大小: %zu 字节\n", sizeof(wc));

    // UTF-16 字符常量
    char16_t c16 = u'A';
    printf("u'A'     类型: char16_t, 值: 0x%04x\n", c16);

    // UTF-32 字符常量
    char32_t c32 = U'A';
    printf("U'A'     类型: char32_t, 值: 0x%06x\n", c32);

    // 中文字符示例
    printf("\n中文字符示例:\n");
    printf("U'猫'    = 0x%06x\n", U'猫');
    printf("U'🍌'    = 0x%06x\n", U'🍌');

    return 0;
}
```

## 7. 总结 (Summary)

### 核心要点

| 特性 | 说明 |
|------|------|
| 基本语法 | `'字符'` 或 `前缀'字符'` |
| 默认类型 | `int`（非 C++ 的 `char`） |
| UTF-8 前缀 | `u8'...'` (C23)，类型 `char8_t` |
| UTF-16 前缀 | `u'...'` (C11)，类型 `char16_t` |
| UTF-32 前缀 | `U'...'` (C11)，类型 `char32_t` |
| 宽字符前缀 | `L'...'`，类型 `wchar_t` |

### 类型对比

| 前缀 | 类型 | 大小 | 适用场景 |
|------|------|------|----------|
| 无 | `int` | 通常 4 字节 | ASCII 字符 |
| `u8` | `char8_t` | 1 字节 | ASCII/UTF-8 字符 (C23) |
| `u` | `char16_t` | 2 字节 | BMP Unicode 字符 |
| `U` | `char32_t` | 4 字节 | 所有 Unicode 字符 |
| `L` | `wchar_t` | 平台相关 | 平台本地化字符 |

### 版本兼容性提示

- **C23**：新增 `u8'...'`，移除多字符的 `u'...'` 和 `U'...'`
- **C11**：新增 `u'...'` 和 `U'...'`
- **C99**：新增 `\u....` 和 `\U........` 通用字符名

### 学习建议

1. 理解字符常量与字符串字面量的区别
2. 掌握不同前缀对应的编码类型
3. 避免使用多字符常量以确保可移植性
4. Unicode 字符优先使用 `U'...'` 前缀
