# 算术运算符 - Arithmetic Operators

## 1. 概述

算术运算符（arithmetic operators）是 C 语言中最基础的运算符类型，用于对操作数执行标准数学运算。C 语言提供了丰富的一元和二元算术运算符，包括基本算术运算、位运算和移位运算。

算术运算符定义在 C 语言标准中，无需包含特定头文件（除复杂类型运算外）。

### 运算符分类

| 运算符 | 名称 | 示例 | 结果 |
|--------|------|------|------|
| + | 一元加（unary plus） | +a | 提升后的 a 的值 |
| - | 一元减（unary minus） | -a | a 的负值 |
| + | 加法（addition） | a + b | a 和 b 的和 |
| - | 减法（subtraction） | a - b | a 减去 b 的差 |
| * | 乘法（multiplication） | a * b | a 和 b 的积 |
| / | 除法（division） | a / b | a 除以 b 的商 |
| % | 取余（remainder） | a % b | a 除以 b 的余数 |
| ~ | 位非（bitwise NOT） | ~a | a 的按位取反 |
| & | 位与（bitwise AND） | a & b | a 和 b 的按位与 |
| \| | 位或（bitwise OR） | a \| b | a 和 b 的按位或 |
| ^ | 位异或（bitwise XOR） | a ^ b | a 和 b 的按位异或 |
| << | 左移（bitwise left shift） | a << b | a 左移 b 位 |
| >> | 右移（bitwise right shift） | a >> b | a 右移 b 位 |

## 2. 来源与演变

### 历史背景

算术运算符是 C 语言从 B 语言和 BCPL 语言继承而来的核心特性。这些运算符的设计源于早期计算机科学对数学运算的直接映射需求。

### C 标准演变

#### C89/C90 标准
- 确立了基本算术运算符的语义
- 规定了整数除法的截断行为（由实现定义方向）
- 定义了位运算符的基本语义

#### C99 标准
- 明确规定整数除法向零截断（truncate towards zero）
- 增加了对复数类型（complex types）的支持
- 增加了对虚数类型（imaginary types）的支持
- 引入了浮点环境控制（`#pragma STDC FENV_ACCESS`）
- 引入了浮点收缩控制（`#pragma STDC FP_CONTRACT`）
- 引入了复数范围限制控制（`#pragma STDC CX_LIMITED_RANGE`）

#### C11 标准
- 增加了对 `_Alignof` 运算符的支持
- 改进了类型提升规则

#### C17 标准
- 澄清了部分未定义行为
- 改进了对边界情况的处理规范

## 3. 语法与参数

### 一元算术运算符

```
+ 表达式    // 一元加（提升）
- 表达式    // 一元减（取负）
```

**参数说明：**
- `表达式`：任意算术类型的表达式

**类型转换：**
- 首先对操作数进行整数提升（integer promotion）
- 一元加返回提升后的值
- 一元减返回提升后值的负值（NaN 的负值仍是 NaN）
- 结果类型为提升后的类型，值类别为非左值（non-lvalue）

### 加法运算符

```
左值 + 右值    // 加法
左值 - 右值    // 减法
```

**参数要求：**

加法（+）操作数必须满足以下条件之一：
- 两个操作数都是算术类型（包括复数和虚数）
- 一个是指向完整对象类型的指针，另一个是整数类型

减法（-）操作数必须满足以下条件之一：
- 两个操作数都是算术类型（包括复数和虚数）
- 左值是指向完整对象类型的指针，右值是整数类型
- 两个操作数都是指向兼容类型的完整对象的指针（忽略限定符）

### 乘法运算符

```
左值 * 右值    // 乘法
左值 / 右值    // 除法
左值 % 右值    // 取余
```

**参数要求：**
- 乘法和除法：操作数必须是算术类型
- 取余：操作数必须是整数类型

**类型转换：**
- 首先执行常规算术转换（usual arithmetic conversions）

### 位运算符

```
~ 右值            // 位非
左值 & 右值        // 位与
左值 | 右值        // 位或
左值 ^ 右值        // 位异或
```

**参数要求：**
- 操作数必须是整数类型

**类型转换：**
- `&`、`^`、`|` 运算符对两个操作数执行常规算术转换
- `~` 运算符对其操作数执行整数提升

### 移位运算符

```
左值 << 右值    // 左移
左值 >> 右值    // 右移
```

**参数要求：**
- 两个操作数都必须是整数类型

**类型转换：**
- 对每个操作数单独执行整数提升（注意：这与其他二元算术运算符不同）
- 结果类型为左值提升后的类型

**未定义行为：**
- 右值为负数
- 右值大于或等于提升后左值的位数

## 4. 底层原理

### 整数溢出

#### 无符号整数

无符号整数运算总是执行模 2^n 运算，其中 n 是该整数的位数。例如：
- 对于 `unsigned int`，对 `UINT_MAX` 加 1 得到 0
- 从 0 减 1 得到 `UINT_MAX`

#### 有符号整数

当有符号整数运算发生溢出（结果不适合结果类型）时，行为是**未定义的**（undefined behavior）：
- 可能在典型平台（二进制补码）上回绕
- 可能因平台或编译器选项（如 GCC 和 Clang 的 `-ftrapv`）而触发陷阱
- 可能被编译器完全优化掉

### 浮点数环境

如果设置了 `#pragma STDC FENV_ACCESS` 为 `ON`，所有浮点算术运算符遵守当前浮点舍入方向，并按照 `math_errhandling` 的规定报告浮点算术错误，除非是静态初始化器的一部分（此时不引发浮点异常，舍入模式为最近舍入）。

### 浮点收缩

除非 `#pragma STDC FP_CONTRACT` 设置为 `OFF`，所有浮点运算可以执行，就好像中间结果具有无限范围和精度。这意味着可以优化掉原本会观察到的舍入错误和浮点异常。

例如：
- 允许用单条融合乘加（fused multiply-add）CPU 指令实现 `(x*y) + z`
- 允许将 `a = x*x*x*x;` 优化为 `tmp = x*x; a = tmp*tmp;`

与收缩无关，浮点运算的中间结果可能具有与其类型指示的范围和精度不同的范围和精度，参见 `FLT_EVAL_METHOD`。

### 复数运算规则

#### 复数加减

复数和虚数的加减定义如下：

| + 或 - | u | iv | u + iv |
|--------|---|-----|---------|
| x | x ± u | x ± iv | (x ± u) ± iv |
| iy | ±u + iy | i(y ± v) | ±u + i(y ± v) |
| x + iy | (x ± u) + iy | x + i(y ± v) | (x ± u) + i(y ± v) |

#### 复数乘法

| * | u | iv | u + iv |
|---|---|-----|---------|
| x | xu | i(xv) | (xu) + i(xv) |
| iy | i(yu) | −yv | (−yv) + i(yu) |
| x + iy | (xu) + i(yu) | (−yv) + i(xv) | **特殊规则** |

在 C 语言中，任何至少有一个无穷部分的复数值都是无穷大，即使其另一部分是 NaN。因此，通常的算术规则不适用于复数-复数乘法。

除了无穷大处理外，复数乘法不允许中间结果溢出，除非 `#pragma STDC CX_LIMITED_RANGE` 设置为 `ON`，此时可以按照 `(x+iy)×(u+iv) = (xu-yv)+i(yu+xv)` 计算值，程序员负责限制操作数的范围并处理无穷大。

#### 复数除法

| / | u | iv |
|---|---|-----|
| x | x/u | i(−x/v) |
| iy | i(y/u) | y/v |
| x + iy | (x/u) + i(y/u) | (y/v) + i(−x/v) |

类似乘法，复数除法也有特殊的无穷大处理规则和 `CX_LIMITED_RANGE` 优化。

### 指针算术

#### 指针与整数加减

如果指针 `P` 指向索引为 `I` 的数组元素：
- `P + N` 和 `N + P` 指向同一数组的索引 `I + N` 的元素
- `P - N` 指向同一数组的索引 `I - N` 的元素

**行为定义条件：**
- 原始指针和结果指针都必须指向同一数组的元素或该数组末尾的下一个位置
- 执行 `p - 1`（当 p 指向数组第一个元素时）是未定义行为

#### 指针相减

如果指针 `P1` 指向索引为 `I`（或末尾下一位置）的数组元素，`P2` 指向同一数组的索引为 `J`（或末尾下一位置）的元素：
- `P1 - P2` 的值等于 `I - J`，类型为 `ptrdiff_t`（有符号整数类型，通常是可以声明的最大对象大小的一半）

**行为定义条件：**
- 结果必须适合 `ptrdiff_t`

**特殊规则：**
对于指针算术的目的，不作为任何数组元素的指针被视为指向大小为 1 的数组的第一个元素。

### 除法和取余规则

#### 整数除法

- C99 前：结果向实现定义的方向截断
- C99 起：结果向零截断（truncate towards zero）

#### 取余运算

取余运算符 `%` 产生第一个操作数除以第二个操作数的余数。

余数的符号定义确保：如果商 `a/b` 在结果类型中可表示，则 `(a/b)*b + a%b == a`。

**未定义行为：**
- 第二个操作数为零
- 商 `a/b` 在结果类型中不可表示（例如 `INT_MIN % -1` 在二进制补码系统上是未定义的）

**注意：** 取余运算符不适用于浮点类型，库函数 `fmod` 提供该功能。

### 移位运算规则

#### 左移

对于无符号 `lhs`：
- `LHS << RHS` 的值为 `LHS * 2^RHS`，对返回类型的最大值加 1 取模
- 移出的位被丢弃

对于有符号 `lhs` 且值为非负：
- 如果 `LHS * 2^RHS` 在提升后的 `lhs` 类型中可表示，则值为该结果
- 否则行为未定义

#### 右移

对于无符号 `lhs` 和值为非负的有符号 `lhs`：
- `LHS >> RHS` 的值为 `LHS / 2^RHS` 的整数部分

对于负的 `lhs`：
- 行为由实现定义
- 大多数实现执行算术右移（结果保持负值）
- 大多数实现对有符号 `lhs` 右移会用原始符号位填充新的高位（非负填充 0，负数填充 1）

## 5. 使用场景

### 适合使用算术运算符的场景

| 运算符 | 适用场景 | 优势 |
|--------|----------|------|
| 基本算术运算符（+, -, *, /） | 数值计算 | 直观、高效 |
| 取余运算符（%） | 循环索引、奇偶判断、取模运算 | O(1) 时间复杂度 |
| 位运算符（&, \|, ^, ~） | 位掩码操作、标志位管理、权限控制 | 直接操作位，性能极高 |
| 移位运算符（<<, >>） | 快速乘除 2 的幂次、位字段操作 | 比乘除法更快 |

### 最佳实践

#### 1. 避免有符号整数溢出

```c
// ❌ 错误：可能导致溢出
int a = INT_MAX;
int b = a + 1;  // 未定义行为

// ✅ 正确：使用无符号类型或检查
int a = INT_MAX;
if (a < INT_MAX - 1) {
    int b = a + 1;
}
```

#### 2. 谨慎使用一元减

```c
// ❌ 错误：INT_MIN 的负值溢出
int a = INT_MIN;
int b = -a;  // 未定义行为

// ✅ 正确：使用更大类型或检查
int a = INT_MIN;
long long b = -(long long)a;  // 使用更大类型
```

#### 3. 使用无符号类型进行模运算

```c
// ✅ 推荐：无符号运算总是定义良好的
unsigned int a = 0;
unsigned int b = a - 1;  // 结果为 UINT_MAX，行为定义良好
```

#### 4. 指针算术注意事项

```c
// ❌ 错误：指针越界
int arr[5];
int *p = &arr[0];
int *q = p - 1;  // 未定义行为

// ✅ 正确：保持在有效范围内
int arr[5];
int *p = &arr[2];
int *q = p - 1;  // 指向 arr[1]，有效
```

### 常见陷阱

#### 1. 整数除法截断

```c
// 注意：整数除法向零截断
int a = 7 / 2;    // 结果为 3，不是 3.5
int b = -7 / 2;   // C99 后结果为 -3（向零截断）
int c = 7 / -2;   // C99 后结果为 -3（向零截断）
```

#### 2. 取余符号

```c
// 余数符号与被除数相同
int a = 7 % 3;    // 结果为 1
int b = -7 % 3;   // 结果为 -1
int c = 7 % -3;   // 结果为 1
```

#### 3. 移位越界

```c
// ❌ 错误：移位数量超出范围
int x = 1;
int y = x << 32;  // 32 位 int 上未定义行为

// ✅ 正确：确保移位量有效
int x = 1;
int shift = 31;
if (shift >= 0 && shift < sizeof(x) * 8) {
    int y = x << shift;
}
```

#### 4. 位运算符优先级

```c
// 注意：位运算符优先级低于比较运算符
// ❌ 错误
if (a & b == 0)  // 解析为 a & (b == 0)

// ✅ 正确
if ((a & b) == 0)  // 明确意图
```

### 性能考虑

| 运算 | 相对性能 | 说明 |
|------|---------|------|
| 位运算（&, \|, ^, ~） | 最快 | 单周期操作 |
| 移位运算（<<, >>） | 最快 | 单周期操作 |
| 加减法（+, -） | 很快 | 通常 1-2 周期 |
| 乘法（*） | 中等 | 通常 3-5 周期 |
| 除法（/） | 较慢 | 通常 10-30 周期 |
| 取余（%） | 较慢 | 通常与除法相当 |

**优化建议：**
- 用移位替代 2 的幂次乘除
- 用位掩码替代除法取余（如 `x & 7` 代替 `x % 8`）
- 编译器通常会自动优化这些情况

## 6. 代码示例

### 基础用法：一元运算符

```c
#include <stdio.h>
#include <limits.h>

int main(void) {
    // 一元加：整数提升
    char c = 'a';
    printf("sizeof char: %zu, sizeof int: %zu\n",
           sizeof c, sizeof +c);  // +c 提升为 int

    // 一元减：取负
    int a = 5;
    printf("-%d = %d\n", a, -a);

    // 无符号数取负：模运算
    unsigned int u = 1;
    printf("-1u = %u\n", -u);  // 结果为 UINT_MAX

    return 0;
}
```

**输出：**
```
sizeof char: 1, sizeof int: 4
-5 = -5
-1u = 4294967295
```

### 加减法运算

```c
#include <stdio.h>
#include <complex.h>

int main(void) {
    // 整数加减
    int a = 10, b = 3;
    printf("%d + %d = %d\n", a, b, a + b);
    printf("%d - %d = %d\n", a, b, a - b);

    // 浮点加减
    double x = 3.14, y = 2.71;
    printf("%.2f + %.2f = %.2f\n", x, y, x + y);

    // 复数加减
    double complex z1 = 1.0 + 2.0 * I;
    double complex z2 = 3.0 + 4.0 * I;
    double complex sum = z1 + z2;
    printf("(1+2i) + (3+4i) = %.1f%+.1fi\n",
           creal(sum), cimag(sum));

    return 0;
}
```

**输出：**
```
10 + 3 = 13
10 - 3 = 7
3.14 + 2.71 = 5.85
(1+2i) + (3+4i) = 4.0+6.0i
```

### 指针算术

```c
#include <stdio.h>

int main(void) {
    int arr[] = {10, 20, 30, 40, 50};
    int *p = arr;

    // 指针加法
    printf("*(p + 2) = %d\n", *(p + 2));  // 30

    // 指针减法
    printf("*(p + 4 - 1) = %d\n", *(p + 4 - 1));  // 40

    // 指针相减
    int *end = arr + 5;
    printf("end - p = %ld\n", (long)(end - p));  // 5

    // 遍历数组
    for (int *q = arr; q < end; q++) {
        printf("%d ", *q);
    }
    printf("\n");

    return 0;
}
```

**输出：**
```
*(p + 2) = 30
*(p + 4 - 1) = 40
end - p = 5
10 20 30 40 50
```

### 乘除法运算

```c
#include <stdio.h>
#include <math.h>
#include <complex.h>

int main(void) {
    // 整数乘除
    int a = 17, b = 5;
    printf("%d * %d = %d\n", a, b, a * b);
    printf("%d / %d = %d\n", a, b, a / b);  // 向零截断
    printf("%d %% %d = %d\n", a, b, a % b);

    // 负数除法
    printf("-17 / 5 = %d\n", -17 / 5);   // -3 (向零截断)
    printf("17 / -5 = %d\n", 17 / -5);   // -3 (向零截断)
    printf("-17 %% 5 = %d\n", -17 % 5);  // -2 (符号与被除数相同)

    // 浮点除法
    double x = 17.0, y = 5.0;
    printf("%.1f / %.1f = %.2f\n", x, y, x / y);

    // 复数乘法（特殊规则）
    double complex z = (1 + 0*I) * (INFINITY + I*INFINITY);
    printf("(1+0i)*(inf+inf*i) = %f + i*%f\n", creal(z), cimag(z));

    return 0;
}
```

**输出：**
```
17 * 5 = 85
17 / 5 = 3
17 % 5 = 2
-17 / 5 = -3
17 / -5 = -3
-17 % 5 = -2
17.0 / 5.0 = 3.40
(1+0i)*(inf+inf*i) = inf + i*inf
```

### 位运算

```c
#include <stdio.h>
#include <stdint.h>

int main(void) {
    uint32_t a = 0x12345678;
    uint16_t mask = 0x00f0;

    printf("原始值:        %#010x\n", a);
    printf("掩码:          %#06x\n", mask);
    printf("提升后掩码:    %#010x\n", (uint32_t)mask);
    printf("设置位 (|):    %#010x\n", a | mask);
    printf("清除位 (& ~):  %#010x\n", a & ~mask);
    printf("选择位 (&):    %#010x\n", a & mask);
    printf("异或 (^):      %#010x\n", a ^ mask);
    printf("位非 (~):      %#010x\n", ~a);

    // 位运算应用
    uint8_t flags = 0x00;

    // 设置位
    flags |= 0x01;  // 设置第 0 位
    flags |= 0x04;  // 设置第 2 位
    printf("设置位后:      %#04x\n", flags);

    // 检查位
    if (flags & 0x01) {
        printf("第 0 位已设置\n");
    }

    // 清除位
    flags &= ~0x04;  // 清除第 2 位
    printf("清除位后:      %#04x\n", flags);

    // 切换位
    flags ^= 0x01;  // 切换第 0 位
    printf("切换位后:      %#04x\n", flags);

    return 0;
}
```

**输出：**
```
原始值:        0x12345678
掩码:          0x00f0
提升后掩码:    0x000000f0
设置位 (|):    0x123456f8
清除位 (& ~):  0x12345608
选择位 (&):    0x00000070
异或 (^):      0x12345688
位非 (~):      0xedcba987
设置位后:      0x05
第 0 位已设置
清除位后:      0x01
切换位后:      0x00
```

### 移位运算

```c
#include <stdio.h>

int main(void) {
    unsigned int u = 0x123;
    printf("0x%03x << 1  = 0x%03x\n", u, u << 1);
    printf("0x%03x << 4  = 0x%03x\n", u, u << 4);
    printf("0x%03x >> 1  = 0x%03x\n", u, u >> 1);

    // 左移相当于乘以 2 的幂
    int n = 5;
    printf("%d << 3 = %d (等于 %d * 8)\n", n, n << 3, n * 8);

    // 右移相当于除以 2 的幂（对于无符号和非负有符号数）
    printf("%d >> 2 = %d (等于 %d / 4)\n", n, n >> 2, n / 4);

    // 有符号负数右移（实现定义，通常为算术右移）
    int neg = -1000;
    printf("%d >> 1 = %d\n", neg, neg >> 1);  // 通常为 -500

    // 字符提升
    char c = 0x10;
    printf("0x%02x << 10 = 0x%x (char 提升为 int)\n",
           (unsigned char)c, c << 10);

    return 0;
}
```

**输出：**
```
0x123 << 1  = 0x246
0x123 << 4  = 0x1230
0x123 >> 1  = 0x091
5 << 3 = 40 (等于 5 * 8)
5 >> 2 = 1 (等于 5 / 4)
-1000 >> 1 = -500
0x10 << 10 = 0x4000 (char 提升为 int)
```

### 常见错误及修正

#### 错误 1：有符号整数溢出

```c
#include <stdio.h>
#include <limits.h>

int main() {
    // ❌ 错误：有符号整数溢出是未定义行为
    // int x = INT_MAX;
    // int y = x + 1;  // 未定义行为

    // ✅ 修正：使用无符号类型
    unsigned int x = UINT_MAX;
    unsigned int y = x + 1;  // 定义良好的回绕到 0
    printf("UINT_MAX + 1 = %u\n", y);

    // ✅ 或使用更大类型
    long long lx = INT_MAX;
    long long ly = lx + 1;  // 在 long long 范围内
    printf("INT_MAX + 1 = %lld\n", ly);

    return 0;
}
```

#### 错误 2：INT_MIN 的负值

```c
#include <stdio.h>
#include <limits.h>

int main() {
    // ❌ 错误：-INT_MIN 是未定义行为
    // int x = INT_MIN;
    // int y = -x;  // 未定义行为

    // ✅ 修正：使用更大类型
    int x = INT_MIN;
    long long y = -(long long)x;
    printf("负的 INT_MIN = %lld\n", y);

    // ✅ 或使用无符号类型（注意解释）
    unsigned int uy = -(unsigned int)x;
    printf("负的 INT_MIN (无符号) = %u\n", uy);

    return 0;
}
```

#### 错误 3：除以零

```c
#include <stdio.h>

int main() {
    int a = 10, b = 0;

    // ❌ 错误：除以零是未定义行为
    // int c = a / b;  // 未定义行为

    // ✅ 修正：检查除数
    if (b != 0) {
        int c = a / b;
        printf("%d / %d = %d\n", a, b, c);
    } else {
        printf("错误：除数不能为零\n");
    }

    return 0;
}
```

#### 错误 4：移位量越界

```c
#include <stdio.h>
#include <limits.h>

int main() {
    int x = 1;

    // ❌ 错误：移位量超出类型位数
    // int y = x << 32;  // 32 位 int 上未定义行为
    // int z = x << -1;  // 负数移位量，未定义行为

    // ✅ 修正：检查移位量
    int shift = 31;
    if (shift >= 0 && shift < (int)(sizeof(int) * CHAR_BIT)) {
        int y = x << shift;
        printf("1 << %d = %d\n", shift, y);
    }

    return 0;
}
```

#### 错误 5：指针越界

```c
#include <stdio.h>

int main() {
    int arr[5] = {1, 2, 3, 4, 5};
    int *p = arr;

    // ❌ 错误：指针越界
    // int *q = p - 1;  // 指向 arr[-1]，未定义行为
    // int *r = p + 10; // 远超数组末尾

    // ✅ 修正：保持在有效范围
    int *start = arr;
    int *end = arr + 5;  // 指向末尾下一位置

    for (int *q = start; q < end; q++) {
        printf("%d ", *q);
    }
    printf("\n");

    // 指向末尾下一位置是允许的（但不能解引用）
    int *after_end = arr + 5;
    if (after_end == end) {
        printf("可以指向末尾下一位置\n");
    }

    return 0;
}
```

## 7. 总结

C 语言算术运算符提供了丰富的数值和位操作能力，是程序设计的基础工具。

### 核心要点

| 运算符类型 | 关键点 | 注意事项 |
|-----------|--------|----------|
| 一元运算符（+, -） | 执行整数提升 | INT_MIN 取负是 UB |
| 加减运算符 | 支持算术和指针 | 指针必须保持有效范围 |
| 乘除运算符 | 整数除法向零截断 | 除以零是 UB |
| 取余运算符 | 余数符号与被除数相同 | 不支持浮点类型 |
| 位运算符 | 按位操作 | 操作数必须是整数类型 |
| 移位运算符 | 移位量必须有效 | 负数右移是实现定义的 |

### 类型提升和转换

1. **整数提升**：`char`、`short` 等小整数类型自动提升为 `int` 或 `unsigned int`
2. **常规算术转换**：二元运算符通常将操作数转换为公共类型
3. **移位例外**：移位运算符对每个操作数单独进行整数提升

### 未定义行为汇总

| 情况 | 说明 |
|------|------|
| 有符号整数溢出 | 结果不适合结果类型 |
| INT_MIN 取负 | 数学结果超出范围 |
| 除以零 | 整数除法和取余 |
| 移位量无效 | 负数或超出位数 |
| 指针算术越界 | 超出数组范围（首元素前或末尾下一位置后） |
| INT_MIN % -1 | 商不可表示 |

### 实现定义行为

| 情况 | 说明 |
|------|------|
| 负数右移 | 通常为算术右移（保留符号位） |
| 整数除法截断方向（C99 前） | C99 后统一为向零截断 |

### 性能对比

- **最快**：位运算、移位运算
- **很快**：加减法
- **中等**：乘法
- **较慢**：除法、取余

### 最佳实践建议

1. **优先使用无符号类型进行模运算**
2. **避免有符号整数溢出**
3. **使用位掩码代替复杂算术**
4. **检查除数和移位量**
5. **指针算术保持在有效范围内**
6. **使用括号明确位运算优先级**
