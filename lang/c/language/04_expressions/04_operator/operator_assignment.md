# 赋值运算符 (Assignment Operators)

## 1. 概述 (Overview)

赋值运算符 (Assignment Operators) 是 C 语言中的二元运算符，用于修改左侧变量的值。赋值运算符分为两类：

- **简单赋值运算符**：`=`
- **复合赋值运算符**：`+=`, `-=`, `*=`, `/=`, `%=`, `&=`, `|=`, `^=`, `<<=`, `>>=`

简单赋值运算符将右侧的值赋给左侧的变量，而复合赋值运算符则在赋值前先执行相应的算术或位运算。赋值运算符的核心作用是将值存储到左值 (lvalue) 指定的内存位置，并返回该值以便支持链式赋值。

## 2. 来源与演变 (Origin and Evolution)

### 历史背景

赋值运算符从 C 语言的早期版本就存在，其设计动机源于以下需求：

1. **值存储**：提供一种直接修改变量值的方式
2. **代码简洁性**：复合赋值运算符避免了重复书写变量名
3. **效率优化**：编译器可以对复合赋值进行优化

### 版本演变

| 版本 | 变更内容 |
|------|----------|
| C89/C90 | 定义了基本赋值运算符的语义和类型要求 |
| C99 | 增加了对 `_Bool` 类型的支持；允许 `restrict` 限定符用于指针类型的左值 |
| C11 | 引入了对原子类型 (atomic) 的支持，复合赋值操作可以表现为原子的读-修改-写操作 |
| C23 | 增加了 `nullptr_t` 类型的支持 |

### 跨语言对比

在 C++ 中，赋值运算符返回左值 (lvalue)，而在 C 中返回非左值 (non-lvalue)。这意味着在 C++ 中 `(a=b)=c` 是合法的，而在 C 中是非法的。

## 3. 语法与参数 (Syntax and Parameters)

### 3.1 简单赋值运算符

**语法形式**：
```
lhs = rhs
```

**参数说明**：

| 参数 | 说明 |
|------|------|
| `lhs` | 可修改的左值表达式 (modifiable lvalue)，可以是任何完整对象类型 |
| `rhs` | 可以隐式转换为 `lhs` 类型的表达式，或与 `lhs` 类型兼容的表达式 |

**类型要求**：

`lhs` 和 `rhs` 必须满足以下条件之一：

1. **结构体/联合体类型**：两者具有兼容的结构体或联合体类型
2. **算术类型**：两者都是算术类型（`lhs` 可以是 `volatile` 限定或原子类型）
3. **指针类型**：
   - 两者都是指向兼容类型（忽略限定符）的指针
   - 其中一个是指向 `void` 的指针，且转换不会增加被指向类型的限定符
   - `lhs` 可以是 `volatile` 或 `restrict`（C99 起）限定或原子类型
4. **指针与空指针**：
   - `lhs` 是（可能限定的或原子的）指针，`rhs` 是空指针常量（如 `NULL`）或 `nullptr_t` 值（C23 起）
5. **布尔类型与指针**（C99 起）：
   - `lhs` 具有 `_Bool` 类型（可能限定或原子），`rhs` 是指针或 `nullptr_t` 值（C23 起）
6. **nullptr_t 类型**（C23 起）：
   - 两者都是 `nullptr_t` 类型

### 3.2 复合赋值运算符

**语法形式**：
```
lhs op= rhs
```

**支持的运算符**：

| 运算符 | 名称 | 等价形式 |
|--------|------|----------|
| `+=` | 加法赋值 | `lhs = lhs + rhs` |
| `-=` | 减法赋值 | `lhs = lhs - rhs` |
| `*=` | 乘法赋值 | `lhs = lhs * rhs` |
| `/=` | 除法赋值 | `lhs = lhs / rhs` |
| `%=` | 取模赋值 | `lhs = lhs % rhs` |
| `&=` | 按位与赋值 | `lhs = lhs & rhs` |
| `\|=` | 按位或赋值 | `lhs = lhs \| rhs` |
| `^=` | 按位异或赋值 | `lhs = lhs ^ rhs` |
| `<<=` | 左移赋值 | `lhs = lhs << rhs` |
| `>>=` | 右移赋值 | `lhs = lhs >> rhs` |

**参数说明**：

| 参数 | 说明 |
|------|------|
| `op` | 复合赋值运算符之一 |
| `lhs`, `rhs` | 算术类型表达式（`lhs` 可以是限定或原子类型）；对于 `+=` 和 `-=`，也接受指针类型，限制与 `+` 和 `-` 运算符相同 |

**语义等价**：

表达式 `lhs @= rhs` 完全等价于 `lhs = lhs @ (rhs)`，区别在于 `lhs` 只被求值一次。

### 3.3 原子类型的复合赋值（C11 起）

如果 `lhs` 是原子类型，复合赋值操作表现为单个原子的读-修改-写操作，内存顺序为 `memory_order_seq_cst`。

对于整数原子类型，复合赋值 `@=` 等价于：

```c
T1* addr = &lhs;
T2 val = rhs;
T1 old = *addr;
T1 new;
do {
    new = old @ val;
} while (!atomic_compare_exchange_strong(addr, &old, new));
```

## 4. 底层原理 (Underlying Principles)

### 4.1 类型转换

赋值操作执行以下步骤：

1. **隐式转换**：将 `rhs` 的值转换为 `lhs` 的类型
2. **值存储**：用转换后的值替换 `lhs` 指定对象中的值
3. **返回值**：返回存储在 `lhs` 中的值（作为非左值）

### 4.2 求值顺序

- **副作用顺序**：更新 `lhs` 的副作用在 `lhs` 和 `rhs` 的值计算之后排序
- **操作数求值**：`lhs` 和 `rhs` 的副作用之间没有排序关系
- **重要后果**：表达式如 `i = ++i;` 是未定义行为 (undefined behavior)

### 4.3 浮点精度处理

赋值会去除浮点表达式中的额外范围和精度（参见 `FLT_EVAL_METHOD`）。

### 4.4 内存重叠

如果 `rhs` 和 `lhs` 在内存中重叠（例如它们是同一个联合体的成员），行为是未定义的，除非重叠是完全的且类型是兼容的。

### 4.5 数组的特殊规则

虽然数组本身不可赋值，但包装在结构体中的数组可以赋值给另一个相同（或兼容）结构体类型的对象。

## 5. 使用场景 (Use Cases)

### 5.1 适用场景

1. **变量初始化后的修改**：更新变量的值
2. **链式赋值**：多个变量赋相同的值
3. **复合操作**：对变量执行运算并更新其值
4. **指针操作**：修改指针指向的位置
5. **位操作**：设置、清除或切换特定位

### 5.2 最佳实践

1. **避免求值顺序问题**：不要在同一表达式中多次修改同一变量
2. **类型匹配**：确保赋值两边的类型兼容，避免隐式转换导致的精度丢失
3. **指针安全**：赋值前检查指针的有效性
4. **使用复合赋值**：代码更简洁，`lhs` 只求值一次，更安全

### 5.3 常见陷阱

| 陷阱 | 说明 | 示例 |
|------|------|------|
| 未定义行为 | 同一表达式中多次修改同一变量 | `i = ++i;` |
| 数组赋值错误 | 数组不能直接赋值 | `arr1 = arr2;` // 错误 |
| 指针限定符 | 不能增加被指向类型的限定符 | `char**` 不能赋值给 `const char**` |
| 浮点精度 | 赋值可能丢失精度 | `double` 到 `float` 的赋值 |

### 5.4 与初始化的区别

赋值与初始化（initialization）是不同的：

- **初始化**：在声明时给出初始值，如 `int i = 1;`
- **赋值**：修改变量的值，如 `i = 2;`

## 6. 代码示例 (Examples)

### 6.1 基础用法

```c
#include <stdio.h>

int main(void)
{
    // 初始化，不是赋值
    int i = 1, j = 2, k = 3;

    // 链式赋值
    i = j = k;   // i 和 j 的值都变为 3

    printf("%d %d %d\n", i, j, k);  // 输出: 3 3 3

    return 0;
}
```

### 6.2 指针赋值

```c
#include <stdio.h>

int main(void)
{
    const char c = 'A';
    const char *p = &c;          // 初始化，不是赋值
    const char **cpp = &p;        // 初始化，不是赋值

    // cpp = &p;  // 错误: char** 不能转换为 const char**

    *cpp = &c;     // 正确: char* 可以转换为 const char*
    printf("%c\n", **cpp);  // 输出: A

    cpp = 0;       // 正确: 空指针常量可以转换为任何指针类型

    return 0;
}
```

### 6.3 数组赋值

```c
#include <stdio.h>

int main(void)
{
    // 数组不能直接赋值
    int arr1[2] = {1, 2}, arr2[2] = {3, 4};
    // arr1 = arr2;  // 错误: 不能对数组赋值

    // 数组包装在结构体中可以赋值
    struct { int arr[2]; } sam1 = { {5, 6} }, sam2 = { {7, 8} };
    sam1 = sam2;  // 正确

    printf("%d %d\n", sam1.arr[0], sam1.arr[1]);  // 输出: 7 8

    return 0;
}
```

### 6.4 复合赋值运算符

```c
#include <stdio.h>

int main(void)
{
    int x = 10;
    int hundred = 100;
    int ten = 10;
    int fifty = 50;

    printf("%d %d %d %d\n", x, hundred, ten, fifty);
    // 输出: 10 100 10 50

    hundred *= x;   // hundred = 100 * 10 = 1000
    ten     /= x;   // ten = 10 / 10 = 1
    fifty   %= x;    // fifty = 50 % 10 = 0

    printf("%d %d %d %d\n", x, hundred, ten, fifty);
    // 输出: 10 1000 1 0

    return 0;
}
```

### 6.5 常见错误

#### 错误示例 1：对赋值结果再次赋值

```c
#include <stdio.h>

int main(void)
{
    int i = 1, j = 2, k = 3;

    // (i = j) = k;  // 错误: 在 C 中赋值结果不是左值

    return 0;
}
```

**修正方法**：如果需要实现类似功能，使用中间变量：

```c
int i = 1, j = 2, k = 3;
i = j;
i = k;
```

#### 错误示例 2：未定义的求值顺序

```c
#include <stdio.h>

int main(void)
{
    int i = 0;

    // i = ++i;  // 未定义行为: 同一表达式中读取和修改同一变量

    return 0;
}
```

**修正方法**：将操作分开：

```c
i = 0;
++i;  // 或 i++;
i = i;  // 如果确实需要赋值
```

## 7. 总结 (Summary)

### 7.1 核心要点

1. **简单赋值** (`=`) 将右侧值存储到左侧变量，支持隐式类型转换
2. **复合赋值** (`op=`) 组合运算和赋值，`lhs` 只求值一次
3. **返回值**：C 中赋值表达式返回非左值，不支持 `(a=b)=c` 形式
4. **类型要求**：必须满足类型兼容性或可转换性
5. **求值顺序**：操作数求值无序，可能导致未定义行为

### 7.2 技术对比

| 特性 | C 语言 | C++ |
|------|--------|-----|
| 赋值结果类型 | 非左值 (non-lvalue) | 左值 (lvalue) |
| `(a=b)=c` 合法性 | 非法 | 合法 |
| 数组赋值 | 不支持（除非包装在结构体中） | 同 C |
| 原子类型支持 | C11 起 | C++11 起 |

### 7.3 学习建议

1. **理解左值概念**：掌握什么是可修改的左值
2. **注意类型转换**：了解隐式类型转换的规则和潜在问题
3. **避免未定义行为**：不要在同一表达式中多次修改同一变量
4. **使用复合赋值**：代码更简洁、更安全
5. **区分初始化与赋值**：两者语法相似但语义不同
