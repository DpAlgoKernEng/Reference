# 替代运算符与记号 (Alternative Operators and Tokens)

## 1. 概述 (Overview)

C 源代码可以使用任何包含 ISO 646:1983 不变字符集（invariant character set）的 8 位字符集编写，包括非 ASCII 字符集。然而，多个 C 运算符和标点符号需要使用 ISO 646 字符集之外的字符：`{, }, [, ], #, \, ^, |, ~`。

为了支持某些或所有这些符号不存在的字符编码（如德国 DIN 66003），C 语言提供了两种解决方案：
- **运算符宏**：使用 ISO 646 兼容字符的替代拼写
- **替代记号**：特殊的多字符组合，被解释为单个非 ISO 646 字符

### 核心概念

| 概念 | 说明 |
|------|------|
| 替代运算符宏 | 定义在 `<iso646.h>` 中的宏，提供运算符的替代拼写 |
| 替代记号 (Alternative Tokens) | 语言核心部分的双字符组（digraphs），直接被编译器识别 |
| 三字符组 (Trigraphs) | 三字符序列，在 C23 中已移除 |

## 2. 来源与演变 (Origin and Evolution)

### 历史背景

在 20 世纪 80 年代和 90 年代初期，计算机系统中存在多种字符编码标准。ISO 646 是一个 7 位字符编码标准，它为不同国家预留了一些字符位置，导致这些位置的字符在不同国家有不同的解释。

### 版本演进

| 版本 | 变更内容 |
|------|----------|
| C95 | 引入 `<iso646.h>` 头文件，提供替代运算符宏；引入替代记号（双字符组） |
| C23 | 正式移除三字符组支持 |

### 设计动机

解决在非 ASCII 字符集环境下编写 C 代码的问题，特别是：
- 德国 DIN 66003 编码中不包含 `[, ], {, }, |, \` 等字符
- 其他欧洲国家编码也存在类似问题
- 需要确保代码可移植性

## 3. 语法与参数 (Syntax and Parameters)

### 3.1 运算符宏（C95）

以下替代拼写定义在 `<iso646.h>` 头文件中：

| 主运算符 | 替代拼写 | 说明 |
|----------|----------|------|
| `&&` | `and` | 逻辑与运算符 |
| `&=` | `and_eq` | 按位与赋值运算符 |
| `&` | `bitand` | 按位与运算符 |
| `\|` | `bitor` | 按位或运算符 |
| `~` | `compl` | 按位取反运算符 |
| `!` | `not` | 逻辑非运算符 |
| `!=` | `not_eq` | 不等于运算符 |
| `\|\|` | `or` | 逻辑或运算符 |
| `\|=` | `or_eq` | 按位或赋值运算符 |
| `^` | `xor` | 按位异或运算符 |
| `^=` | `xor_eq` | 按位异或赋值运算符 |

**注意**：字符 `&` 和 `!` 在 ISO 646 中是不变的，但为了适应更严格的历史字符集，仍然提供了替代拼写。

**没有提供 `==` 的替代拼写**：因为字符 `=` 在所有支持的字符集中都存在。

### 3.2 替代记号（C95）

以下替代记号是语言核心部分，在语言的各个方面，每个替代记号与其主记号行为完全相同（除了拼写，字符串化运算符可以使其拼写可见）：

| 主记号 | 替代记号 | 说明 |
|--------|----------|------|
| `{` | `<%` | 左花括号 |
| `}` | `%>` | 右花括号 |
| `[` | `<:` | 左方括号 |
| `]` | `:>` | 右方括号 |
| `#` | `%:` | 预处理指令符号 |
| `##` | `%:%:` | 记号粘贴运算符 |

双字符替代记号有时被称为"双字符组"（digraphs），尽管 `%:%:` 是四个字符，也被视为双字符组。

### 3.3 三字符组（C23 中已移除）

以下三字符组在注释和字符串字面量被识别之前进行处理：

| 主字符 | 三字符组 | 说明 |
|--------|----------|------|
| `{` | `??<` | 左花括号 |
| `}` | `??>` | 右花括号 |
| `[` | `??(` | 左方括号 |
| `]` | `??)` | 右方括号 |
| `#` | `??=` | 预处理指令符号 |
| `\` | `??/` | 反斜杠 |
| `^` | `??'` | 脱字符 |
| `\|` | `??!` | 竖线 |
| `~` | `??-` | 波浪号 |

## 4. 底层原理 (Underlying Principles)

### 4.1 处理阶段

C 编译器对替代记号和三字符组的处理发生在不同的翻译阶段：

| 阶段 | 处理内容 | 说明 |
|------|----------|------|
| 第 1 阶段 | 三字符组替换 | 在任何其他处理之前进行 |
| 第 2 阶段 | 行拼接 | 处理反斜杠换行 |
| 第 3 阶段 | 词法分析 | 双字符组在此阶段被识别 |
| 第 7 阶段 | 宏展开 | `<iso646.h>` 中的宏在此展开 |

### 4.2 三字符组的早期处理风险

由于三字符组在注释和字符串字面量识别之前处理，可能导致意外行为：

```c
// 注释：下面这行会被执行吗?????/
// 这行实际上被注释掉了！

// 上面的 ??/ 被替换为 \，导致行拼接
```

### 4.3 替代记号的词法特性

替代记号是真正的记号（token），不是宏：
- 编译器直接识别
- 不需要包含任何头文件
- 字符串化运算符 `#` 会保留其原始拼写

```c
#define STRINGIZE(x) #x
STRINGIZE(<%)  // 结果为 "<%"，而不是 "{"
```

## 5. 使用场景 (Use Cases)

### 5.1 适用场景

| 场景 | 说明 |
|------|------|
| 历史代码维护 | 维护使用替代记号编写的旧代码 |
| 跨平台移植 | 处理使用受限字符集的系统 |
| 国际化编程 | 在非 ASCII 环境下编写代码 |
| 代码混淆 | 使用替代记号增加代码可读性难度 |

### 5.2 最佳实践

**推荐做法**：
1. **现代代码应避免使用**：几乎所有现代系统都支持完整的 ASCII 字符集
2. **优先使用主记号**：提高代码可读性和可维护性
3. **仅在必要时使用 `<iso646.h>`**：某些键盘布局下 `&&` 和 `||` 输入困难

**注意事项**：
1. 三字符组在 C23 中已移除，不应在新代码中使用
2. 替代记号会使代码难以阅读，应谨慎使用
3. 某些代码格式化工具可能不支持替代记号

### 5.3 常见陷阱

| 陷阱 | 说明 | 解决方案 |
|------|------|----------|
| 字符串中的意外替换 | 三字符组在字符串中也会被替换 | 使用 `??` 的转义或升级到 C23 |
| 注释中的三字符组 | 三字符组在注释中也会被替换 | 避免在注释中使用 `??` 序列 |
| 代码混淆 | 替代记号降低可读性 | 统一代码风格，避免混用 |

## 6. 代码示例 (Examples)

### 6.1 基础用法：运算符宏

```c
#include <stdio.h>
#include <iso646.h>  // 引入替代运算符宏

int main(void) {
    int a = 5, b = 3;

    // 使用替代运算符宏
    if (a > 0 and b > 0) {
        printf("Both positive\n");
    }

    if (a not_eq b) {
        printf("a != b\n");
    }

    // 按位运算
    int c = a bitand b;  // 等价于 a & b
    int d = a bitor b;   // 等价于 a | b
    int e = compl a;     // 等价于 ~a

    printf("a & b = %d\n", c);
    printf("a | b = %d\n", d);
    printf("~a = %d\n", e);

    return 0;
}
```

**输出**：
```
Both positive
a != b
a & b = 1
a | b = 7
~a = -6
```

### 6.2 基础用法：替代记号（双字符组）

```c
#include <stdio.h>

int main(void)
<%  // 替代 {
    int arr<:3:>;  // 替代 arr[3]

    arr<:0:> = 10;
    arr<:1:> = 20;
    arr<:2:> = 30;

    for (int i = 0; i < 3; i++)
    <%
        printf("arr[%d] = %d\n", i, arr<:i:>);
    %>

    return 0;
%>  // 替代 }
```

**输出**：
```
arr[0] = 10
arr[1] = 20
arr[2] = 30
```

### 6.3 高级用法：预处理指令中的替代记号

```c
%:include <stdio.h>  // 替代 #include
%:define MAX(a, b) ((a) > (b) ? (a) : (b))  // 替代 #define

int main(void)
<%
    int x = 10, y = 20;
    printf("Max: %d\n", MAX(x, y));

    // 记号粘贴
    %:define CONCAT(a, b) a %:%: b  // 替代 ##
    int CONCAT(var, 1) = 100;  // 创建变量 var1

    return 0;
%>
```

### 6.4 完整示例：混合使用（改编自原文档）

```c
%:include <stdio.h>
%:include <stdlib.h>
#include <iso646.h>

int main(int argc, char** argv)
<%
    if (argc > 1 and argv<:1:> not_eq NULL)
    <%
        printf("Hello %s\n", argv<:1:>);
    %>
    else
    <%
        printf("Hello %s\n", argc ? argv<:0:> : "world");
    %>

    return EXIT_SUCCESS;
%>
```

### 6.5 常见错误：三字符组的意外行为（C23 之前）

```c
/* 错误示例：C23 之前的行为 */

#include <stdio.h>

int main(void)
<%
    // 这个注释中的 ??/ 会被替换为 \
    // 导致下一行被注释掉！
    printf("This might not print??/n");  // ??/n 变成 \n

    // 字符串中的三字符组
    printf("What's going on??!");  // 输出: What's going on|

    return 0;
%>
```

**修正版本**：

```c
#include <stdio.h>

int main(void)
<%
    // 避免在注释和字符串中使用 ?? 序列
    printf("This will print correctly\n");

    // 如果需要在字符串中使用，拆分 ??
    printf("Question marks: ?" "?!");  // 输出: ??!

    return 0;
%>
```

### 6.6 现代代码示例：可读性对比

```c
/* 现代推荐写法 */
#include <stdio.h>
#include <stdbool.h>

bool is_valid(int value) {
    return (value >= 0) && (value <= 100);  // 清晰易读
}

/* 不推荐的替代写法 */
#include <stdio.h>
#include <iso646.h>

bool is_valid_alt(int value) {
    return (value >= 0) and (value <= 100);  // 需要额外头文件
}
```

## 7. 总结 (Summary)

### 7.1 核心要点

| 特性 | 引入版本 | 状态 | 建议 |
|------|----------|------|------|
| 运算符宏 (`<iso646.h>`) | C95 | 仍然有效 | 现代代码不需要使用 |
| 替代记号（双字符组） | C95 | 仍然有效 | 避免使用，降低可读性 |
| 三字符组 | C89 | C23 移除 | 不再使用 |

### 7.2 技术对比

| 特性 | 三字符组 | 双字符组 | 运算符宏 |
|------|----------|----------|----------|
| 处理阶段 | 第 1 阶段 | 第 3 阶段 | 第 7 阶段 |
| 需要头文件 | 否 | 否 | 是 |
| 影响字符串字面量 | 是 | 否 | 否 |
| 现代可用性 | 已弃用 | 可用但不推荐 | 可用但不推荐 |

### 7.3 学习建议

1. **了解历史**：理解这些特性存在的原因有助于阅读历史代码
2. **避免使用**：现代开发环境中，应使用标准运算符和标点符号
3. **升级代码**：如遇到使用三字符组的旧代码，应考虑重构
4. **注意可移植性**：C23 之后，使用三字符组的代码将无法编译

### 7.4 参考资源

- C95 标准：引入 `<iso646.h>` 和替代记号
- C23 标准：移除三字符组
- ISO 646:1983：定义了不变字符集
