# 值类别 (Value Categories)

## 1. 概述 (Overview)

在 C 语言中，每个表达式（包括运算符及其操作数、函数调用、常量、变量名等）都具有两个独立的属性：**类型 (type)** 和 **值类别 (value category)**。

每个表达式都属于以下三种值类别之一：

| 值类别 | 英文名称 | 简要描述 |
|--------|----------|----------|
| 左值 | lvalue | 标识一个对象的表达式 |
| 非左值对象表达式 | non-lvalue object expression (rvalue) | 不标识对象，表示值的表达式 |
| 函数指示符 | function designator | 标识函数的表达式 |

值类别决定了表达式在特定上下文中的行为方式，例如能否作为赋值的左操作数、能否取地址等。

## 2. 来源与演变 (Origin and Evolution)

### 历史背景

"左值"（lvalue，即 left value）这一术语源于 CPL 编程语言，反映了左值表达式传统上作为赋值运算符左操作数的使用方式。这一术语后来被 C 语言继承并发展。

### C 语言标准演进

| 标准 | 章节 | 说明 |
|------|------|------|
| C89/C90 | 3.2.2.1 | 首次标准化左值和函数指示符的概念 |
| C99 | 6.3.2.1 | 明确了复合字面量（compound literals）作为左值 |
| C11 | 6.3.2.1 | 继承 C99，无重大变更 |
| C17 | 6.3.2.1 | 继续沿用现有定义 |

### 与 C++ 的差异

需要注意的是，C 和 C++ 在值类别上存在显著差异：
- C++ 中的前置自增/自减表达式是左值，而 C 中不是
- C++ 中的赋值表达式是左值，而 C 中不是
- C++ 中的条件运算符在特定条件下是左值
- C++ 中的逗号运算符在特定条件下是左值

## 3. 语法与参数 (Syntax and Parameters)

### 3.1 左值表达式 (Lvalue Expressions)

左值表达式是指任何具有对象类型（`void` 类型除外）的表达式，它潜在地指定一个对象。如果在求值时左值实际上未指定对象，则行为是未定义的。

**定义要点**：
- 左值表达式求值结果为**对象身份 (object identity)**
- 必须具有对象类型（非 `void`）
- 当求值时必须实际指定一个有效对象

### 3.2 可修改左值 (Modifiable Lvalue)

**可修改左值**是指满足以下所有条件的左值表达式：

1. 具有完整类型
2. 不是数组类型
3. 不是 `const` 限定类型
4. 如果是结构体或联合体，则没有任何成员（递归地）是 `const` 限定类型

只有可修改左值才能用于：
- 自增/自减运算符的操作数
- 赋值运算符和复合赋值运算符的左操作数

### 3.3 非左值对象表达式 (Non-lvalue Object Expressions)

非左值对象表达式也称为**右值 (rvalue)**，是不指定对象但表示值的表达式。右值没有对象身份或存储位置。

**关键特征**：
- 不能对右值取地址
- 表示值本身，而非对象

### 3.4 函数指示符 (Function Designator)

函数指示符是由函数声明引入的标识符，具有函数类型。在大多数上下文中，函数指示符会自动转换为指向函数的非左值指针。

## 4. 底层原理 (Underlying Principles)

### 4.1 左值上下文 (Lvalue Contexts)

左值表达式可以在以下**左值上下文**中使用：

| 上下文 | 条件限制 |
|--------|----------|
| 取地址运算符 `&` 的操作数 | 左值不能指定位域，且不能声明为 `register` |
| 前置/后置自增/自减运算符 | 必须是可修改左值 |
| 成员访问运算符 `.` 的左操作数 | - |
| 赋值运算符 `=` 和复合赋值运算符的左操作数 | 必须是可修改左值 |

### 4.2 左值转换 (Lvalue Conversion)

当左值表达式用于除 `sizeof`、`_Alignof` 或上述左值上下文之外的任何上下文时，会发生**左值转换 (lvalue conversion)**：

**转换过程**：
1. 从对象位置读取存储的值
2. 结果为该值类型的非左值表达式

```
左值表达式 ──左值转换──> 右值（值本身）
```

**类型限定符的处理**：
- `const`、`volatile`、`restrict` 限定符仅适用于左值
- 左值转换会剥离这些限定符
- 原子类型 (atomic type) 也会在左值转换中失去原子性

### 4.3 数组到指针转换 (Array-to-Pointer Conversion)

当数组左值用于除以下情况之外的任何上下文时，会发生**数组到指针转换**：
- `sizeof` 运算符
- `_Alignof` 运算符
- 取地址运算符 `&`
- 用于初始化的字符串字面量

### 4.4 函数指示符转换

当函数指示符用于除以下情况之外的任何上下文时，会转换为指向函数的非左值指针：
- 取地址运算符 `&`
- `sizeof` 运算符（会导致编译错误）
- `_Alignof` 运算符（会导致编译错误）

**注意**：函数调用运算符是为函数指针定义的，而非函数指示符本身。

### 4.5 特殊情况：void 类型

`void` 类型的表达式被视为非左值对象表达式，其产生的值没有表示形式，也不需要存储。

### 4.6 特殊情况：结构体/联合体右值中的数组成员

如果结构体或联合体右值具有数组类型的成员（可能是嵌套的），该右值实际上指定了一个具有临时生存期的对象。可以通过以下方式访问该临时对象：
- 对数组成员进行下标索引
- 通过对数组成员进行数组到指针转换后得到的指针进行间接访问

## 5. 使用场景 (Use Cases)

### 5.1 左值表达式的使用场景

#### 左值表达式列表

以下表达式是左值：

| 表达式类型 | 说明 |
|-----------|------|
| 标识符 | 包括函数命名参数，必须声明为指定对象（非函数或枚举常量） |
| 字符串字面量 | 如 `"hello"` |
| 复合字面量 (C99) | 如 `(int){42}` |
| 括号表达式 | 如果去除括号的表达式是左值 |
| 成员访问 `.` | 如果左操作数是左值，如 `s.member` |
| 指针成员访问 `->` | 如 `ptr->member` |
| 间接访问 `*` | 应用于指向对象的指针，如 `*ptr` |
| 下标运算符 `[]` | 如 `arr[i]` |

### 5.2 非左值对象表达式的使用场景

#### 非左值对象表达式列表

以下表达式是非左值对象表达式：

| 表达式类型 | 说明 |
|-----------|------|
| 整数常量 | 如 `42`、`0xFF` |
| 字符常量 | 如 `'a'`、`L'x'` |
| 浮点常量 | 如 `3.14`、`1.0e-10` |
| 函数调用表达式 | 如 `func()` |
| 强制转换表达式 | 如 `(int)x`，注意复合字面量除外 |
| 成员访问 `.` | 应用于非左值结构体/联合体，如 `func().x`、`(x,s1).a` |
| 算术运算符结果 | 如 `a + b`、`-x`、`a * b` |
| 关系运算符结果 | 如 `a < b`、`x == y` |
| 逻辑运算符结果 | 如 `a && b`、`!x` |
| 位运算符结果 | 如 `a & b`、`x << 2` |
| 自增/自减运算符结果 | 包括前置和后置形式（C++ 中前置是左值） |
| 赋值运算符结果 | 如 `a = b`（C++ 中是左值） |
| 条件运算符 | 如 `a ? b : c`（C++ 中在特定条件下是左值） |
| 逗号运算符 | 如 `a, b`（C++ 中如果第二个操作数是左值则结果也是左值） |
| 取地址运算符 | 如 `&x`，即使应用于 `*` 运算符的结果 |

### 5.3 函数指示符的使用

函数指示符作为表达式的一种，主要用于：
- 函数调用：`func(args)`（自动转换为函数指针）
- 取地址：`&func`（得到函数指针）
- 初始化函数指针：`fp = func`（自动转换）

## 6. 代码示例 (Examples)

### 6.1 左值表达式示例

```c
#include <stdio.h>

int main(void) {
    // 左值表达式示例

    // 1. 变量标识符是左值
    int x = 10;
    x = 20;              // x 是左值，可作为赋值左操作数

    // 2. 字符串字面量是左值
    char *s = "hello";   // "hello" 是左值，但类型为 char[6]
    printf("%s\n", s);

    // 3. 下标运算符结果是左值
    int arr[5] = {1, 2, 3, 4, 5};
    arr[2] = 30;         // arr[2] 是左值

    // 4. 间接访问运算符结果是左值
    int *ptr = &x;
    *ptr = 100;          // *ptr 是左值

    // 5. 成员访问运算符结果是左值
    struct Point { int x; int y; };
    struct Point p = {1, 2};
    p.x = 10;            // p.x 是左值（左操作数 p 是左值）

    // 6. 指针成员访问运算符结果是左值
    struct Point *pp = &p;
    pp->y = 20;          // pp->y 是左值

    // 7. 括号表达式
    (x) = 200;           // (x) 是左值，因为 x 是左值

#if __STDC_VERSION__ >= 199901L
    // 8. C99 复合字面量是左值
    int *pl = &(int){42};  // (int){42} 是左值，可取地址
    printf("compound literal: %d\n", *pl);
#endif

    return 0;
}
```

### 6.2 可修改左值 vs 不可修改左值

```c
#include <stdio.h>

int main(void) {
    // 可修改左值
    int a = 10;
    a++;                 // OK: a 是可修改左值
    a = 20;              // OK

    // 不可修改左值
    const int b = 5;
    // b = 10;           // 错误: b 是 const 限定的左值，不可修改
    // b++;              // 错误

    // 数组名是左值但不可修改
    int arr[3] = {1, 2, 3};
    // arr = {4, 5, 6};  // 错误: 数组类型不能赋值

    // 结构体中包含 const 成员
    struct Data {
        int x;
        const int y;
    };

    struct Data d1 = {1, 2};
    d1.x = 10;           // OK
    // d1.y = 20;        // 错误: d1.y 是 const 限定
    // d1 = (struct Data){3, 4};  // 错误: d1 含有 const 成员，整体不可修改

    return 0;
}
```

### 6.3 右值（非左值对象表达式）示例

```c
#include <stdio.h>

int get_value(void) {
    return 42;
}

struct Point {
    int x, y;
};

struct Point get_point(void) {
    return (struct Point){1, 2};
}

int main(void) {
    // 右值示例

    // 1. 常量是右值
    int a = 10;          // 10 是右值
    double b = 3.14;     // 3.14 是右值

    // 2. 算术运算结果是右值
    int c = a + 5;       // a + 5 是右值
    int d = a * b;       // a * b 是右值

    // 3. 函数调用结果是右值
    int e = get_value(); // get_value() 是右值

    // 4. 不能对右值取地址
    // int *p1 = &(a + 5);   // 错误: 不能取右值的地址
    // int *p2 = &get_value();  // 错误

    // 5. 右值不能作为赋值左操作数
    // (a + 1) = 10;       // 错误: 右值不能赋值

    // 6. 自增/自减结果是右值（与 C++ 不同）
    int x = 5;
    int y = x++;         // x++ 的结果是右值
    // (x++) = 10;        // 错误: 自增结果不是左值

    // 7. 赋值表达式结果是右值（与 C++ 不同）
    int z;
    int w = (z = 10);    // (z = 10) 的结果是右值
    // (z = 10) = 20;     // 错误: 赋值结果不是左值

    // 8. 条件运算符结果是右值（与 C++ 不同）
    int m = 1, n = 2;
    int result = (m > 0) ? m : n;  // 结果是右值

    // 9. 结构体右值中的数组成员访问
    struct Point pt = get_point();
    // get_point().x 是右值，但可以通过临时对象访问
    int val = get_point().x;  // OK: 访问临时结构体的成员

    return 0;
}
```

### 6.4 函数指示符示例

```c
#include <stdio.h>

void greet(void) {
    printf("Hello!\n");
}

int add(int a, int b) {
    return a + b;
}

int main(void) {
    // 函数指示符示例

    // 1. 函数名是函数指示符
    greet();             // greet 是函数指示符，自动转换为函数指针

    // 2. 取函数指示符的地址
    void (*fp1)(void) = &greet;  // 显式取地址
    void (*fp2)(void) = greet;  // 隐式转换
    fp1();               // 通过函数指针调用
    fp2();

    // 3. 函数指针的用法
    int (*adder)(int, int) = add;
    int sum = adder(3, 4);
    printf("Sum: %d\n", sum);

    // 4. sizeof 不能用于函数指示符
    // size_t s = sizeof(greet);  // 错误: 不能对函数使用 sizeof

    // 5. _Alignof 不能用于函数指示符
    // size_t a = _Alignof(greet);  // 错误

    return 0;
}
```

### 6.5 左值转换示例

```c
#include <stdio.h>

int main(void) {
    // 左值转换演示

    int x = 10;
    int y = 20;

    // x 和 y 作为左值出现在 sizeof 中，不发生转换
    printf("sizeof(x) = %zu\n", sizeof(x));

    // x 和 y 用在表达式中，发生左值转换
    int sum = x + y;     // x 和 y 被转换为其存储的值 10 和 20

    printf("x + y = %d\n", sum);

    // 左值转换剥离限定符
    const int cx = 100;
    // cx 作为左值在 sizeof 中不发生转换
    printf("sizeof(cx) = %zu\n", sizeof(cx));

    // cx 用于表达式时发生左值转换，结果为 int 类型的右值（const 被剥离）
    int val = cx + 5;    // cx 发生左值转换，类型变为 int
    printf("cx + 5 = %d\n", val);

    // 数组到指针转换
    int arr[5] = {1, 2, 3, 4, 5};
    int *p = arr;        // arr 发生数组到指针转换，变为 int*
    printf("arr[0] = %d\n", *p);

    // sizeof 中数组不转换
    printf("sizeof(arr) = %zu (bytes)\n", sizeof(arr));  // 输出 20（5 * 4）

    // 取地址时数组不转换
    int (*ap)[5] = &arr; // 取数组的地址
    printf("(*ap)[0] = %d\n", (*ap)[0]);

    return 0;
}
```

### 6.6 常见错误及修正

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    // 常见错误 1: 对右值取地址
    int x = 10;
    // int *p = &(x + 5);  // 错误: 不能对右值取地址

    // 修正: 使用变量存储中间结果
    int temp = x + 5;
    int *p = &temp;       // OK

    // 常见错误 2: 对右值赋值
    // (x + 1) = 20;       // 错误: 右值不能作为赋值左操作数

    // 修正: 直接赋值给变量
    x = 20;               // OK

    // 常见错误 3: 自增/自减应用于非左值
    // int y = (x++)++;    // 错误: x++ 的结果是右值

    // 修正: 分步操作
    x++;
    x++;                  // OK

    // 常见错误 4: 误解 const 变量
    const int c = 100;
    // c = 200;            // 错误: const 变量不可修改

    // 注意: const 变量的左值转换剥离 const
    int d = c;             // OK: d = 100，类型为 int
    d = 200;               // OK: d 可以修改，与 c 无关

    // 常见错误 5: 对 register 变量取地址
    register int reg = 10;
    // int *rp = &reg;      // 错误: 不能对 register 变量取地址

    // 修正: 移除 register 关键字
    int non_reg = 10;
    int *rp = &non_reg;    // OK

    // 常见错误 6: 对位域取地址
    struct Flags {
        unsigned int a : 1;
        unsigned int b : 7;
    } flags = {1, 42};

    // unsigned int *bp = &flags.b;  // 错误: 不能对位域取地址

    // 修正: 先复制值
    unsigned int b_val = flags.b;  // OK

    // 常见错误 7: 误解函数指示符
    void greet(void) { printf("Hello\n"); }  // 假设在 main 外部

    // size_t s = sizeof(greet);  // 错误: sizeof 不能用于函数

    // 修正: 使用函数指针
    void (*fp)(void) = greet;
    size_t fp_size = sizeof(fp);  // OK: 计算指针大小

    return 0;
}
```

## 7. 总结 (Summary)

### 7.1 核心要点

| 概念 | 要点 |
|------|------|
| **左值** | 标识对象，可取地址（特定条件下），可修改（如果是可修改左值） |
| **右值** | 表示值，无对象身份，不可取地址，不可修改 |
| **函数指示符** | 标识函数，自动转换为函数指针 |
| **左值转换** | 左值在非左值上下文中转换为右值 |
| **可修改左值** | 完整、非数组、非 const 限定的左值 |

### 7.2 值类别对比表

| 特性 | 左值 | 右值 | 函数指示符 |
|------|------|------|------------|
| 可取地址 | 是（有限制） | 否 | 是 |
| 可赋值 | 可修改左值可以 | 否 | 否 |
| 类型 | 对象类型 | 对象类型 | 函数类型 |
| 求值结果 | 对象身份 | 值 | 函数身份 |
| 自动转换 | 左值转换 | 无 | 函数指针转换 |

### 7.3 C 与 C++ 值类别差异

| 表达式 | C 中 | C++ 中 |
|--------|------|--------|
| 前置 `++x` / `--x` | 右值 | 左值 |
| 赋值 `x = y` | 右值 | 左值 |
| 条件 `a ? b : c`（b、c 同类型左值） | 右值 | 左值 |
| 逗号 `a, b`（b 是左值） | 右值 | 左值 |

### 7.4 学习建议

1. **理解本质**：左值表示"对象身份"，右值表示"值本身"，函数指示符表示"函数身份"
2. **掌握转换规则**：左值转换和数组到指针转换是 C 语言类型系统的核心
3. **注意限定符**：`const`、`volatile`、`restrict` 仅对左值有意义
4. **区分 C 和 C++**：C++ 扩展了左值的范围，混淆使用可能导致可移植性问题
5. **实践验证**：通过编译器错误信息理解值类别的限制
