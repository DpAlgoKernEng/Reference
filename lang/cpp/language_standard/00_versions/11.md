# C++11 语言标准

## 1. 概述

**C++11** 是 C++ 语言的第二个主要版本，也是自 C++98 以来最重要的更新。该版本引入了大量新特性，旨在标准化现有实践并为 C++ 程序员提供更强大的抽象能力。

C++11 在 C++ 语言发展史上具有里程碑意义：
- 从 C++03 到 C++11 经历了 8 年，这是迄今为止各版本间隔时间最长的一次
- 自 C++11 起，C++ 语言确立了每 3 年定期更新的发布节奏
- 新特性来源广泛：TR1 技术报告、Boost 库、C 语言等

## 2. 来源与演变

### 2.1 历史背景

在 2011 年 8 月 12 日 ISO 最终批准之前，该版本曾被称为 **C++0x**，因为原计划在 2010 年之前发布。由于开发周期较长，C++11 成为了 C++03 之后时隔 8 年的重大更新。

### 2.2 特性来源

C++11 的特性来自多个渠道：

| 来源 | 具体内容 |
|------|----------|
| TR1 | TR1 的全部内容（除特殊数学函数外） |
| Boost | 线程库、`exception_ptr`、`error_code` 和 `error_condition`、迭代器改进（`begin`、`end`、`next`、`prev`） |
| C 语言 | C 风格 Unicode 转换函数 |

### 2.3 缺陷报告修复

C++11 修复了大量遗留缺陷：
- 核心语言缺陷：741 个（CWG #4 至 CWG #1246）
- 库缺陷：868 个（LWG #23 至 LWG #2042）

## 3. 核心语言特性

C++11 引入了以下核心语言特性：

### 3.1 类型系统增强

| 特性 | 说明 |
|------|------|
| `auto` 和 `decltype` | 自动类型推导 |
| `nullptr` | 空指针常量，替代 `NULL` 和 `0` |
| `long long` | 64 位整型 |
| `char16_t` 和 `char32_t` | Unicode 字符类型 |
| 类型别名（Type Alias `using`） | 替代 `typedef` 的新型别名语法 |
| 强类型枚举（Scoped Enum） | 作用域限定枚举，避免名称污染 |
| 广义 POD（Generalized POD） | 重新定义平凡类型和标准布局类型 |
| 广义联合体（Generalized Union） | 支持非平凡成员的联合体 |

### 3.2 函数与控制流

| 特性 | 说明 |
|------|------|
| 尾置返回类型（Trailing Return Type） | 函数返回类型置于参数列表之后 |
| `constexpr` 和字面类型 | 编译期常量表达式计算 |
| 默认化与删除函数（Defaulted/Deleted Functions） | `= default` 和 `= delete` |
| 委托构造函数（Delegating Constructor） | 构造函数调用同类其他构造函数 |
| 继承构造函数（Inheriting Constructor） | 派生类继承基类构造函数 |
| `override` 和 `final` | 虚函数覆盖标记和禁止派生标记 |
| `noexcept` 说明符与运算符 | 异常说明和异常检查 |
| `static_assert` | 编译期静态断言 |
| 范围 `for`（Range-based For） | 基于范围的循环遍历 |
| Lambda 表达式 | 匿名函数对象 |

### 3.3 模板与泛型

| 特性 | 说明 |
|------|------|
| 可变参数模板（Variadic Template） | 支持任意数量模板参数 |
| 模板别名（Template Alias） | 使用 `using` 定义模板别名 |
| 外部模板（Extern Template） | 显式实例化声明，减少编译时间 |

### 3.4 内存与并发

| 特性 | 说明 |
|------|------|
| 右值引用（Rvalue Reference） | 移动语义基础 |
| 移动构造函数与移动赋值运算符 | 资源转移而非拷贝 |
| 多线程内存模型（Multithreaded Memory Model） | 定义多线程环境下的内存行为 |
| 线程本地存储（Thread-local Storage） | `thread_local` 关键字 |
| `alignof` 和 `alignas` | 对齐查询和对齐指定 |
| GC 接口 | 垃圾收集器接口（C++23 中移除） |

### 3.5 其他特性

| 特性 | 说明 |
|------|------|
| 列表初始化（List Initialization） | 统一初始化语法 `{}` |
| 花括号或等号初始化器（Brace-or-Equal Initializer） | 类成员就地初始化 |
| Unicode 字符串字面量 | `u""`、`U""`、`u8""` |
| 原始字符串字面量（Raw String Literal） | 不含转义字符的字符串 |
| 用户定义字面量（User-defined Literal） | 自定义字面量运算符 |
| 属性（Attribute） | `[[noreturn]]`、`[[carries_dependency]]` 等 |
| 内联命名空间（Inline Namespace） | 命名空间成员自动暴露 |
| 右尖括号修正 | `>>` 不再需要空格分隔 |

### 4.2 库功能增强

#### 4.2.1 并发支持库

完整的多线程支持，包括：
- 线程（`std::thread`）
- 互斥量（`std::mutex`、`std::shared_mutex`）
- 条件变量（`std::condition_variable`）
- 异步操作（`std::async`、`std::future`、`std::promise`）
- 原子操作（`std::atomic`）

#### 4.2.2 智能指针

| 类型 | 说明 |
|------|------|
| `std::unique_ptr` | 独占所有权智能指针 |
| `std::shared_ptr` | 共享所有权智能指针 |
| `std::weak_ptr` | 弱引用指针 |

#### 4.2.3 容器增强

- `emplace()` 系列方法：就地构造，避免拷贝
- `std::forward_list`：单向链表
- `std::array`：固定大小数组
- `std::unordered_map`、`std::unordered_set`：哈希容器
- 全容器支持右值引用优化

#### 4.2.4 新增算法


#### 4.2.5 迭代器改进

- `std::begin` / `std::end`：统一获取迭代器
- `std::next` / `std::prev`：迭代器前进后退
- `std::move_iterator`：移动迭代器适配器

#### 4.2.6 其他增强

- `std::function`：通用函数包装器
- `std::exception_ptr`：异常捕获与延迟处理
- `std::error_code` / `std::error_condition`：错误码机制
- 有状态和作用域分配器
- Unicode 转换 facet

## 5. 使用场景

### 5.1 适用场景

C++11 特性适用于以下场景：

| 场景 | 推荐特性 |
|------|----------|
| 减少类型声明冗余 | `auto`、`decltype` |
| 编译期计算 | `constexpr` |
| 多线程编程 | `<thread>`、`<atomic>`、`<mutex>` |
| 资源管理 | `std::unique_ptr`、`std::shared_ptr`、RAII |
| 函数式编程 | Lambda 表达式 |
| 泛型编程 | 可变参数模板、`std::function` |
| 性能优化 | 右值引用、移动语义 |
| 代码简化 | 范围 `for`、列表初始化 |

### 5.2 最佳实践

#### 5.2.1 类型推导

```cpp
// ✅ 推荐：简化复杂类型声明
auto iter = container.begin();
auto lambda = [](int x) { return x * 2; };

// ❌ 避免：过度使用导致可读性下降
auto x = some_function();  // 不清楚返回类型时应避免
```

#### 5.2.2 智能指针优先

```cpp
// ✅ 推荐：使用智能指针管理资源
auto ptr = std::make_unique<MyClass>();
auto shared = std::make_shared<MyClass>();

// ❌ 避免：裸指针管理资源
MyClass* ptr = new MyClass();  // 需要手动 delete
```

#### 5.2.3 移动语义

```cpp
// ✅ 推荐：移动而非拷贝
std::vector<int> v1 = {1, 2, 3};
std::vector<int> v2 = std::move(v1);  // v1 被清空

// ✅ 推荐：函数返回时利用移动语义
std::vector<int> create_vector() {
    std::vector<int> result = {1, 2, 3};
    return result;  // 自动移动，无需显式 std::move
}
```

### 5.3 注意事项

- **编译器支持**：GCC 4.8+、Clang 3.3+、MSVC 19.0+（2015）完整支持 C++11
- **缺陷报告成员函数**：部分缺陷修复新增的成员函数在 C++98 模式下不可用（如 `std::basic_string::back()`、`shrink_to_fit()` 等）
- **Unicode 字符类型**：`char16_t` 和 `char32_t` 支持需要编译器和运行时配合

## 6. 代码示例

### 6.1 基础特性示例

```cpp
#include <iostream>
#include <vector>
#include <memory>
#include <thread>

#include <functional>
#include <algorithm>

// auto 类型推导
auto process_data(const std::vector<int>& data) {
    auto result = 0;
    for (const auto& item : data) {  // 范围 for
        result += item;
    }
    return result;

// constexpr 编译期计算
constexpr int factorial(int n) {
    return n <= 1 ? 1 : n * factorial(n - 1);
}

// nullptr 替代 NULL
void process_pointer(int* ptr) {
    if (ptr != nullptr) {
        std::cout << "Valid pointer\n";
    }
}

// 强类型枚举

// 初始化列表
std::vector<int> numbers = {1, 2, 3, 4, 5};

// Lambda 表达式

int main() {
    // constexpr 使用
    static_assert(factorial(5) == 120, "Factorial calculation error");
    
    // 智能指针
    auto ptr = std::make_unique<int>(42);
    auto shared = std::make_shared<int>(100);
    
    // 算法使用
    std::vector<int> data = {1, 2, 3, 4, 5};
    bool all_positive = std::all_of(data.begin(), data.end(), 
                                    [](int x) { return x > 0; });

    return 0;
```

### 6.2 多线程示例

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <atomic>
#include <vector>

std::mutex mtx;
std::atomic<int> counter{0};

void increment(int times) {
    for (int i = 0; i < times; ++i) {
        // 使用原子操作
        counter++;
        
        // 或使用互斥量保护临界区
        {
            std::lock_guard<std::mutex> lock(mtx);
            // 执行需要同步的操作
        }
    }
}

int main() {
    std::vector<std::thread> threads;
    
    // 创建多个线程
    for (int i = 0; i < 10; ++i) {
        threads.emplace_back(increment, 1000);
    }
    
    // 等待所有线程完成
    for (auto& t : threads) {
        t.join();
    }
    
    std::cout << "Counter: " << counter << std::endl;
    return 0;
}
```

### 6.3 可变参数模板示例

```cpp
#include <iostream>

// 可变参数模板：递归展开
template<typename T>
void print(T value) {
    std::cout << value << std::endl;
}

template<typename T, typename... Args>

    std::cout << first << ", ";
    print(rest...);  // 递归调用，展开剩余参数
}

```

### 6.4 右值引用与移动语义示例

```cpp
#include <iostream>
#include <utility>
#include <vector>

class Buffer {
public:
    Buffer(size_t size) : size_(size), data_(new int[size]) {}
    
    // 移动构造函数
    Buffer(Buffer&& other) noexcept 
        : size_(other.size_), data_(other.data_) {
        other.size_ = 0;
        other.data_ = nullptr;
    }
    
    // 移动赋值运算符
    Buffer& operator=(Buffer&& other) noexcept {
        if (this != &other) {
            delete[] data_;
            size_ = other.size_;
            data_ = other.data_;
            other.size_ = 0;
            other.data_ = nullptr;
        }
        return *this;
    }
    
    // 禁止拷贝
    Buffer(const Buffer&) = delete;
    Buffer& operator=(const Buffer&) = delete;
    
    ~Buffer() { delete[] data_; }
    
private:
    size_t size_;
    int* data_;
};

int main() {
    Buffer b1(100);
    Buffer b2 = std::move(b1);  // 移动而非拷贝
    
    // b1 被清空，b2 获得资源所有权
    return 0;
}
```

## 7. 总结

### 7.1 核心要点

C++11 是现代 C++ 的起点，引入的关键特性包括：

1. **类型推导**：`auto` 和 `decltype` 大幅简化代码
2. **资源管理**：智能指针实现安全的 RAII 模式
3. **并发支持**：标准化的多线程库和原子操作
4. **移动语义**：右值引用带来显著的性能提升
5. **泛型增强**：可变参数模板扩展泛型编程能力
6. **Lambda 表达式**：简化函数式编程风格代码

### 7.2 版本演进对比

| 版本 | 发布时间 | 间隔 | 主要特点 |
|------|----------|------|----------|
| C++98 | 1998 | - | 首个 ISO 标准 |
| C++03 | 2003 | 5 年 | 技术修正，小改动 |
| C++11 | 2011 | 8 年 | 首次重大更新，现代 C++ 起点 |
| C++14 | 2014 | 3 年 | C++11 完善，小增量 |
| C++17 | 2017 | 3 年 | 结构化绑定、`std::optional` 等 |
| C++20 | 2020 | 3 年 | 概念（Concept）、范围（Range）等 |

### 7.3 学习建议

1. **优先掌握**：`auto`、智能指针、Lambda 表达式、范围 `for`
2. **深入理解**：右值引用与移动语义、`constexpr`
3. **按需学习**：多线程库、可变参数模板、`decltype`
4. **编译器选择**：确保使用完整支持 C++11 的编译器版本
