# C++26 语言标准

## 1. 概述

C++26 是 C++ 语言的下一代国际标准（International Standard，IS），是继 C++23 之后的最新版本。目前标准制定工作正在进行中，根据 P1000R6 文档，计划于 2024-05-14 发布 IS。

C++26 延续了 C++ 语言每三年一个版本的发布周期，在语言特性和标准库方面都引入了大量改进和新功能，旨在提升开发效率、增强类型安全性、改善并发编程体验。

## 2. 来源与演变

### 制定背景

C++26 的开发遵循 ISO C++ 标准委员会的惯例，由 WG21（C++ 标准工作组）负责制定。标准的演进主要围绕以下几个方向：

| 改进方向 | 说明 |
|---------|------|
| 语言增强 | 简化语法、提升表达能力 |
| 标准库扩展 | 新增实用组件和算法 |
| 并发改进 | 更好的多线程和原子操作支持 |
| 性能优化 | 减少运行时开销 |
| 向后兼容 | 保持与现有代码的兼容性 |

### 标准时间线

| 里程碑 | 时间 |
|-------|------|
| 标准制定启动 | 2023 年 |
| IS 计划发布 | 2024-05-14 |
| 预计正式发布 | 2026 年 |

### 最佳实践

#### 使用无名占位符

```cpp
// C++26 之前：需要命名未使用的变量
auto [value, unused] = get_pair();

// C++26：使用 _ 作为占位符
auto [value, _] = get_pair();
```

#### 删除函数说明原因

```cpp
// C++26 之前
void old_function() = delete;

// C++26：提供删除原因
void old_function() = delete("Use new_function() instead");
```

#### 运行时格式字符串

```cpp
// C++26：支持运行时格式字符串
std::string format_str = get_format_from_user();
std::runtime_format(format_str);  // 安全的运行时格式化
```

#### 饱和算术

```cpp
#include <numeric>

// C++26：饱和算术运算
int result = std::add_sat(a, b);      // 不会溢出
int diff = std::sub_sat(a, b);        // 不会下溢
int prod = std::mul_sat(a, b);        // 乘法饱和
```

### 常见陷阱

#### 编译器兼容性

- C++26 特性仍在积极开发中，不同编译器支持程度不一
- 建议使用 `__has_include` 和 `__cpp_lib_*` 宏检测特性可用性

```cpp
#if __cpp_lib_saturation_arithmetic >= 202311L
    int result = std::add_sat(a, b);
#else
    // 回退实现
    int result = std::min(a + b, INT_MAX);
#endif
```

#### 版本迁移

- 废弃特性在 C++26 中可能被移除
- 需要逐步替换旧代码，如 `<codecvt>` 应改用其他编码转换库

## 6. 代码示例

### 基础用法

#### 结构化绑定增强

```cpp
#include <map>
#include <string>
#include <iostream>

// C++26：结构化绑定作为条件
void process_map() {
    std::map<std::string, int> data{{"a", 1}, {"b", 2}};
    
    // 条件中使用结构化绑定
    if (auto [it, inserted] = data.insert({"c", 3}); inserted) {
        std::cout << "Inserted: " << it->first << "\n";
    }
}

// C++26：结构化绑定引入参数包
template<typename... Args>
void process_tuple(std::tuple<Args...> t) {
    auto [...items] = t;  // 参数包展开
    // 使用 items...
}
```

#### 参数包索引

```cpp
#include <tuple>
#include <iostream>

// C++26：参数包索引
template<typename... Args>
auto get_second(Args... args) {
    return args...[1];  // 直接索引参数包
}

int main() {
    std::cout << get_second(1, 2, 3, 4) << "\n";  // 输出: 2
    return 0;
}
```

#### 饱和算术运算

```cpp
#include <numeric>
#include <iostream>
#include <climits>

void demo_saturation() {
    int a = INT_MAX;
    int b = 1;
    
    // 普通加法会溢出（未定义行为）
    // int overflow = a + b;  // 危险！
    
    // C++26：饱和加法
    int safe_result = std::add_sat(a, b);
    std::cout << "Saturated: " << safe_result << "\n";  // 输出: INT_MAX
    
    // 饱和减法
    int c = INT_MIN;
    int d = 1;
    int sub_result = std::sub_sat(c, d);
    std::cout << "Sub saturated: " << sub_result << "\n";  // 输出: INT_MIN
}
```

### 高级用法

#### 危险指针（Hazard Pointer）

```cpp
#include <hazard_pointer>
#include <memory>

// C++26：危险指针用于无锁数据结构
template<typename T>
class lock_free_stack {
    struct node {
        std::shared_ptr<T> data;
        node* next;
    };
    
    std::atomic<node*> head;
    
public:
    void push(T const& value) {
        auto new_node = new node{std::make_shared<T>(value), head.load()};
        while (!head.compare_exchange_weak(new_node->next, new_node));
    }
    
    std::shared_ptr<T> pop() {
        // 使用危险指针安全访问
        std::hazard_pointer hp = std::make_hazard_pointer();
        node* old_head = head.load();
        
        do {
            hp.protect(old_head);
            if (old_head != head.load()) continue;
            // 安全处理节点
        } while (/* ... */);
        
        hp.reset();
        return old_head->data;
    }
};
```

#### SIMD 数据并行

```cpp
#include <simd>
#include <vector>
#include <iostream>

// C++26：SIMD 并行计算
void demo_simd() {
    std::vector<float> a(1000), b(1000), c(1000);
    
    // 使用 SIMD 类型加速计算
    constexpr size_t width = std::simd_size_v<float>;
    
    for (size_t i = 0; i < a.size(); i += width) {
        std::simd<float> va(&a[i], std::element_aligned);
        std::simd<float> vb(&b[i], std::element_aligned);
        std::simd<float> vc = va + vb;  // 并行加法
        vc.copy_to(&c[i], std::element_aligned);
    }
}
```

#### 线性代数接口

```cpp
#include <linalg>
#include <vector>
#include <iostream>

// C++26：线性代数运算
void demo_linalg() {
    std::vector<double> A = {1, 2, 3, 4};  // 2x2 矩阵
    std::vector<double> x = {1, 1};         // 向量
    std::vector<double> y(2);
    
    // 矩阵-向量乘法
    std::linalg::matrix_vector_product(
        std::mdspan(A.data(), 2, 2),
        std::mdspan(x.data(), 2),
        std::mdspan(y.data(), 2)
    );
    
    // y = A * x
    std::cout << "Result: " << y[0] << ", " << y[1] << "\n";
}
```

#### std::inplace_vector

```cpp
#include <inplace_vector>
#include <iostream>

// C++26：固定容量向量
void demo_inplace_vector() {
    std::inplace_vector<int, 10> vec;  // 最大容量 10
    
    vec.push_back(1);
    vec.push_back(2);
    vec.push_back(3);
    
    // 无堆分配，适合嵌入式系统
    for (auto v : vec) {
        std::cout << v << " ";
    }
    
    // 尝试超过容量会抛出异常
    try {
        for (int i = 0; i < 20; ++i) {
            vec.push_back(i);  // 超过容量会失败
        }
    } catch (const std::bad_alloc&) {
        std::cout << "\nCapacity exceeded!\n";
    }
}
```

### 常见错误及修正

#### 错误：使用已移除的废弃特性

```cpp
// ❌ 错误：C++26 已移除 codecvt
#include <codecvt>  // 编译错误

// ✅ 正确：使用平台特定的编码转换
#include <iconv.h>  // 或其他第三方库
```

#### 错误：未处理饱和算术

```cpp
// ❌ 错误：可能溢出
int add_values(int a, int b) {
    return a + b;  // 未定义行为风险
}

// ✅ 正确：使用饱和算术
#include <numeric>
int add_values(int a, int b) {
    return std::add_sat(a, b);  // C++26
}
```

## 7. 总结

### 核心要点回顾

| 类别 | 关键改进 |
|------|---------|
| 语言特性 | 参数包索引、无名占位符、= delete("reason")、结构化绑定增强 |
| 标准库 | SIMD、线性代数、危险指针、RCU、inplace_vector、饱和算术 |
| 并发编程 | constexpr atomic、原子最大最小值、独立内存回收机制 |
| 安全性 | 禁止临时对象引用绑定、未初始化读取错误行为 |
| 弃用移除 | codecvt、wstring_convert、strstreams、废弃数组比较 |

### 版本对比

| 特性 | C++23 | C++26 |
|------|-------|-------|
| 参数包处理 | 递归展开 | 支持索引访问 |
| 结构化绑定 | 基本支持 | 条件中使用、引入参数包 |
| 格式化 | 编译期 | 运行时格式字符串 |
| 并发 | atomic_ref | hazard_pointer、RCU |
| 数值计算 | 无 | SIMD、线性代数 |

### 学习建议

1. **渐进式采用**：从已广泛支持的特性开始，如无名占位符、参数包索引
2. **关注编译器支持**：使用功能测试宏（Feature Test Macros）检测特性可用性
3. **实践新特性**：尝试 SIMD 和线性代数接口，提升数值计算性能
4. **清理废弃代码**：逐步移除 codecvt、wstring_convert 等已移除的特性
