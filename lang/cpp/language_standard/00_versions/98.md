# C++98 语言标准

## 1. 概述

**C++98**（ISO/IEC 14882:1998）是 C++ 编程语言的第一个国际标准。该标准将 C++ 语言近二十年的演进成果正式标准化，标志着 C++ 从一门实验性语言走向成熟的工业级编程语言。

C++98 在编程语言发展史上具有里程碑意义：
- 结束了 C++ 语言各编译器实现不一致的局面
- 确立了标准模板库（Standard Template Library，STL）作为语言核心组成部分
- 为后续 C++11/14/17/20/23 等版本的演进奠定了坚实基础

## 2. 来源与演变

### 2.1 历史背景

C++ 语言的起源可追溯至 1979 年，Bjarne Stroustrup 在贝尔实验室开始开发"带类的 C"（C with Classes）。在 C++98 标准化之前，各编译器实现主要遵循以下来源：

| 来源 | 说明 |
|------|------|
| Stroustrup 著作 | 《The C++ Programming Language》一书中的语言描述 |
| 编译器扩展 | 各编译器厂商的专有扩展 |
| 实践经验 | 程序员社区的实践经验积累 |

### 2.2 标准化进程

| 时间节点 | 事件 |
|----------|------|
| 1989 年 | ANSI X3J16 委员会启动标准化工作 |
| 1991 年 | ISO 加入，成立联合标准委员会（WG21） |
| 1998 年 9 月 | C++98 标准正式发布 |

### 2.3 C++03 修订

C++03（ISO/IEC 14882:2003）是 C++98 的技术勘误版本，修复了大量缺陷但未引入新特性。两个版本通常被视为同一代标准。

## 3. 核心语言特性

### 3.1 模板（Template）

C++98 引入了完整的模板机制：

| 模板类型 | 说明 |
|----------|------|
| 函数模板（Function Template） | 泛型函数定义 |
| 类模板（Class Template） | 泛型类定义 |
| 模板特化（Template Specialization） | 为特定类型提供专门实现 |
| 偏特化（Partial Specialization） | 类模板的部分特化 |

**模板参数类型：**

| 参数类型 | 语法 | 说明 |
|----------|------|------|
| 类型参数 | `typename T` | 表示任意类型 |
| 非类型参数 | `int N` | 编译期常量值 |
| 模板模板参数 | `template<typename> class C` | 模板作为参数 |

### 3.2 面向对象特性

| 特性 | 说明 |
|------|------|
| 多重继承（Multiple Inheritance） | 类可继承多个基类 |
| 虚基类（Virtual Base Class） | 解决菱形继承问题 |
| 抽象类（Abstract Class） | 包含纯虚函数的类 |
| 运行时类型信息（RTTI） | `typeid` 和 `dynamic_cast` |

**RTTI 使用示例：**

```cpp
class Base { virtual ~Base() = default; };
class Derived : public Base {};

Base* ptr = new Derived;

// typeid 获取类型信息
if (typeid(*ptr) == typeid(Derived)) {
    std::cout << "ptr points to Derived" << std::endl;
}

// dynamic_cast 安全向下转型
if (Derived* d = dynamic_cast<Derived*>(ptr)) {
    // 转换成功，d 不为空
}
```

### 3.3 异常处理（Exception Handling）

C++98 提供了完整的异常处理机制：

| 特性 | 说明 |
|------|------|
| `try`/`catch`/`throw` | 异常抛出与捕获 |
| 异常规范（Exception Specification） | 函数声明允许抛出的异常类型 |
| 标准异常层次 | `std::exception` 及其派生类 |

**异常规范（C++11 弃用，C++17 移除）：**

```cpp
// C++98 异常规范语法（已弃用）
void func() throw(std::runtime_error);  // 仅抛出 runtime_error
void func_nothrow() throw();             // 不抛出异常（等同于 noexcept）
```

### 3.4 命名空间（Namespace）

| 特性 | 说明 |
|------|------|
| 命名空间定义 | `namespace name { ... }` |
| 命名空间别名 | `namespace alias = original;` |
| using 声明 | `using std::vector;` |
| using 指令 | `using namespace std;` |
| 匿名命名空间 | `namespace { ... }`（内部链接） |

### 3.5 其他核心特性

| 特性 | 说明 |
|------|------|
| `new`/`delete` 运算符 | 动态内存管理 |
| `bool` 类型 | 布尔类型及 `true`/`false` 字面量 |
| 新式类型转换 | `static_cast`、`const_cast`、`reinterpret_cast`、`dynamic_cast` |
| 成员函数限定符 | `static`、`const`、`volatile` |
| 引用（Reference） | 变量别名 |
| 内联函数（Inline Function） | 建议编译器内联展开 |
| 默认参数（Default Argument） | 函数参数默认值 |
| 运算符重载（Operator Overloading） | 自定义运算符行为 |
| 函数重载（Function Overloading） | 同名不同参的函数 |
| 友元声明（Friend Declaration） | 授权特定函数/类访问私有成员 |
| `explicit` 关键字 | 禁止隐式构造转换 |

### 4.3 auto_ptr 智能指针

`std::auto_ptr` 是 C++98 提供的智能指针，用于自动内存管理：

**特点与限制：**

| 特性 | 说明 |
|------|------|
| 所有权转移 | 拷贝时转移所有权，非共享 |
| 不可用于容器 | 不能放入标准容器 |
| 已弃用 | C++11 弃用，C++17 移除 |

**使用示例：**

```cpp
#include <memory>

void example() {
    std::auto_ptr<int> p1(new int(10));
    std::auto_ptr<int> p2 = p1;  // 所有权转移，p1 变为空

    if (p1.get() == nullptr) {
        std::cout << "p1 is null after transfer" << std::endl;
    }
}
```

**替代方案：** 推荐使用 C++11 引入的 `std::unique_ptr`。

## 5. 使用场景

### 5.1 适用场景

| 场景 | 说明 |
|------|------|
| 遗留系统维护 | 需要兼容老旧代码库 |
| 嵌入式开发 | 某些嵌入式编译器仅支持 C++98 |
| 教学目的 | 理解 C++ 基础和演进历程 |
| 高可移植性 | 所有编译器均完全支持 |

### 5.2 注意事项

| 问题 | 说明 |
|------|------|
| 缺少现代特性 | 无 `auto`、`nullptr`、Lambda、移动语义等 |
| `auto_ptr` 缺陷 | 所有权转移语义容易导致错误 |
| 异常规范已弃用 | C++11 弃用，C++17 移除 |
| 模板错误信息晦涩 | 缺少概念（Concepts）约束 |

### 5.3 最佳实践

```cpp
// 使用 const 引用避免不必要的拷贝
void process(const std::vector<int>& data);

// 使用 RAII 管理资源
class FileHandle {
public:
    explicit FileHandle(const char* path) : file_(std::fopen(path, "r")) {}
    ~FileHandle() { if (file_) std::fclose(file_); }
    // 禁止拷贝
    FileHandle(const FileHandle&) = delete;
    FileHandle& operator=(const FileHandle&) = delete;
private:
    std::FILE* file_;
};

// 使用标准算法替代手写循环
std::for_each(vec.begin(), vec.end(), &process);
```

## 6. 代码示例

### 6.1 模板基础

```cpp
// 函数模板
template<typename T>
T max(T a, T b) {
    return a > b ? a : b;
}

// 类模板
template<typename T>
class Container {
public:
    void push(const T& value) {
        data_.push_back(value);
    }
    size_t size() const { return data_.size(); }
private:
    std::vector<T> data_;
};

// 模板特化
template<>
class Container<bool> {
    // 针对 bool 的特化实现
};
```

### 6.2 STL 容器使用

```cpp
#include <vector>
#include <map>
#include <algorithm>
#include <iostream>

void stl_example() {
    // vector 示例
    std::vector<int> vec;
    vec.push_back(1);
    vec.push_back(2);
    vec.push_back(3);

    // map 示例
    std::map<std::string, int> scores;
    scores["Alice"] = 95;
    scores["Bob"] = 87;

    // 算法示例
    std::sort(vec.begin(), vec.end());

    // 遍历
    for (std::vector<int>::iterator it = vec.begin();
         it != vec.end(); ++it) {
        std::cout << *it << std::endl;
    }
}
```

### 6.3 异常处理

```cpp
#include <stdexcept>
#include <iostream>

void divide(int a, int b) {
    if (b == 0) {
        throw std::runtime_error("Division by zero");
    }
    std::cout << "Result: " << a / b << std::endl;
}

void exception_example() {
    try {
        divide(10, 0);
    } catch (const std::runtime_error& e) {
        std::cerr << "Error: " << e.what() << std::endl;
    } catch (...) {
        std::cerr << "Unknown error" << std::endl;
    }
}
```

### 6.4 常见错误

```cpp
// 错误：auto_ptr 放入容器
std::vector<std::auto_ptr<int> > vec;  // 危险！编译通过但行为未定义

// 正确：使用原始指针管理或升级到 unique_ptr
std::vector<int*> vec;  // 需要手动管理内存

// 错误：异常规范过于严格
void func() throw(int, double);  // 已弃用

// 正确：不使用异常规范，或使用 noexcept
void func() noexcept;  // C++11 风格
```

## 7. 总结

### 7.1 核心要点

| 要点 | 说明 |
|------|------|
| 首个国际标准 | 确立了 C++ 的正式规范 |
| STL 纳入标准 | 容器、迭代器、算法成为核心组件 |
| 面向对象完整 | 多重继承、RTTI、异常处理完备 |
| 模板机制成熟 | 支持函数模板、类模板、特化 |

### 7.2 版本演进

```
C++98 → C++03（缺陷修复）→ C++11（重大更新）→ C++14 → C++17 → C++20 → C++23
```

### 7.3 编译器支持

| 编译器 | 支持情况 |
|--------|----------|
| GCC | 完全支持（`-std=c++98`） |
| Clang | 完全支持（`-std=c++98`） |
| MSVC | 完全支持 |
| Intel C++ | 完全支持 |
| IBM XL C++ | 完全支持 |

### 7.4 学习建议

1. **理解演进历程**：C++98 是现代 C++ 的基础，理解其特性有助于学习后续版本
2. **避免过时特性**：`auto_ptr`、异常规范等已弃用，应使用现代替代方案
3. **掌握 STL 核心**：容器、迭代器、算法至今仍是 C++ 编程的核心
4. **升级建议**：新项目建议使用 C++17 或更新标准，充分利用现代特性
