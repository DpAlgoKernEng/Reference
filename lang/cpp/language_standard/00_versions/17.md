# C++17

## 1. 概述

C++17 是继 C++14 之后的一个重要版本，引入了大量新的语言特性和标准库功能。该标准于 2017 年 12 月正式发布，被视为 C++ 语言演进中的一个重要里程碑，为现代 C++ 编程带来了显著改进。

C++17 的核心定位是在保持向后兼容性的同时，提供更强大、更简洁、更高效的编程能力。它吸收了多个技术规范（Technical Specification，简称 TS）的成果，将文件系统、可选值、变体等实用特性纳入标准库。

## 2. 来源与演变

### 设计背景

C++17 的开发始于 C++14 发布后，目标是整合多个实验性技术规范的成熟特性，同时解决长期存在的语言缺陷。标准的制定遵循 ISO/IEC 14882:2017 规范。

### 特性来源

C++17 整合了以下技术规范的特性：

| 来源 | 引入特性 |
|------|----------|
| File System TS | 文件系统库（filesystem） |
| Library fundamentals v1 TS | `std::any`、`std::optional`、`std::string_view`、`std::apply`、多态分配器（polymorphic allocator）、搜索器（searcher） |
| Library fundamentals v2 TS | `std::void_t`、逻辑类型特征（`std::conjunction`、`std::disjunction`、`std::negation`）、`std::not_fn`、`std::gcd`、`std::lcm` |
| Parallelism v1 TS | 执行策略（execution policy）、`std::reduce`、`std::inclusive_scan`、`std::exclusive_scan`（移除了 `exception_list`） |
| Mathematical special functions IS | 数学特殊函数 |
| C11 标准 | `std::aligned_alloc`、`std::timespec_get` |

### 版本演进

```
C++11 → C++14 → C++17 → C++20 → C++23
        (小版本)  (大版本)  (大版本)  (小版本)
```

## 3. 语法与参数

### 移除的特性

C++17 移除了以下已废弃的特性：

| 移除特性 | 说明 |
|----------|------|
| `std::auto_ptr` | 被 `std::unique_ptr` 替代 |
| 已废弃的函数对象 | 旧式函数适配器 |
| `std::random_shuffle` | 被 `std::shuffle` 替代 |
| `std::unexpected` | 异常处理相关 |
| 已废弃的 `iostreams` 别名 | 旧式 IO 别名 |
| 三字符组（trigraphs） | 如 `??=` 表示 `#` |
| `register` 关键字 | 存储类说明符 |
| `bool` 自增操作 | `++b` 操作 |
| 动态异常说明 | `throw(type-list)` 语法 |

### 废弃的特性

以下特性在 C++17 中被标记为废弃：

| 废弃特性 | 替代方案 |
|----------|----------|
| `std::iterator` | 直接定义迭代器成员 |
| `std::raw_storage_iterator` | 使用现代内存管理 |
| `std::get_temporary_buffer` | 直接使用分配器 |
| `std::is_literal_type` | 通常不再需要 |
| `std::result_of` | `std::invoke_result` |
| `<codecvt>` 整个头文件 | 使用其他编码转换方案 |

#### Lambda 捕获 *this

## 4. 底层原理

### 类型系统增强

**noexcept 作为类型组成部分**：函数签名现在包含 noexcept 说明，使 noexcept 函数指针只能指向 noexcept 函数。这允许编译器进行更激进的优化，因为 noexcept 函数保证不会抛出异常。

**保证拷贝消除（Guaranteed Copy Elision）**：C++17 强制要求在某些情况下消除临时对象的拷贝。当返回纯右值（prvalue）时，对象直接在目标位置构造，无需中间拷贝。

### 编译期计算扩展

**if constexpr**：在编译期根据类型信息选择代码路径。不同于运行期 if，未选中的分支不会被编译，这对于模板元编程尤为重要。

**constexpr lambda**：lambda 表达式可以在 constexpr 上下文中使用，扩展了编译期计算的能力范围。

### 内存模型改进

**临时对象具化（Temporary Materialization）**：C++17 明确了临时对象的创建时机，仅在需要时才创建临时对象，这优化了不必要的对象构造。

### 最佳实践

```cpp

std::map<std::string, int> config;
for (const auto& [key, value] : config) {
    std::cout << key << ": " << value << "\n";
}

// 2. 使用 std::optional 表示可能失败的操作

    try {
        return std::stoi(s);
    } catch (...) {
        return std::nullopt;
    }
}

// 3. 使用 if 初始化语句限制变量作用域

    // 锁在条件检查后自动释放
}

// 4. 使用 [[nodiscard]] 防止忽略重要返回值

// 5. 使用 std::string_view 避免不必要的字符串拷贝
```

### 注意事项

```cpp
// 结构化绑定注意：绑定的变量是引用
auto [x, y] = pair;       // x, y 是副本
auto& [xr, yr] = pair;    // xr, yr 是引用

// std::string_view 不拥有数据，需注意生命周期
std::string_view get_prefix() {
    std::string s = "hello world";
    return s.substr(0, 5);  // 危险：s 已销毁
}

// std::variant 访问需处理所有类型
std::variant<int, std::string> v;
// 使用 std::visit 或 get_if 安全访问
```

### 常见陷阱

1. **结构化绑定的复制语义**：`auto [a, b] = obj` 实际会复制整个对象，然后绑定到成员
2. **std::string_view 生命周期**：不要返回指向临时字符串的 string_view
3. **std::optional 空状态检查**：使用前必须检查 `has_value()`
4. **并行算法异常**：并行算法中的异常可能导致程序终止

## 6. 代码示例

### 基础用法

```cpp
#include <optional>
#include <variant>
#include <string_view>
#include <filesystem>
#include <tuple>
#include <map>

// std::optional - 可选值
std::optional<int> find_value(int key) {
    if (key > 0) return key * 2;
    return std::nullopt;
}

void use_optional() {
    auto result = find_value(5);
    if (result) {
        std::cout << "Found: " << *result << "\n";
    } else {
        std::cout << "Not found\n";
    }
    
    // 使用 value_or 提供默认值
    int value = find_value(-1).value_or(0);
}

// std::variant - 类型安全联合体
using Value = std::variant<int, double, std::string>;

void use_variant() {
    Value v = 42;
    v = 3.14;
    v = "hello";
    
    // 安全访问
    if (auto* p = std::get_if<int>(&v)) {
        std::cout << "Integer: " << *p << "\n";
    }
    
    // 使用 visit 处理所有类型
    std::visit([](auto&& arg) {
        std::cout << arg << "\n";
    }, v);
}

// std::string_view - 字符串视图
void use_string_view() {
    std::string s = "Hello World";
    std::string_view sv = s;
    
    // 零拷贝操作
    auto first_word = sv.substr(0, 5);  // "Hello"
    auto pos = sv.find("World");
    
    // 接受多种字符串类型
    void process(std::string_view text);
    process("literal");        // 字符串字面量
    process(std::string("tmp")); // std::string
    process(sv);               // string_view
}

// std::filesystem - 文件系统
void use_filesystem() {
    namespace fs = std::filesystem;
    
    // 检查文件存在
    if (fs::exists("config.txt")) {
        // 获取文件大小
        auto size = fs::file_size("config.txt");
        
        // 遍历目录
        for (const auto& entry : fs::directory_iterator(".")) {
            std::cout << entry.path() << "\n";
        }
        
        // 创建目录
        fs::create_directory("output");
        
        // 复制文件
        fs::copy("src.txt", "dst.txt");
    }
}

// 结构化绑定
void use_structured_binding() {
    std::map<int, std::string> m = {{1, "one"}, {2, "two"}};
    
    // 遍历 map
    for (auto& [key, value] : m) {
        value += "_modified";
    }
    
    // 解构 tuple
    std::tuple<int, double, std::string> t{1, 2.0, "three"};
    auto [a, b, c] = t;
    
    // 解构数组
    int arr[] = {1, 2, 3};
    auto [x, y, z] = arr;
}
```

### 高级用法

```cpp
#include <execution>
#include <algorithm>
#include <vector>
#include <numeric>

// 并行算法
void use_parallel_algorithms() {
    std::vector<int> v(1000000);
    
    // 并行排序
    std::sort(std::execution::par, v.begin(), v.end());
    
    // 并行求和
    int sum = std::reduce(std::execution::par, 
                          v.begin(), v.end(), 0);
    
    // 并行转换
    std::transform(std::execution::par,
                   v.begin(), v.end(),
                   v.begin(),
                   [](int x) { return x * 2; });
}

// if constexpr - 编译期分支
template<typename T>
void serialize(T value) {
    if constexpr (std::is_same_v<T, int>) {
        std::cout << "int: " << value;
    } else if constexpr (std::is_same_v<T, std::string>) {
        std::cout << "string: " << value;
    } else if constexpr (std::is_arithmetic_v<T>) {
        std::cout << "numeric: " << value;
    } else {
        // 其他类型的处理
        std::cout << "other type";
    }
}

// 折叠表达式
template<typename... Args>
auto sum_all(Args... args) {
    return (... + args);  // 左折叠：(arg1 + (arg2 + (arg3 + ...)))
}

template<typename... Args>
auto product_all(Args... args) {
    return (... * args);  // 右折叠：(((... * arg1) * arg2) * arg3)
}

template<typename... Args>
void print_all(Args... args) {
    (std::cout << ... << args) << "\n";  // 左折叠输出
}

// 多态分配器
#include <memory_resource>

void use_polymorphic_allocator() {
    // 使用预分配内存池
    std::array<std::byte, 1024> buffer;
    std::pmr::monotonic_buffer_resource pool{buffer.data(), buffer.size()};
    
    std::pmr::vector<int> v{&pool};
    v.push_back(1);
    v.push_back(2);
    
    // 内存池分配，性能更高
}

// std::scoped_lock - 多锁管理
#include <mutex>

void use_scoped_lock() {
    std::mutex mtx1, mtx2;
    
    // 同时锁定多个互斥量，避免死锁
    {
        std::scoped_lock lock(mtx1, mtx2);
        // 安全操作
    }  // 自动解锁两个互斥量
}

// std::apply - tuple 参数展开
void use_apply() {
    auto add = [](int a, int b) { return a + b; };
    std::tuple<int, int> args{1, 2};
    
    int result = std::apply(add, args);  // result = 3
    
    // 用于成员函数
    struct S {
        void foo(int a, const std::string& b) {}
    };
    S s;
    std::apply(&S::foo, std::tuple_cat(std::make_tuple(&s), 
                                       std::make_tuple(1, "hello")));
}
```

### 常见错误及修正

```cpp
// 错误 1：std::string_view 返回临时数据
// ❌ 错误
std::string_view get_name() {
    std::string name = "temporary";
    return name;  // name 已销毁，返回悬空视图
}

// ✅ 正确
std::string get_name() {
    return "temporary";  // 返回拥有数据的字符串
}

// 错误 2：std::optional 空值访问
// ❌ 错误
std::optional<int> opt;
int value = opt.value();  // 未检查，可能抛出异常

// ✅ 正确
std::optional<int> opt;
if (opt.has_value()) {
    int value = opt.value();
}
// 或使用 value_or
int value = opt.value_or(0);

// 错误 3：std::variant 类型不匹配
// ❌ 错误
std::variant<int, std::string> v = 42;
std::string s = std::get<std::string>(v);  // 抛出异常

// ✅ 正确
std::variant<int, std::string> v = 42;
if (auto* p = std::get_if<int>(&v)) {
    std::cout << "int: " << *p;
}

// 错误 4：if 初始化语句作用域误解
// ❌ 错误理解
if (auto lock = std::lock_guard{mtx}; true) {
    // lock 在此块内有效
}
lock.~lock_guard();  // 错误：lock 已销毁

// ✅ 正确理解：lock 在 if 语句结束时销毁
if (auto lock = std::lock_guard{mtx}; condition()) {
    // lock 在此块内有效
} else {
    // lock 也在此块内有效
}  // lock 在此处销毁

// 错误 5：结构化绑定的复制语义误解
// ❌ 错误理解
struct Point { int x, y; };
Point p{1, 2};
auto [x, y] = p;  // x, y 是 p.x, p.y 的副本
x = 10;  // 不影响 p.x

// ✅ 正确：需要修改原对象时使用引用
auto& [xr, yr] = p;
xr = 10;  // 修改 p.x
```

## 7. 总结

### 核心要点回顾

C++17 是一个重要的语言版本，主要贡献包括：

1. **实用性增强**：`std::optional`、`std::variant`、`std::string_view`、`std::filesystem` 等实用特性
2. **代码简化**：结构化绑定、if 初始化、内联变量简化了常见模式
3. **泛型编程改进**：if constexpr、折叠表达式、类模板参数推导增强了模板能力
4. **性能优化**：保证拷贝消除、并行算法提升了运行效率
5. **安全性提升**：`[[nodiscard]]`、`std::scoped_lock` 等减少了常见错误

### 相关技术对比

| 特性 | C++14 | C++17 | C++20 |
|------|-------|-------|-------|
| 泛型 lambda | 支持 | 支持 | 支持（模板 lambda） |
| constexpr | 函数 | 函数 + lambda + if | 全面 constexpr |
| 结构化绑定 | 无 | 支持 | 支持 |
| 可选值 | 无 | `std::optional` | 支持 |
| 文件系统 | 无 | `std::filesystem` | 支持 |
| 概念（concept） | 无 | 无 | 支持 |
| 范围库（ranges） | 无 | 无 | 支持 |

### 学习建议

1. **优先掌握**：结构化绑定、`std::optional`、`std::string_view` - 这些是最常用的特性
2. **进阶学习**：if constexpr、折叠表达式 - 提升模板编程能力
3. **深入理解**：保证拷贝消除、noexcept 类型系统 - 优化和底层知识
4. **实践应用**：并行算法、文件系统库 - 实际项目应用
