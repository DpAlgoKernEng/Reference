# C++17 编译器支持

## 1. 概述

C++17（ISO/IEC 14882:2017）是 C++ 语言的重大版本更新，于 2017 年正式发布。本文档详细记录了主流编译器对 C++17 核心语言特性和标准库特性的支持情况，帮助开发者评估平台兼容性和选择合适的编译器版本。

C++17 引入了大量实用特性，包括结构化绑定（Structured Binding）、`std::optional`、`std::variant`、`std::string_view`、文件系统库（Filesystem Library）、并行算法（Parallel Algorithms）等，显著提升了开发效率和代码可读性。

## 2. 来源与演变

### 发布历史

- **2017 年**：C++17 标准正式发布（ISO/IEC 14882:2017）
- **2014 年**：C++14 发布，为 C++17 奠定基础
- **2020 年**：C++20 发布，延续 C++17 的演进路线

### 设计动机

C++17 的主要设计目标：

- 简化常见编程模式，减少样板代码
- 增强类型安全和表达力
- 提升标准库的实用性和性能
- 移除过时特性，保持语言一致性

### 缺陷报告（DR）说明

文档中出现的 `DRnn` 表示缺陷报告（Defect Report），数字 `nn` 表示该修复应用的目标 C++ 版本。例如 `DR11` 表示应用于 C++11 标准的缺陷修复，`DR20` 表示应用于 C++20 的修复。

### 4.3 Lambda 表达式增强

| 特性 | 提案文档 | GCC | Clang | MSVC | Apple Clang |

**Lambda 捕获 *this 示例**：

```cpp
class Processor {
    int value_ = 42;

public:
    auto get_callback() {
        // C++14：需要捕获 this，可能导致悬空指针
        // return [this]() { return value_; };

        // C++17：捕获 *this 的副本，避免悬空指针
        return [*this]() {
            return value_;  // 使用副本，安全
        };
    }
};

// constexpr lambda
constexpr auto square = [](int n) constexpr {
    return n * n;
};

static_assert(square(5) == 25);  // 编译期计算
```

```cpp
    std::lock_guard<std::mutex> lg1(mtx1, std::adopt_lock);
    std::lock_guard<std::mutex> lg2(mtx2, std::adopt_lock);
}
```

**多态内存资源示例**：

```cpp
#include <memory_resource>
#include <vector>
#include <array>

void example() {
    // 栈缓冲区
    std::array<std::byte, 1024> buffer;

    // 栈分配内存池
    std::pmr::monotonic_buffer_resource pool{buffer.data(), buffer.size()};

    // 使用自定义分配器的 vector
    std::pmr::vector<int> vec{&pool};
    vec.push_back(1);
    vec.push_back(2);
    vec.push_back(3);

    // 使用统一内存资源
    std::pmr::unsynchronized_pool_resource unsync_pool;

}
```

## 7. 总结

### 7.1 核心要点回顾

- C++17 是重要的语言更新，引入了大量实用特性
- 主流编译器已全面支持：GCC 7+、Clang 4+、MSVC 19.11+
- 关键特性包括：结构化绑定、`if constexpr`、`std::optional`/`std::variant`、文件系统库、并行算法
- 特性测试宏可帮助检测编译器支持

### 7.2 版本迁移建议

| 场景 | 建议操作 |
|------|----------|
| 新项目 | 直接使用 C++17 标准 |
| C++14 项目 | 逐步迁移，优先采用新特性简化代码 |
| C++11 项目 | 考虑跳过 C++14，直接升级到 C++17 |
| 跨平台项目 | 确保 GCC 7+、Clang 4+、MSVC 2017+ |
