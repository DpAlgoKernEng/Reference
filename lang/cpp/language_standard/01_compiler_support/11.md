# C++11 编译器支持

## 1. 概述

C++11（原名 C++0x）是 C++ 语言的第二个主要版本，于 2011 年正式发布。本文档汇总了主流编译器对 C++11 核心语言特性和库特性的支持情况，帮助开发者了解各编译器的兼容性。

C++11 带来了大量重要的语言改进，包括：

- **类型推导**：`auto` 关键字、`decltype` 说明符
- **移动语义**：右值引用（rvalue reference）、移动构造/赋值
- **智能指针**：`std::unique_ptr`、`std::shared_ptr`、`std::weak_ptr`
- **并发支持**：线程、互斥量、条件变量、原子操作
- **模板增强**：可变参数模板（variadic template）、模板别名
- **其他特性**：lambda 表达式、范围 for 循环、强类型枚举等

## 2. 来源与演变

### 标准背景

C++11 标准由 ISO/IEC 于 2011 年 9 月发布，正式名称为 ISO/IEC 14882:2011。该标准是 C++03 的重大升级，引入了约 80 项新特性，显著增强了语言的表达能力和性能。

### 特性提案

文档中提到的 `N****` 编号（如 N1653、N1720 等）为 C++ 标准委员会的提案编号（Paper），每个特性对应一个或多个提案。`DRnn` 表示缺陷报告（Defect Report），`nn` 表示应用的目标标准版本。

### 版本里程碑

| 时间节点 | 事件 |
|----------|------|
| 2003 年 | C++03 发布（C++11 前一个版本） |
| 2006 年 | C++0x 提案开始形成 |
| 2011 年 | C++11 正式发布 |
| 2014 年 | C++14 发布（C++11 的小幅增强） |
| 2017 年 | C++17 发布 |

## 3. 核心语言特性支持

### 表格说明

- **Yes**：完整支持
- **版本号**（如 4.3）：从该版本开始支持
- **partial**：部分支持
- **FTM**：特性测试宏（Feature Test Macro）
- **星号 \***：有额外说明

### 特性测试宏

C++11 引入的特性测试宏（Feature Test Macro）可用于检测编译器对特定特性的支持：

```cpp
// 示例：检测编译器特性
#if __cpp_constexpr >= 200704L
    // constexpr 可用
#endif

#if __cpp_lambdas >= 200907L
    // lambda 可用
#endif

#if __cpp_range_based_for >= 200907L
    // 范围 for 可用
#endif

#if __cpp_nullptr >= 200707L
    // nullptr 可用
#endif

#if __cpp_rvalue_references >= 200610L
    // 右值引用可用
#endif
```

## 6. 使用场景

### 选择编译器版本

根据项目需求选择合适的编译器版本：

1. **新项目**：建议使用 GCC 5.0+、Clang 3.5+、MSVC VS 2017+
2. **跨平台项目**：以 MSVC 支持为基准（通常是限制因素）
3. **嵌入式开发**：确认目标平台的编译器支持情况

### 特性使用建议

```cpp
// 使用 static_assert 进行编译期检查
static_assert(sizeof(int) == 4, "int must be 4 bytes");

// 使用 auto 简化类型声明
auto iter = container.begin();  // 自动推导迭代器类型

// 使用 nullptr 代替 NULL
int* ptr = nullptr;  // 类型安全的空指针

// 使用范围 for 循环
for (const auto& item : container) {
    process(item);
}

// 使用强类型枚举
enum class Color { Red, Green, Blue };
Color c = Color::Red;  // 必须使用作用域

// 使用 constexpr 进行编译期计算
constexpr int factorial(int n) {
    return n <= 1 ? 1 : n * factorial(n - 1);
}
constexpr int fact_5 = factorial(5);  // 编译期计算

// 使用 lambda 表达式
auto multiply = [](int a, int b) { return a * b; };
int result = multiply(3, 4);  // result = 12

// 使用右值引用和移动语义
std::vector<int> createVector() {
    std::vector<int> result{1, 2, 3, 4, 5};
    return result;  // 移动语义，避免拷贝
}

// 使用初始化列表
std::vector<int> numbers = {1, 2, 3, 4, 5};
std::map<std::string, int> scores = {{"Alice", 90}, {"Bob", 85}};

// 使用 override 和 final
class Derived : public Base {
public:
    void foo() override;  // 确保正确覆盖基类虚函数
    virtual void bar() final;  // 禁止派生类覆盖
};
```

### 并发编程示例

```cpp
#include <thread>
#include <mutex>
#include <atomic>
#include <condition_variable>
#include <future>

// 使用 std::thread
void threadFunction() {
    std::cout << "Hello from thread\n";
}
std::thread t(threadFunction);
t.join();

// 使用 std::mutex 保护共享数据
std::mutex mtx;
int counter = 0;
void increment() {
    std::lock_guard<std::mutex> lock(mtx);
    ++counter;
}

// 使用 std::atomic 进行无锁编程
std::atomic<int> atomicCounter{0};
atomicCounter.fetch_add(1);  // 原子操作

// 使用 std::condition_variable 进行线程同步
std::condition_variable cv;
bool ready = false;
void waitForReady() {
    std::unique_lock<std::mutex> lock(mtx);
    cv.wait(lock, []{ return ready; });
    // 继续执行
}

// 使用 std::future 和 std::async
auto future = std::async(std::launch::async, []{
    return 42;
});
int result = future.get();  // 获取异步结果
```

## 7. 总结

### 核心要点

1. **GCC 和 Clang 对 C++11 支持最为完整**，GCC 4.8+、Clang 3.3+ 已基本支持所有核心特性
2. **MSVC 在 VS 2015 (19.0) 后提供完整支持**，早期版本存在部分限制
3. **大多数现代编译器已完全支持 C++11**，可放心使用

### 推荐配置

| 编译器 | 推荐版本 | 编译选项 |
|--------|----------|----------|
| GCC | 5.0+ | `-std=c++11` 或 `-std=gnu++11` |
| Clang | 3.5+ | `-std=c++11` |
| MSVC | VS 2017+ | `/std:c++11` 或 `/std:c++14` |

### 迁移建议

对于使用 C++03 的项目，建议按以下步骤迁移到 C++11：

1. **升级编译器**：确保编译器版本支持所需特性
2. **逐步采用新特性**：从 `auto`、`nullptr`、范围 for 等基础特性开始
3. **启用智能指针**：使用 `std::unique_ptr` 和 `std::shared_ptr` 替代裸指针
4. **使用移动语义**：实现移动构造函数和移动赋值运算符以提升性能
5. **采用 lambda 表达式**：简化算法调用和回调函数
