# C++98 编译器支持

## 1. 概述

C++98（ISO/IEC 14882:1998）是 C++ 语言的第一个国际标准，为现代 C++ 编译器奠定了基础。所有主流 C++ 编译器都将 C++98 支持作为基准特性完整实现。

C++98 标准确立了 C++ 语言的核心特性集，包括模板（`template`）、多重继承、虚基类、运行时类型信息（RTTI）、异常处理、命名空间等关键特性，以及完整的 STL（标准模板库）。

## 2. 来源与演变

### 标准发布

C++98 于 1998 年由国际标准化组织（ISO）和国际电工委员会（IEC）联合发布，标准号为 ISO/IEC 14882:1998。这是 C++ 语言首次拥有正式的国际标准规范。

### 设计动机

C++98 标准化的主要目标是：

- 统一各编译器厂商的 C++ 实现差异
- 定义标准库接口，特别是 STL（Standard Template Library）
- 提供可移植的 C++ 程序编写规范

### 版本变更

| 时期 | 变更 |
|------|------|
| 1998 年 | C++98 正式发布 |
| 2003 年 | C++03 发布（技术勘误版，无新特性） |
| 2011 年 | C++11 发布，`auto_ptr` 被废弃 |
| 2017 年 | C++17 发布，`auto_ptr` 被移除 |

### export 关键字

`export` 关键字是 C++98 中唯一未被广泛实现的特性，仅 EDG（Edison Design Group）前端支持。该特性在 C++11 中被废弃并移除。

## 3. 语法与参数

### 编译器标志

| 编译器 | C++98 模式标志 |
|--------|----------------|
| GCC | `-std=c++98` 或 `-std=gnu++98` |
| Clang | `-std=c++98` |
| MSVC | 默认（旧版本无需标志） |
| Intel C++ | `-std=c++98` |

### 编译命令示例

```bash
# GCC/Clang 编译 C++98 程序
g++ -std=c++98 -Wall -o program program.cpp

# 使用 GNU 扩展的 C++98
g++ -std=gnu++98 -o program program.cpp

# Intel 编译器
icpc -std=c++98 -o program program.cpp
```

## 5. 使用场景

### 适用场景

C++98 适用于以下场景：

- 需要最大兼容性的遗留代码维护
- 嵌入式系统开发（编译器更新缓慢）
- 跨平台库开发（确保最广泛兼容性）
- 教学目的（学习 C++ 基础特性）

### 已知实现问题

#### 1. export 模板

`export` 关键字从未被广泛实现，仅 EDG 前端支持。该特性在 C++11 中被移除。

```cpp
// C++98 export 语法（未广泛支持）
export template<typename T>
T add(T a, T b);
```

#### 2. 模板实例化

部分旧编译器在模板实例化规则上存在细微差异。

#### 3. 两阶段名称查找

部分编译器在模板中实现了非标准的名称查找行为。

#### 4. auto_ptr 问题

`auto_ptr` 存在以下已知问题：

```cpp
// 问题：auto_ptr 所有权转移语义
std::auto_ptr<int> p1(new int(42));
std::auto_ptr<int> p2 = p1;  // p1 变为空！

// 问题：不能用于容器
std::vector<std::auto_ptr<int>> vec;  // 未定义行为
```

### 兼容性说明

- C++98 是所有 C++ 编译器的基准特性集
- 现代编译器可能对后续标准中废弃的 C++98 构造发出警告
- `auto_ptr` 在 C++11 中被废弃，C++17 中被移除
- 建议新代码使用 `std::unique_ptr` 替代 `auto_ptr`

## 6. 代码示例

### 基础用法

```cpp
// C++98 基本程序结构
#include <iostream>
#include <vector>
#include <string>
#include <algorithm>

int main() {
    // 使用 vector 容器
    std::vector<int> numbers;
    numbers.push_back(1);
    numbers.push_back(2);
    numbers.push_back(3);

    // 使用算法
    std::sort(numbers.begin(), numbers.end());

    // 使用 string
    std::string message = "Hello, C++98";
    std::cout << message << std::endl;

    // 使用迭代器
    for (std::vector<int>::iterator it = numbers.begin();
         it != numbers.end(); ++it) {
        std::cout << *it << " ";
    }
    std::cout << std::endl;

    return 0;
}
```

### 模板编程

```cpp
// C++98 模板类
template<typename T>
class Container {
public:
    void add(const T& value) {
        data_.push_back(value);
    }

    size_t size() const {
        return data_.size();
    }

private:
    std::vector<T> data_;
};

// 函数模板
template<typename T>
T maximum(T a, T b) {
    return a > b ? a : b;
}

// 模板偏特化
template<typename T>
class Container<T*> {
    // 指针类型的特殊处理
};
```

### 异常处理

```cpp
#include <stdexcept>
#include <iostream>

void validate(int value) {
    if (value < 0) {
        throw std::invalid_argument("Value cannot be negative");
    }
    if (value > 100) {
        throw std::out_of_range("Value exceeds maximum");
    }
}

int main() {
    try {
        validate(-1);
    }
    catch (const std::invalid_argument& e) {
        std::cerr << "Invalid argument: " << e.what() << std::endl;
    }
    catch (const std::out_of_range& e) {
        std::cerr << "Out of range: " << e.what() << std::endl;
    }
    catch (const std::exception& e) {
        std::cerr << "Exception: " << e.what() << std::endl;
    }
    return 0;
}
```

### RTTI 使用

```cpp
#include <iostream>
#include <typeinfo>

class Base {
public:
    virtual ~Base() {}
};

class Derived : public Base {};

int main() {
    Base* ptr = new Derived;

    // 使用 typeid
    std::cout << "Type: " << typeid(*ptr).name() << std::endl;

    // 使用 dynamic_cast
    Derived* derived = dynamic_cast<Derived*>(ptr);
    if (derived) {
        std::cout << "Cast successful" << std::endl;
    }

    delete ptr;
    return 0;
}
```

### 常见错误及修正

```cpp
// 错误：auto_ptr 存入容器
std::vector<std::auto_ptr<int>> bad_vec;  // 危险！

// 正确：使用原始指针或智能指针（C++11 后）
std::vector<int*> good_vec;

// 错误：模板定义位置不当
// 模板定义必须放在头文件中，或使用显式实例化

// 正确：模板定义在头文件
// file: container.h
template<typename T>
class MyContainer {
    // 完整定义
};
```

## 7. 总结

### 核心要点

- C++98 是所有现代 C++ 编译器支持的基准标准
- 核心语言特性（模板、异常、RTTI 等）已得到完整广泛支持
- STL 容器和算法在各编译器实现中表现一致
- `export` 关键字是唯一未被广泛实现的特性

### 版本演进

| 版本 | 状态 | 说明 |
|------|------|------|
| C++98 | 完全支持 | 基准标准 |
| C++03 | 完全支持 | 技术勘误，无新特性 |
| C++11 | 部分废弃 | `auto_ptr` 被废弃 |
| C++17 | 移除特性 | `auto_ptr` 被移除 |

### 相关标准对比

| 特性 | C++98 | C++11 | C++17 |
|------|-------|-------|-------|
| `auto_ptr` | 支持 | 废弃 | 移除 |
| `auto` 类型推导 | 无 | 支持 | 支持 |
| `nullptr` | 无 | 支持 | 支持 |
| 智能指针 | 无 | `unique_ptr`/`shared_ptr` | 增强 |
| Lambda | 无 | 支持 | 支持 |

### 学习建议

1. **理解基础特性**：掌握模板、异常处理、RTTI 等核心概念
2. **避免废弃特性**：新代码应避免使用 `auto_ptr`
3. **关注兼容性**：如需最大兼容性，可使用 `-std=c++98` 标志
4. **逐步迁移**：从 C++98 迁移到 C++11 及更高版本可显著提升代码质量
