# C++23 编译器支持情况

## 1. 概述

C++23 是 C++ 语言的最新标准版本，于 2023 年正式发布。本文档汇总了各主流编译器对 C++23 核心语言特性和标准库特性的支持情况，帮助开发者了解不同编译器版本的兼容性。

### 编译器版本说明

| 编译器 | 说明 |
|--------|------|
| GCC | GNU Compiler Collection |
| Clang | LLVM 编译器前端 |
| MSVC | Microsoft Visual C++ |
| Apple Clang | Apple 修改版 Clang |
| EDG eccp | Edison Design Group 编译器 |
| Intel C++ | Intel C++ 编译器 |
| Nvidia HPC C++ | Nvidia 高性能计算编译器（原 PGI） |
| Nvidia nvcc | Nvidia CUDA 编译器 |
| Cray | Cray 编译器 |

### 符号说明

- `*`：鼠标悬停在标有星号的单元格上可查看额外说明
- `DRnn`：缺陷报告（Defect Report），`nn` 表示应用的 C++ 版本，例如 DR20 表示应用于 C++20
- `FTM`：特性测试宏（Feature Test Macro）可用

## 2. 核心语言特性

### 2.1 新增语法特性

#### size_t 字面量后缀

| 特性 | 论文 | GCC | Clang | MSVC | Apple Clang |
|------|------|-----|-------|------|-------------|
| `size_t` 字面量后缀 | P0330R8 | 11 | 13 | 19.43* | 13.1.6* |

允许使用 `z` 或 `Z` 后缀创建 `size_t` 类型的字面量：

```cpp
auto size = 42z;    // 类型为 size_t
auto ssize = -42z;  // 类型为 signed size_t (ssize_t)
```

#### Lambda 表达式简化

| 特性 | 论文 | GCC | Clang | MSVC | Apple Clang |
|------|------|-----|-------|------|-------------|
| 使 `()` 对 lambda 更可选 | P1102R2 | 11 | 13 | - | 13.1.6* |

对于无捕获的 lambda，可以省略空的参数列表：

```cpp
auto f = []{ return 42; };  // C++23: 可省略空 ()
auto g = []() { return 42; }; // 传统写法仍然有效
```

#### if consteval

| 特性 | 论文 | GCC | Clang | MSVC | Apple Clang |
|------|------|-----|-------|------|-------------|
| `if consteval` | P1938R3 | 12 | 14 | - | 14.0.0* |

用于检测是否在编译期求值上下文中执行：

```cpp
constexpr int compute(int n) {
    if consteval {
        // 编译期执行路径
        return n * 2;
    } else {
        // 运行期执行路径
        return n + 1;
    }
}
```

#### 显式对象成员函数（推导 this）

| 特性 | 论文 | GCC | Clang | MSVC | Apple Clang |
|------|------|-----|-------|------|-------------|
| 显式对象成员函数 | P0847R7 | 14 | 18/19* | 19.32*(partial)* | - |

允许显式声明 `this` 参数，简化 CRTP 模式和值类别处理：

```cpp
struct Widget {
    // 显式对象参数
    void foo(this Widget& self, int x) {
        self.value = x;
    }
    
    // 支持值类别推导
    void bar(this auto&& self) {
        // self 可以是左值引用或右值引用
    }
};
```

#### 多维下标运算符

| 特性 | 论文 | GCC | Clang | MSVC | Apple Clang |
|------|------|-----|-------|------|-------------|
| 多维下标运算符 | P2128R6 | 12 | 15 | - | 14.0.3* |

允许下标运算符接受多个参数：

```cpp
class Matrix {
    double data[3][3];
public:
    double& operator[](size_t i, size_t j) {
        return data[i][j];
    }
};

Matrix m;
m[1, 2] = 3.14;  // C++23: 多维下标访问
```

#### auto(x) 和 auto{x}

| 特性 | 论文 | GCC | Clang | MSVC | Apple Clang |
|------|------|-----|-------|------|-------------|
| `auto(x)` 和 `auto{x}` | P0849R8 | 12 | 15 | - | 14.0.3* |

允许从表达式中推导类型并创建副本：

```cpp
auto a = auto(42);     // int
auto b = auto{x};      // 创建 x 的副本
```

#### <expected>

| 特性 | 论文 | GCC | Clang | MSVC | Apple Clang |
|------|------|-----|-------|------|-------------|
| `<expected>` | P0323R12, P2549R1 | 12 | 16 | 19.33* | 15.0.0* |

用于错误处理的预期类型：

```cpp
#include <expected>

std::expected<int, std::string> divide(int a, int b) {
    if (b == 0) {
        return std::unexpected("Division by zero");
    }
    return a / b;
}

auto result = divide(10, 2);
if (result) {
    std::cout << result.value() << std::endl;  // 5
}
```


### 6.3 可移植性注意事项

1. **特性检测**：使用特性测试宏而非编译器版本判断
   ```cpp
   #ifdef __cpp_lib_expected
       // 使用 std::expected
   #else
       // 回退方案
   #endif
   ```

2. **部分支持**：部分特性可能仅有部分实现
   - MSVC 对 `mdspan` 和显式对象参数支持有限
   - Clang 对某些 constexpr 特性支持不完整

3. **平台差异**：
   - Apple Clang 版本号与 LLVM Clang 不同
   - Nvidia 编译器支持滞后

## 7. 总结

### 7.1 核心要点

1. **C++23** 提供了丰富的语言和库特性增强
2. **主流编译器**支持程度不一，GCC 和 Clang 领先
3. **特性测试宏**是检测支持情况的标准方法

### 7.2 亮点特性

| 类别 | 特性 | 影响 |
|------|------|------|
| 语言 | 显式对象参数 | 简化 CRTP，值类别处理 |
| 语言 | 多维下标运算符 | 直观的多维数组访问 |
| 库 | `<expected>` | 类型安全的错误处理 |
| 库 | `<print>` | 类型安全的格式化输出 |
| 库 | `std::mdspan` | 多维数组视图 |
| 库 | Range 视图增强 | 更强大的函数式编程 |

### 7.3 学习建议

1. **优先掌握**：`if consteval`、`std::expected`、Range 视图
2. **逐步迁移**：使用特性测试宏保持向后兼容
3. **关注更新**：编译器支持持续改进
