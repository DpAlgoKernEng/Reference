# C++14 编译器支持

## 1. 概述

C++14 是 C++11 的次要版本更新，于 2014 年 8 月正式发布。与 C++11 相比，C++14 主要包含语言特性的完善和标准库的扩展，而非大规模的语法革新。本文档记录各主流编译器对 C++14 核心语言特性和库特性的支持情况。

**表格说明**：
- `*` 表示鼠标悬停可查看额外说明
- `DRnn` 表示缺陷报告（Defect Report），`nn` 表示目标 C++ 版本

## 2. 来源与演变

### 历史背景

C++14 的开发始于 C++11 发布后不久，目标是在 C++11 的基础上进行小规模改进和缺陷修复。C++14 的设计理念是"小而精"，专注于提升开发体验和代码可读性。

### 主要改进方向

| 方向 | 说明 |
|------|------|
| 语言简化 | 简化泛型编程和函数返回类型推导 |
| 表达力增强 | 二进制字面量、数字分隔符等 |
| 库功能扩展 | `std::make_unique`、`std::integer_sequence` 等 |
| 编译期计算 | 扩展 `constexpr` 功能 |

### 版本标识

- **标准版本**：ISO/IEC 14882:2014
- **发布日期**：2014 年 8 月 18 日
- **前置版本**：C++11（ISO/IEC 14882:2011）
- **后续版本**：C++17（ISO/IEC 14882:2017）

### 特性详解

#### 二进制字面量

使用 `0b` 或 `0B` 前缀表示二进制数：

```cpp
int binary = 0b1010;      // 等于 10
int mask = 0b11110000;    // 位掩码
```

#### `decltype(auto)` 与返回类型推导

允许函数返回类型自动推导：

```cpp
// C++11 需要尾置返回类型
auto f1() -> decltype(expr) { return expr; }

// C++14 简化写法
decltype(auto) f2() { return expr; }

// 普通函数返回类型推导
auto add(int a, int b) { return a + b; }
```

#### 泛型 Lambda 捕获

支持在 Lambda 捕获列表中初始化变量：

```cpp
// 初始化捕获
auto ptr = std::make_unique<int>(42);
auto f = [p = std::move(ptr)]() { return *p; };

// 移动捕获
std::vector<int> v = {1, 2, 3};
auto g = [vec = std::move(v)]() { return vec.size(); };
```

#### 泛型 Lambda 表达式

Lambda 参数支持 `auto`：

```cpp
// C++11 需要显式类型
auto f1 = [](int x) { return x * 2; };

// C++14 泛型 Lambda
auto f2 = [](auto x) { return x * 2; };
auto f3 = [](auto a, auto b) { return a + b; };
```

#### 变量模板

定义参数化的变量：

```cpp
template<typename T>
constexpr T pi = T(3.14159265358979323846);

// 使用
float f = pi<float>;
double d = pi<double>;
long double ld = pi<long double>;
```

#### 扩展 `constexpr`

放宽 `constexpr` 函数限制：

```cpp
// C++14 允许局部变量、循环、条件语句
constexpr int factorial(int n) {
    if (n <= 1) return 1;
    int result = 1;
    for (int i = 2; i <= n; ++i) {
        result *= i;
    }
    return result;
}
```

#### 聚合类型默认成员初始化器

允许在聚合类型中使用默认初始化器：

```cpp
struct Point {
    int x = 0;
    int y = 0;
};

Point p{1, 2};      // x=1, y=2
Point q{};          // x=0, y=0
```

#### `[[deprecated]]` 属性

标记废弃的实体：

```cpp
[[deprecated("Use newFunction() instead")]]
void oldFunction() {}

[[deprecated]]
int legacyValue = 42;
```

#### 单引号数字分隔符

提高数字可读性：

```cpp
int million = 1'000'000;
int hex = 0xFF'FF'FF;
int binary = 0b1010'1010'1010'1010;
double pi = 3.141'592'653'589'793;
```

### 核心库特性详解

#### `std::make_unique`

安全的智能指针创建函数：

```cpp
#include <memory>

// C++11 风格（容易出错）
std::unique_ptr<int> p1(new int(42));

// C++14 安全方式
auto p2 = std::make_unique<int>(42);
auto p3 = std::make_unique<std::string>("Hello");
auto p4 = std::make_unique<std::vector<int>>(10, 0);
```

#### `std::integer_sequence`

编译期整数序列，用于参数包展开：

```cpp
#include <utility>
#include <tuple>
#include <iostream>

// 打印元组所有元素
template<typename Tuple, std::size_t... Is>
void print_tuple_impl(const Tuple& t, std::index_sequence<Is...>) {
    ((std::cout << (Is == 0 ? "" : ", ") << std::get<Is>(t)), ...);
}

template<typename... Args>
void print_tuple(const std::tuple<Args...>& t) {
    print_tuple_impl(t, std::index_sequence_for<Args...>{});
}
```

#### `std::quoted`

字符串引号包装，用于 IO 流：

```cpp
#include <iomanip>
#include <sstream>
#include <string>

std::stringstream ss;
std::string original = "Hello, World!";

// 写入带引号
ss << std::quoted(original);

// 读取自动去引号
std::string extracted;
ss >> std::quoted(extracted);
```

#### 异构关联查找

允许使用不同类型作为查找键：

```cpp
#include <set>
#include <string>

// 设置透明比较器
std::set<std::string> s = {"apple", "banana", "cherry"};

// C++14 前需要构造临时 string
// s.find(std::string("apple"));

// C++14 可直接使用字符串字面量
// 需要启用异构查找（自定义比较器）
struct StringComparator {
    using is_transparent = void;
    bool operator()(const std::string& lhs, const std::string& rhs) const {
        return lhs < rhs;
    }
};
std::set<std::string, StringComparator> s2;
```

#### `std::exchange`

原子交换并返回旧值：

```cpp
#include <utility>

int x = 10;
int y = std::exchange(x, 20);  // x=20, y=10

// 常用于移动语义
class Resource {
    std::unique_ptr<Data> data_;
public:
    std::unique_ptr<Data> release() {
        return std::exchange(data_, nullptr);
    }
};
```

#### `std::get<T>()`

按类型访问元组元素：

```cpp
#include <tuple>
#include <string>

std::tuple<int, std::string, double> t(42, "hello", 3.14);

// C++11 按索引访问
int i = std::get<0>(t);

// C++14 按类型访问
std::string s = std::get<std::string>(t);
double d = std::get<double>(t);
```

#### `std::shared_timed_mutex`

支持超时的共享互斥量：

```cpp
#include <shared_mutex>
#include <chrono>

std::shared_timed_mutex mtx;

void reader() {
    // 尝试获取共享锁，最多等待 100ms
    if (mtx.try_lock_shared_for(std::chrono::milliseconds(100))) {
        // 读操作
        mtx.unlock_shared();
    }
}

void writer() {
    // 尝试获取独占锁，最多等待 100ms
    if (mtx.try_lock_for(std::chrono::milliseconds(100))) {
        // 写操作
        mtx.unlock();
    }
}
```

## 5. 使用场景

### 适用场景

| 场景 | 推荐使用的 C++14 特性 |
|------|----------------------|
| 泛型编程 | 泛型 Lambda、`decltype(auto)` |
| 编译期计算 | 扩展 `constexpr` |
| 智能指针管理 | `std::make_unique` |
| 元编程 | `std::integer_sequence` |
| 代码可读性 | 数字分隔符、二进制字面量 |
| 并发编程 | `std::shared_timed_mutex` |
| API 演进 | `[[deprecated]]` 属性 |

### 编译器版本选择建议

| 编译器 | 最低支持版本 | 完整支持版本 |
|--------|-------------|-------------|
| GCC | 4.9 | 5.0+ |
| Clang | 3.4 | 3.5+ |
| MSVC | 18.0 | 19.0 (VS 2015) |
| Apple Clang | Yes | 默认支持 |

### 最佳实践

1. **启用 C++14 标准**：编译时添加 `-std=c++14` 或 `-std=c++1y`
2. **检查编译器支持**：使用 `__cplusplus` 宏验证（C++14 值为 `201402L`）
3. **渐进式升级**：C++14 与 C++11 高度兼容，可平滑升级

```cpp
// 检查 C++14 支持
#if __cplusplus >= 201402L
    // C++14 代码
    auto f = [](auto x) { return x * 2; };
#else
    // C++11 降级代码
    template<typename T>
    auto f(T x) -> decltype(x * 2) { return x * 2; }
#endif
```

### 常见陷阱

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| 返回类型推导限制 | 递归函数需有非递归返回路径 | 确保所有返回路径类型一致 |
| `constexpr` 限制 | 仍有部分限制（如动态内存分配）| C++20 进一步放宽限制 |
| Lambda 捕获生命周期 | 初始化捕获需注意对象生命周期 | 使用智能指针管理资源 |
| 编译器版本差异 | 部分特性支持不完整 | 查阅文档确认支持情况 |

## 6. 代码示例

### 基础用法：泛型 Lambda 与返回类型推导

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

// C++14 泛型 Lambda
auto print = [](const auto& container) {
    for (const auto& item : container) {
        std::cout << item << " ";
    }
    std::cout << std::endl;
};

// 返回类型推导
template<typename Container>
auto get_size(const Container& c) {
    return c.size();  // 自动推导为 size_t
}

// decltype(auto) 保持引用和 cv 限定符
template<typename T>
decltype(auto) forward_value(T&& value) {
    return std::forward<T>(value);
}

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    print(v);
    std::cout << "Size: " << get_size(v) << std::endl;
    return 0;
}
```

### 高级用法：元编程与参数包展开

```cpp
#include <utility>
#include <array>
#include <tuple>

// 使用 integer_sequence 展开参数包
template<typename T, T... Is>
constexpr auto make_array(std::integer_sequence<T, Is...>) {
    return std::array<T, sizeof...(Is)>{Is...};
}

// 创建 0 到 N-1 的数组
template<std::size_t N>
constexpr auto make_index_array() {
    return make_array(std::make_index_sequence<N>{});
}

// 调用函数并传入元组参数
template<typename F, typename Tuple, std::size_t... Is>
constexpr auto apply_impl(F&& f, Tuple&& t, std::index_sequence<Is...>) 
    -> decltype(std::forward<F>(f)(std::get<Is>(std::forward<Tuple>(t))...)) {
    return std::forward<F>(f)(std::get<Is>(std::forward<Tuple>(t))...);
}

template<typename F, typename Tuple>
constexpr auto apply(F&& f, Tuple&& t)
    -> decltype(apply_impl(std::forward<F>(f), std::forward<Tuple>(t),
        std::make_index_sequence<std::tuple_size<std::decay_t<Tuple>>::value>{})) {
    return apply_impl(std::forward<F>(f), std::forward<Tuple>(t),
        std::make_index_sequence<std::tuple_size<std::decay_t<Tuple>>::value>{});
}

int add(int a, int b, int c) {
    return a + b + c;
}

int main() {
    constexpr auto arr = make_index_array<5>();  // {0, 1, 2, 3, 4}
    auto tup = std::make_tuple(1, 2, 3);
    auto result = apply(add, tup);  // result = 6
    return 0;
}
```

### 实际应用：智能指针与资源管理

```cpp
#include <memory>
#include <vector>
#include <string>
#include <iostream>

class Widget {
public:
    Widget(std::string name, int value) 
        : name_(std::move(name)), value_(value) {
        std::cout << "Widget created: " << name_ << std::endl;
    }
    ~Widget() {
        std::cout << "Widget destroyed: " << name_ << std::endl;
    }
    
    void show() const {
        std::cout << name_ << ": " << value_ << std::endl;
    }
    
private:
    std::string name_;
    int value_;
};

int main() {
    // 使用 make_unique 创建独占指针
    auto w1 = std::make_unique<Widget>("Widget1", 100);
    w1->show();
    
    // 创建数组
    auto arr = std::make_unique<int[]>(10);
    for (int i = 0; i < 10; ++i) {
        arr[i] = i * 10;
    }
    
    // 在容器中使用
    std::vector<std::unique_ptr<Widget>> widgets;
    widgets.push_back(std::make_unique<Widget>("Widget2", 200));
    widgets.push_back(std::make_unique<Widget>("Widget3", 300));
    
    for (const auto& w : widgets) {
        w->show();
    }
    
    return 0;  // 自动释放资源
}
```

## 7. 总结

### 核心要点回顾

| 类别 | 关键特性 | 主要收益 |
|------|----------|----------|
| 语言简化 | 函数返回类型推导、泛型 Lambda | 减少样板代码 |
| 表达力增强 | 二进制字面量、数字分隔符 | 提高代码可读性 |
| 泛型编程 | 变量模板、扩展 `constexpr` | 增强编译期计算能力 |
| 资源管理 | `std::make_unique` | 更安全的智能指针使用 |
| 元编程 | `std::integer_sequence` | 简化参数包操作 |
| 并发支持 | `std::shared_timed_mutex` | 读写分离锁 |

### 编译器支持总结

- **GCC**：5.0+ 完整支持，4.9 部分支持
- **Clang**：3.4+ 完整支持
- **MSVC**：VS 2015 (19.0) 完整支持
- **Apple Clang**：默认支持

### 升级建议

1. **从 C++11 升级**：C++14 与 C++11 高度兼容，升级成本低
2. **推荐最低版本**：建议使用 GCC 5+、Clang 3.5+、MSVC 2015+
3. **编译标志**：`-std=c++14` 或 `-std=gnu++14`（GNU 扩展）

### 相关技术对比

| 特性 | C++11 | C++14 | C++17 |
|------|-------|-------|-------|
| Lambda | 类型固定 | 泛型 Lambda | `constexpr` Lambda |
| 返回类型推导 | Lambda only | 普通函数支持 | 改进 |
| `constexpr` | 单一返回语句 | 支持复杂逻辑 | 更宽松 |
| 结构化绑定 | 不支持 | 不支持 | 支持 |
| `optional` | 不支持 | 不支持 | 支持 |
