# 默认构造函数 (Default Constructor)

## 1. 概述

默认构造函数（default constructor）是可以无参调用的构造函数。它是 C++ 中最基本的构造函数类型之一，用于在创建对象时执行默认初始化（default initialization）或值初始化（value initialization）。

默认构造函数在以下场景中具有关键作用：
- 对象的默认初始化
- 值初始化
- 数组元素的初始化
- 容器中元素的默认构造

## 2. 来源与演变

### 首次引入

默认构造函数的概念自 **C++98** 标准起就已存在，是面向对象编程的基础特性之一。

### 历史背景

在 C++ 早期设计中，构造函数的自动化程度有限。随着语言发展，默认构造函数的规则逐渐完善：
- C++98：确立基本的隐式声明和定义规则
- C++11：引入 `= default` 和 `= delete` 语法，增强控制能力
- C++17：完善 `noexcept` 异常规范
- C++20：引入"合格默认构造函数"（eligible default constructor）概念，支持约束模板
- C++23：对 `constexpr` 构造函数规则进行调整

### 关键缺陷报告

| DR | 应用版本 | 原行为 | 修正行为 |
|-----|---------|--------|----------|
| CWG 1353 | C++98 | 隐式声明默认构造函数为未定义的情况未考虑多维数组类型 | 纳入考虑 |
| CWG 2084 | C++11 | 默认成员初始化器对联合体的默认构造函数是否删除无影响 | 可阻止删除 |
| CWG 2595 | C++20 | 存在更受约束但不满足约束的默认构造函数时，其他构造函数不合格 | 可合格 |
| CWG 2871 | C++98 | 存在用户声明的构造函数模板时仍隐式声明默认构造函数 | 不隐式声明 |

## 3. 语法与参数

### 声明语法

| 语法形式 | 版本 | 说明 |
|---------|------|------|
| `class-name(parameter-list(optional));` | C++98 | 类内声明默认构造函数 |
| `class-name(parameter-list(optional)) function-body` | C++98 | 类内定义默认构造函数 |
| `class-name() = default;` | C++11 起 | 显式默认化 |
| `class-name(parameter-list(optional)) = delete;` | C++11 起 | 显式删除 |
| `class-name::class-name(parameter-list(optional)) function-body` | C++98 | 类外定义默认构造函数 |
| `class-name::class-name() = default;` | C++11 起 | 类外显式默认化 |

### 参数说明

| 参数 | 说明 |
|------|------|
| `class-name` | 正在声明默认构造函数的类名 |
| `parameter-list` | 参数列表，其中所有参数（除参数包外）都必须有默认实参 |
| `function-body` | 默认构造函数的函数体 |

### 参数列表要求

默认构造函数的参数列表必须满足：**所有参数都具有默认实参**（参数包除外，C++11 起）。这意味着函数可以无参调用。

```cpp
// 有效的默认构造函数声明
struct A {
    A(int x = 0);              // 所有参数都有默认值
    A(int x, int y = 0);       // 部分参数有默认值，仍可无参调用（错误示例）
    A(int x = 1, int y = 2);   // 所有参数都有默认值
};

// 注意：A(int x, int y = 0) 不是默认构造函数，因为 x 没有默认值
```

## 4. 底层原理

### 隐式声明的默认构造函数

当类类型没有用户声明的构造函数或构造函数模板时，编译器会隐式声明一个默认构造函数作为其类的内联（inline）公有成员。

**隐式声明的条件**：
- 类中没有用户声明的构造函数
- 类中没有用户声明的构造函数模板（CWG 2871 修正）

### 隐式定义的默认构造函数

如果默认构造函数满足以下条件，编译器会隐式定义它（生成函数体并编译）：

**C++11 之前**：隐式声明的默认构造函数

**C++11 起**：隐式声明或首次声明时显式默认化的默认构造函数，且未被定义为删除

**隐式定义的效果**：
- 相当于一个空函数体、空初始化列表的用户定义构造函数
- 调用基类的默认构造函数
- 调用非静态成员的默认构造函数

### 删除的默认构造函数

隐式声明或显式默认化的默认构造函数在以下情况下被定义为删除：

| 条件 | 说明 |
|------|------|
| 联合体所有变体成员都是 const 限定类型 | 无法构造 |
| 非联合体的匿名联合体成员都是 const 限定类型 | 无法构造 |
| 有引用类型的非静态数据成员且无默认初始化器 | 引用必须初始化 |
| 有 const 限定的非变体非静态数据成员且无默认成员初始化器、类型不可默认构造 | const 成员必须初始化 |
| 有类类型子对象 `M` 的析构函数被删除或不可访问 | 无法析构 |
| 有类类型子对象 `M` 无法找到可用的默认构造函数 | 无法构造子对象 |

### 平凡的默认构造函数

默认构造函数是平凡的（trivial）需满足所有以下条件：

1. 构造函数不是用户提供的（即隐式定义或首次声明时默认化）
2. 类没有虚成员函数
3. 类没有虚基类
4. 类没有带默认初始化器的非静态成员（C++11 起）
5. 每个直接基类都有平凡的默认构造函数
6. 每个类类型的非静态成员都有平凡的默认构造函数

**平凡构造函数的特点**：
- 不执行任何操作
- 所有与 C 语言兼容的数据类型（POD 类型）都是平凡可默认构造的

### 合格的默认构造函数

| 版本 | 合格条件 |
|------|---------|
| C++11 前 | 用户声明，或隐式声明且可定义 |
| C++11 起 | 未被删除 |
| C++20 起 | 满足所有条件：未删除、关联约束满足、没有更受约束的候选 |

合格默认构造函数的平凡性决定类是否为隐式生命周期类型（implicit-lifetime type）和平凡类型（trivial type）。

## 5. 使用场景

### 适合使用默认构造函数的场景

| 场景 | 说明 |
|------|------|
| 简单数据类 | 包含基本类型成员，无需特殊初始化 |
| 容器元素类型 | 标准容器（如 `std::vector`）要求元素可默认构造 |
| 工厂模式 | 通过默认构造创建对象后再设置属性 |
| 延迟初始化 | 对象先默认构造，稍后填充数据 |

### 不适合使用默认构造函数的场景

| 场景 | 替代方案 | 原因 |
|------|---------|------|
| 类包含引用成员 | 使用带参构造函数 | 引用必须初始化 |
| 类包含 const 成员 | 使用带参构造函数 | const 成员必须初始化 |
| 类需要资源分配 | 使用带参构造函数 | 避免无效状态 |
| 类不变量需要保证 | 使用带参构造函数 | 防止无效对象 |

### 最佳实践

1. **优先使用 `= default`**：当需要编译器生成默认构造函数时，显式使用 `= default` 表达意图
2. **避免无效状态**：如果对象需要有效状态才能工作，不要提供默认构造函数
3. **使用成员初始化器**：C++11 起，使用默认成员初始化器简化构造函数

```cpp
// 推荐：使用默认成员初始化器
struct Good {
    int x = 0;
    std::string name = "default";
    // 默认构造函数自动使用这些初始值
};

// 避免：默认构造后对象处于无效状态
struct Bad {
    int* ptr;  // 未初始化，可能为野指针
    // 默认构造函数导致无效状态
};
```

### 常见陷阱

1. **用户定义的复制构造函数阻止隐式默认构造函数**：如果定义了复制构造函数，编译器不再隐式生成默认构造函数

2. **平凡性影响序列化**：平凡的默认构造函数允许使用 `memcpy` 等操作，非平凡的则不行

3. **虚函数导致非平凡**：存在虚函数时，默认构造函数需要设置虚表指针

## 6. 代码示例

### 基础用法

```cpp
#include <iostream>

// 用户定义的默认构造函数
struct A {
    int x;
    A(int x = 1) : x(x) {}  // 用户定义的默认构造函数
};

// 隐式定义的默认构造函数
struct B : A {
    // B::B() 隐式定义，调用 A::A()
};

// 包含成员的类
struct C {
    A a;
    // C::C() 隐式定义，调用 A::A()
};

int main() {
    A a;       // 调用 A::A(1)
    B b;       // 调用 B::B() -> A::A(1)
    C c;       // 调用 C::C() -> A::A(1)

    std::cout << a.x << '\n';  // 输出: 1
    std::cout << b.x << '\n';  // 输出: 1 (继承自 A)
    std::cout << c.a.x << '\n'; // 输出: 1

    return 0;
}
```

### 显式默认化和删除

```cpp
// 定义其他构造函数后显式默认化
struct D : A {
    D(int y) : A(y) {}
    D() = default;  // 显式默认化，调用 A::A()
};

// 因其他构造函数存在而不会隐式声明
struct E : A {
    E(int y) : A(y) {}
    // E::E() 未声明，因为存在其他构造函数
};

// 删除的默认构造函数
struct F {
    int& ref;      // 引用成员
    const int c;   // const 成员
    // F::F() 隐式定义为删除
};

int main() {
    D d1;      // OK，调用 D::D() -> A::A(1)
    D d2(42);  // OK，调用 D::D(42) -> A::A(42)

    // E e;    // 错误：E 没有默认构造函数
    // F f;    // 错误：F 的默认构造函数被删除

    return 0;
}
```

### 常见错误及修正

#### 错误 1：定义复制构造函数后误以为存在默认构造函数

```cpp
// 错误：用户声明的复制构造函数阻止隐式生成默认构造函数
struct G {
    G(const G&) {}  // 用户定义的复制构造函数
    // G::G() 隐式定义为删除
};

struct H {
    H(const H&) = delete;  // 删除的复制构造函数
    // H::H() 隐式定义为删除
};

struct I {
    I(const I&) = default;  // 默认化的复制构造函数
    // I::I() 隐式定义为删除
};

int main() {
    // G g;  // 错误：默认构造函数被删除
    // H h;  // 错误：默认构造函数被删除
    // I i;  // 错误：默认构造函数被删除

    // 修正方案：显式默认化
    struct G2 {
        G2(const G2&) {}
        G2() = default;  // 显式默认化
    };

    G2 g2;  // OK
    return 0;
}
```

#### 错误 2：引用成员和 const 成员未初始化

```cpp
// 错误：包含引用和 const 成员的类无法默认构造
struct Bad {
    int& ref;     // 引用成员必须初始化
    const int c;  // const 成员必须初始化
};

// 修正：提供带参构造函数或使用默认成员初始化器
struct Good {
    int& ref;
    const int c;

    Good(int& r, int v) : ref(r), c(v) {}  // 带参构造函数
};

struct Good2 {
    int x = 42;           // 默认成员初始化器 (C++11)
    std::string s = "";   // 默认成员初始化器
    // 默认构造函数会使用这些初始值
};

int main() {
    // Bad b;  // 错误：默认构造函数被删除

    int value = 10;
    Good g(value, 5);  // OK

    Good2 g2;  // OK，x=42, s=""
    return 0;
}
```

#### 错误 3：平凡性误解

```cpp
#include <iostream>

// 非平凡：有虚函数
struct NonTrivial1 {
    virtual void foo() {}
};

// 非平凡：有默认成员初始化器 (C++11)
struct NonTrivial2 {
    int x = 0;  // 这使得默认构造函数非平凡
};

// 平凡：满足所有条件
struct Trivial {
    int x;
    // 无虚函数、无虚基类、无默认初始化器
};

int main() {
    std::cout << std::is_trivially_default_constructible<NonTrivial1>::value;  // 0
    std::cout << std::is_trivially_default_constructible<NonTrivial2>::value;  // 0
    std::cout << std::is_trivially_default_constructible<Trivial>::value;      // 1

    return 0;
}
```

## 7. 总结

### 核心要点

| 概念 | 要点 |
|------|------|
| 定义 | 可无参调用的构造函数 |
| 隐式声明 | 无用户声明构造函数时自动生成 |
| 隐式定义 | 调用基类和成员的默认构造函数 |
| 删除条件 | 引用成员、const 成员、不可构造子对象 |
| 平凡性 | 无用户提供、无虚函数/虚基类、成员皆平凡 |
| 合格性 | C++20 起需满足约束检查 |

### 关键对比

| 特性 | 用户定义 | 隐式定义 | `= default` |
|------|---------|---------|-------------|
| 函数体可自定义 | 是 | 否 | 否 |
| 平凡性 | 通常非平凡 | 可能平凡 | 首次声明时平凡 |
| 可控性 | 完全可控 | 编译器决定 | 显式请求编译器生成 |

### 学习建议

1. **理解隐式规则**：掌握编译器何时自动生成默认构造函数
2. **善用 `= default`**：C++11 起，明确表达意图
3. **注意删除条件**：避免包含引用、const 成员导致的删除
4. **区分平凡与非平凡**：影响对象的内存布局和序列化行为
5. **与其他构造函数的关系**：定义其他构造函数时注意默认构造函数的处理
