# 值初始化 (Value-initialization)

## 概述 (Overview)

值初始化 (Value-initialization) 是 C++ 中的一种初始化方式，当对象使用空初始化器构造时执行此初始化。值初始化的核心目的是确保对象获得一个确定的初始值，避免出现未定义行为。

值初始化的行为取决于被初始化对象的类型：
- 对于类类型：先进行零初始化（如果满足条件），再进行默认初始化
- 对于数组类型：对每个元素进行值初始化
- 对于标量类型：进行零初始化

## 来源与演变 (Origin and Evolution)

### 历史背景

值初始化是在 C++98 标准中引入的概念，旨在解决空初始化器情况下的初始化语义问题。

### 版本变更

| 版本 | 变更内容 |
|------|----------|
| C++98 | 引入值初始化概念，在此之前空初始化器调用默认初始化 |
| C++11 | 引入列表初始化语法 `{}`，支持使用空花括号进行值初始化 |
| C++11 | 如果类没有默认构造函数但有 `std::initializer_list` 构造函数，则执行列表初始化 |

### 缺陷报告

| DR | 版本 | 原有行为 | 修正行为 |
|----|------|----------|----------|
| CWG 178 | C++98 | 没有值初始化概念，空初始化器调用默认初始化 | 空初始化器调用值初始化 |
| CWG 543 | C++98 | 没有用户提供构造函数的类对象值初始化等同于对每个子对象值初始化 | 先零初始化整个对象，再调用默认构造函数 |
| CWG 1301 | C++11 | 删除了默认构造函数的联合体值初始化导致零初始化 | 执行默认初始化 |
| CWG 1368 | C++98 | 任何用户提供构造函数都会跳过零初始化 | 只有用户提供的默认构造函数才会跳过零初始化 |
| CWG 1502 | C++11 | 没有用户提供默认构造函数的联合体值初始化只进行零初始化 | 零初始化后执行默认初始化 |
| CWG 1507 | C++98 | 没有用户提供构造函数的类对象值初始化不检查平凡默认构造函数的有效性 | 检查平凡默认构造函数的有效性 |
| CWG 2820 | C++98 | 零初始化后的默认初始化需要非平凡构造函数 | 不需要非平凡构造函数 |
| CWG 2859 | C++98 | 即使默认初始化未选择用户提供的构造函数，类对象值初始化也可能涉及零初始化 | 此情况下不进行零初始化 |

## 语法与参数 (Syntax and Parameters)

### 语法形式

```cpp
T()                              // (1) 函数式转换
new T()                          // (2) new 表达式
Class::Class(...) : member() {}  // (3) 成员初始化器
T object {};                     // (4) 空花括号初始化 (C++11)
T {}                             // (5) 函数式列表初始化 (C++11)
new T {}                         // (6) new 列表初始化 (C++11)
Class::Class(...) : member{} {}  // (7) 成员列表初始化器 (C++11)
```

### 语法说明

| 形式 | 说明 | 引入版本 |
|------|------|----------|
| (1) | 创建无名临时对象，使用空括号初始化 | C++98 |
| (2) | 使用 new 表达式创建动态存储期对象，使用空括号初始化 | C++98 |
| (3) | 在构造函数的成员初始化列表中使用空括号初始化成员或基类 | C++98 |
| (4) | 声明命名对象时使用空花括号初始化 | C++11 |
| (5) | 创建无名临时对象，使用空花括号初始化 | C++11 |
| (6) | 使用 new 表达式创建动态存储期对象，使用空花括号初始化 | C++11 |
| (7) | 在构造函数的成员初始化列表中使用空花括号初始化成员或基类 | C++11 |

### 重要语法注意事项

**最令人困惑的语法陷阱**：

```cpp
T object();  // 注意：这不初始化对象！
             // 这是一个函数声明，声明了一个返回 T 类型、无参数的函数
```

在 C++11 之前，命名变量的值初始化写法为：
```cpp
T object = T();  // 值初始化一个临时对象，然后复制初始化
                 // 大多数编译器会优化掉这个复制
```

## 底层原理 (Underlying Principles)

### 值初始化的效果

值初始化的具体效果取决于类型 T：

#### 类类型

如果 `T` 是（可能带有 cv 限定符的）类类型：

1. **零初始化阶段**：如果 `T` 的默认初始化选择了一个构造函数，且该构造函数不是用户声明的（C++11 前）/ 不是用户提供的（C++11 起），则对象首先被零初始化
2. **默认初始化阶段**：无论是否进行了零初始化，对象都会被默认初始化

```cpp
struct ImplicitDefault {
    int mem1;
    std::string mem2;
    // 隐式默认构造函数，非用户提供
};

ImplicitDefault obj{};  // 零初始化 mem1 = 0，默认初始化 mem2 = ""

struct UserProvidedDefault {
    int mem1;
    std::string mem2;
    UserProvidedDefault() {}  // 用户提供的默认构造函数
};

UserProvidedDefault obj{};  // 默认初始化，mem1 值不确定，mem2 = ""
```

#### 数组类型

如果 `T` 是数组类型，对数组的每个元素进行值初始化：

```cpp
int arr[5]{};        // 每个元素都值初始化为 0
int* p = new int[10]();  // 动态数组，每个元素值初始化为 0
```

#### 标量类型

如果 `T` 是标量类型（整型、浮点型、指针等），对象被零初始化：

```cpp
int n{};           // 零初始化，n = 0
double d = double(); // 零初始化，d = 0.0
int* ptr{};         // 零初始化，ptr = nullptr
```

### 与聚合初始化的区别

当使用空花括号 `{}` 且 `T` 是聚合类型时，执行的是聚合初始化而非值初始化：

```cpp
struct Aggregate {
    int x;
    double y;
};

Aggregate a{};  // 聚合初始化，x = 0, y = 0.0
```

## 使用场景 (Use Cases)

### 适用场景

1. **确保对象初始化为零值**
   ```cpp
   int counter{};        // 确保计数器从 0 开始
   double average{};     // 确保平均值为 0.0
   int* pointer{};       // 确保指针为 nullptr
   ```

2. **动态数组初始化**
   ```cpp
   int* arr = new int[100]();  // 所有元素初始化为 0
   ```

3. **容器元素初始化**
   ```cpp
   std::vector<int> v(10);  // 10 个元素，每个都值初始化为 0
   v.resize(20);            // 新增的 10 个元素值初始化为 0
   ```

4. **成员初始化**
   ```cpp
   class Widget {
       int id_{};
       std::string name_{};
   public:
       Widget() = default;  // 成员被值初始化
   };
   ```

### 最佳实践

1. **优先使用花括号初始化**（C++11 起）
   ```cpp
   int x{};      // 推荐：清晰表达值初始化意图
   int y = 0;    // 等效，但意图不够明确
   ```

2. **避免函数声明歧义**
   ```cpp
   Widget w();   // 错误：这是函数声明！
   Widget w{};   // 正确：值初始化
   ```

### 常见陷阱

1. **函数声明歧义**
   ```cpp
   std::string s();  // 这是函数声明，不是对象初始化！
   std::string s{};  // 这才是值初始化
   ```

2. **引用不能值初始化**
   ```cpp
   int& ref{};  // 错误：引用不能值初始化
   int& ref = obj;  // 正确：引用必须绑定到对象
   ```

3. **数组类型的函数式转换限制**
   ```cpp
   int arr[5];
   int()();   // 错误：T() 形式不允许 T 是数组类型
   int{};     // 正确：T{} 形式允许 T 是数组类型
   ```

4. **无默认构造函数的类**
   ```cpp
   struct NoDefault {
       NoDefault(int) {}
   };
   NoDefault obj{};  // 错误：没有默认构造函数
   ```

## 代码示例 (Examples)

### 基础用法

```cpp
#include <cassert>
#include <iostream>
#include <string>
#include <vector>

// 隐式默认构造函数的类
struct T1 {
    int mem1;
    std::string mem2;
    virtual void foo() {}  // 确保 T1 不是聚合类型
};  // 隐式默认构造函数

// 无默认构造函数的类
struct T2 {
    int mem1;
    std::string mem2;
    T2(const T2&) {}  // 用户提供的复制构造函数
};  // 无默认构造函数

// 用户提供的默认构造函数
struct T3 {
    int mem1;
    std::string mem2;
    T3() {}  // 用户提供的默认构造函数
};

// 全局变量：类类型 => 默认初始化
std::string s{};  // 值为 ""

int main() {
    // 标量类型 => 零初始化
    int n{};
    assert(n == 0);

    // 标量类型的函数式转换
    double f = double();
    assert(f == 0.0);

    // 动态数组：每个元素值初始化
    int* a = new int[10]();
    assert(a[9] == 0);

    // 类类型（隐式默认构造函数）
    // mem1 被零初始化为 0，mem2 被默认初始化为 ""
    T1 t1{};
    assert(t1.mem1 == 0);
    assert(t1.mem2 == "");

    // T2 t2{};  // 错误：类没有默认构造函数

    // 类类型（用户提供的默认构造函数）
    // mem1 被默认初始化为不确定值，mem2 被默认初始化为 ""
    T3 t3{};
    std::cout << "t3.mem1 value: " << t3.mem1 << '\n';
    assert(t3.mem2 == "");

    // 容器元素值初始化
    std::vector<int> v(3);
    assert(v[2] == 0);

    delete[] a;
    return 0;
}
```

### 成员初始化示例

```cpp
#include <iostream>
#include <string>

class Entity {
    int id_{};           // 值初始化为 0
    std::string name_{}; // 值初始化为 ""
    double score_{};     // 值初始化为 0.0

public:
    // 成员初始化器中使用值初始化
    Entity() : id_{}, name_{}, score_{} {}

    void print() const {
        std::cout << "id: " << id_
                  << ", name: \"" << name_ << "\""
                  << ", score: " << score_ << '\n';
    }
};

int main() {
    Entity e;
    e.print();  // 输出: id: 0, name: "", score: 0
    return 0;
}
```

### 常见错误及修正

```cpp
#include <iostream>
#include <string>

// 错误示例 1：函数声明歧义
void example1_wrong() {
    std::string s();  // 这声明了一个函数，不是对象！
    // s.size();      // 编译错误

void example1_correct() {
    std::string s{};  // 正确：值初始化
    std::cout << s.size() << '\n';  // 输出: 0

struct NoDefault {
    int value;
    NoDefault(int v) : value(v) {}
};

void example2_wrong() {
    // NoDefault obj{};  // 错误：没有默认构造函数
}

void example2_correct() {
    NoDefault obj{42};  // 正确：使用带参数的构造函数
}

// 错误示例 3：引用不能值初始化
void example3_wrong() {
    // int& ref{};  // 错误：引用不能值初始化
}

void example3_correct() {
    int x = 10;
    int& ref = x;  // 正确：引用必须绑定到对象
}

    example1_correct();
    return 0;
}
```

## 总结 (Summary)

### 核心要点

1. **定义**：值初始化是使用空初始化器构造对象时执行的初始化方式

2. **效果**：
   - 类类型：先零初始化（满足条件时），再默认初始化
   - 数组类型：每个元素值初始化
   - 标量类型：零初始化

3. **语法**：
   - C++11 前使用 `T()`、`new T()`、成员初始化器 `member()`
   - C++11 起推荐使用 `{}` 语法，避免函数声明歧义

4. **适用性**：
   - 引用不能值初始化
   - 无默认构造函数的类不能值初始化
   - 聚合类型使用 `{}` 时执行聚合初始化

### 技术对比

| 特性 | 值初始化 | 默认初始化 | 零初始化 |
|------|----------|------------|----------|
| 标量类型 | 零值 | 不确定值 | 零值 |
| 无用户提供构造函数的类 | 零值成员 | 不确定值成员 | 零值成员 |
| 有用户提供构造函数的类 | 调用构造函数 | 调用构造函数 | N/A |
| 数组 | 每个元素值初始化 | 每个元素默认初始化 | 每个元素零初始化 |

### 学习建议

1. **优先使用花括号语法** `{}` 进行值初始化，这是现代 C++ 的最佳实践

2. **理解与默认初始化的区别**：默认初始化对于内置类型可能产生不确定值，而值初始化保证零值

3. **注意构造函数性质**：区分隐式默认构造函数和用户提供的默认构造函数对值初始化行为的影响

4. **避免函数声明歧义**：永远不要使用 `T object()` 形式来声明对象
