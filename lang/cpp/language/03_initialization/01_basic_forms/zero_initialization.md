# 零初始化 (Zero-initialization)

## 1. 概述

**零初始化 (Zero-initialization)** 是 C++ 中的一种初始化方式，它将对象的初始值设置为零。这是一种隐式的初始化行为，没有专用的语法关键字，而是通过特定的声明形式或其他初始化过程间接触发。

零初始化是 C++ 初始化体系中的基础机制，它在以下场景中自动发生：
- 具有静态或线程存储期的变量
- 值初始化序列的一部分
- 字符数组使用字符串字面量初始化时的剩余部分

## 2. 来源与演变

### 首次引入

零初始化的概念自 **C++98** 标准起就存在，是 C++ 初始化体系的核心组成部分。

### 标准演进

| 版本 | 变化内容 |
|------|----------|
| C++98 | 首次定义零初始化规则 |
| C++11 | 增加线程存储期 (thread-local storage) 变量的零初始化支持；新增 `{}` 列表初始化语法 |

### 缺陷报告修正

C++ 标准委员会对零初始化进行了多次缺陷修正：

| 缺陷报告 | 适用版本 | 原有行为 | 修正行为 |
|----------|----------|----------|----------|
| CWG 277 | C++98 | 指针可能被非常量表达式值 0 初始化，这不是空指针常量 | 必须用值为 0 的整型常量表达式初始化 |
| CWG 694 | C++98 | 类类型的零初始化忽略填充位 (padding bits) | 填充位初始化为零 |
| CWG 903 | C++98 | 标量类型的零初始化使用整型常量表达式的转换值 | 使用整数字面量 0 的转换值 |
| CWG 2026 | C++98 | 零初始化总是最先发生，即使在常量初始化之前 | 如果适用常量初始化，则不进行零初始化 |
| CWG 2196 | C++98 | 类类型的零初始化忽略基类子对象 | 基类子对象也被零初始化 |
| CWG 2253 | C++98 | 零初始化是否适用于未命名的位字段不清楚 | 适用（所有填充位初始化为零） |

## 3. 语法与参数

### 触发零初始化的语法形式

零初始化本身没有专用语法，以下语法形式会触发零初始化：

#### 形式 1：静态存储期变量声明

```cpp
static T object;
```

对于具有静态或线程存储期（C++11 起）的命名变量，如果不受常量初始化约束，会首先进行零初始化。

#### 形式 2：值初始化

```cpp
T();              // 函数式转换
T t = {};         // 复制列表初始化
T{};              // 直接列表初始化 (C++11 起)
```

作为值初始化 (value-initialization) 序列的一部分，用于非类类型和无构造函数的类类型成员。

#### 形式 3：字符数组部分初始化

```cpp
CharT array[n] = "short-sequence";
```

当字符数组使用较短的字符串字面量初始化时，数组的剩余部分被零初始化。

### 参数说明

| 符号 | 说明 |
|------|------|
| `T` | 任意类型 |
| `object` | 变量名 |
| `CharT` | 任意字符类型 (char, wchar_t, char16_t, char32_t) |
| `n` | 数组大小 |
| `short-sequence` | 长度小于 n 的字符串字面量 |

## 4. 底层原理

### 零初始化的效果

零初始化根据类型 `T` 的不同，产生以下效果：

#### 标量类型 (Scalar Type)

如果 `T` 是标量类型（整型、浮点型、指针、枚举等），对象被初始化为整数字面量 `0` 显式转换为 `T` 的值。

```cpp
int i;           // 零初始化为 0
double d;        // 零初始化为 0.0
int* p;          // 零初始化为空指针值（即使空指针值不是整数 0）
```

#### 非联合类类型 (Non-union Class Type)

如果 `T` 是非联合类类型：

1. 所有填充位 (padding bits) 初始化为零
2. 每个非静态数据成员被零初始化
3. 每个非虚基类子对象被零初始化
4. 如果对象不是基类子对象，每个虚基类子对象被零初始化

```cpp
struct A {
    int a, b, c;
};
// A a{}; 之后，a.a=0, a.b=0, a.c=0
```

#### 联合类型 (Union Type)

如果 `T` 是联合类型：

1. 所有填充位初始化为零
2. 第一个非静态命名数据成员被零初始化

```cpp
union U {
    int x;
    double y;
};
// U u{}; 之后，u.x=0
```

#### 数组类型 (Array Type)

如果 `T` 是数组类型，每个元素被零初始化。

```cpp
int arr[5];      // 所有元素为零
double f[3];     // 所有元素为 0.0
```

#### 引用类型 (Reference Type)

如果 `T` 是引用类型，不进行任何操作（引用必须在定义时绑定）。

### 初始化顺序

对于具有静态或线程存储期的变量，初始化顺序如下：

1. **零初始化** - 首先执行
2. **常量初始化** - 如果适用
3. **动态初始化** - 最后执行

### 空指针值的特殊性

零初始化的指针始终是空指针值，即使底层表示中空指针值不是整数 `0`。这保证了代码的可移植性。

## 5. 使用场景

### 适用场景

| 场景 | 说明 |
|------|------|
| 全局变量 | 具有静态存储期的变量自动零初始化 |
| 静态局部变量 | `static` 声明的局部变量在首次使用前零初始化 |
| 线程局部变量 | `thread_local` 变量 (C++11) 零初始化 |
| 聚合类型初始化 | 使用 `{}` 初始化聚合类型时未指定的成员 |
| 动态内存清零 | `new T()` 或 `new T[n]()` 进行值初始化 |

### 最佳实践

1. **依赖零初始化而非手动清零**：对于静态存储期变量，无需显式初始化为零

```cpp
// 推荐：依赖零初始化
static int counter;  // 自动零初始化为 0

// 不推荐：冗余初始化
static int counter = 0;  // 冗余
```

2. **使用 `{}` 进行值初始化**：确保类类型成员被零初始化

```cpp
struct Data {
    int x, y, z;
};

Data d{};  // x=0, y=0, z=0
```

3. **安全删除空指针**：零初始化的指针是空指针，可以安全删除

```cpp
int* p;  // 零初始化为空指针
delete p;  // 安全，删除空指针无操作
```

### 常见陷阱

#### 陷阱 1：局部变量不会零初始化

```cpp
void func() {
    int x;  // 未初始化！值是不确定的
    // 使用 x 是未定义行为
}

void func_correct() {
    int x{};  // 零初始化
    // 或
    int x = 0;  // 显式初始化
}
```

#### 陷阱 2：动态分配的数组默认不零初始化

```cpp
int* p1 = new int[10];    // 未初始化！
int* p2 = new int[10]();  // 零初始化

delete[] p1;
delete[] p2;
```

#### 陷阱 3：类类型成员可能有默认构造函数

```cpp
std::string s;  // 零初始化为不确定值，然后默认构造为 ""
// 注意：类类型首先零初始化，然后调用默认构造函数
```

### 线程安全性

零初始化的静态变量在多线程环境下是线程安全的。C++11 标准保证静态局部变量的初始化是线程安全的（即静态初始化阶段完成）。

## 6. 代码示例

### 基础用法

```cpp
#include <iostream>
#include <string>

struct A {
    int a, b, c;
};

// 全局变量 - 零初始化
double f[3];     // 三个 0.0
int* p;          // 空指针值
std::string s;   // 首先零初始化，然后默认构造为 ""

int main(int argc, char*[]) {
    // 安全删除空指针
    delete p;    // 安全

    // 静态局部变量 - 先零初始化，再复制初始化
    static int n = argc;  // 零初始化为 0，然后复制初始化为 argc
    std::cout << "n = " << n << '\n';

    // 值初始化触发零初始化
    A a = A();   // 等价于 A a{}; 或 A a = {};
    std::cout << "a = {" << a.a << ' ' << a.b << ' ' << a.c << "}\n";

    return 0;
}
```

输出：
```
n = 1
a = {0 0 0}
```

### 高级用法

```cpp
#include <iostream>
#include <cstring>

// 线程局部存储 (C++11)
thread_local int tls_counter;  // 每个线程独立零初始化

struct Complex {
    int x;
    double y;
    char name[32];
};

void demo_thread_local() {
    tls_counter++;
    std::cout << "Thread local counter: " << tls_counter << '\n';
}

int main() {
    // 字符数组部分初始化 - 剩余部分零初始化
    char buffer[10] = "Hi";
    // buffer = {'H', 'i', '\0', '\0', '\0', '\0', '\0', '\0', '\0', '\0'}

    // 值初始化 - 触发零初始化
    Complex c{};
    std::cout << "x=" << c.x << ", y=" << c.y << ", name='"
              << c.name << "'\n";
    // 输出: x=0, y=0, name=''

    // 动态分配的零初始化
    int* arr = new int[100]();  // 所有元素零初始化

    // 检查零初始化效果
    bool all_zero = true;
    for (int i = 0; i < 100; ++i) {
        if (arr[i] != 0) {
            all_zero = false;
            break;
        }
    }
    std::cout << "All elements are zero: " << std::boolalpha << all_zero << '\n';

    delete[] arr;

    return 0;
}
```

### 常见错误及修正

#### 错误 1：假设局部变量会零初始化

```cpp
// 错误：局部变量不会零初始化
void bad_function() {
    int arr[10];  // 未初始化，值不确定
    std::cout << arr[0];  // 未定义行为
}

// 修正：显式初始化
void good_function() {
    int arr[10] = {};  // 零初始化所有元素
    std::cout << arr[0];  // 安全：输出 0
}

// 修正：使用默认初始化语法
void good_function2() {
    int arr[10]{};  // C++11 列表初始化
    std::cout << arr[0];  // 安全：输出 0
}
```

#### 错误 2：忘记动态分配时的零初始化

```cpp
// 错误：动态数组未初始化
void bad_allocation() {
    int* p = new int[100];  // 值不确定
    // 使用前必须手动初始化或假定未初始化
}

// 修正：使用值初始化
void good_allocation() {
    int* p = new int[100]();  // 零初始化所有元素
    // ...
    delete[] p;
}

// 更好：使用智能指针和容器
void modern_approach() {
    auto p = std::make_unique<int[]>(100);  // 零初始化
    // 无需手动删除
}
```

#### 错误 3：混淆零初始化和默认初始化

```cpp
struct Widget {
    int id;
    std::string name;
};

void demonstrate_difference() {
    Widget w1;      // 默认初始化：id 未初始化（值不确定）
    Widget w2{};    // 值初始化：id=0, name=""

    // w1.id 是不确定值 - 未定义行为
    // w2.id 是 0 - 安全
}
```

## 7. 总结

### 核心要点

| 要点 | 说明 |
|------|------|
| **触发条件** | 静态/线程存储期变量、值初始化、字符数组剩余部分 |
| **标量类型** | 初始化为整数 0 转换后的值 |
| **指针类型** | 初始化为空指针值（不一定是全零位） |
| **类类型** | 所有成员和基类子对象零初始化 |
| **数组类型** | 每个元素零初始化 |
| **引用类型** | 不进行零初始化 |

### 初始化类型对比

| 初始化类型 | 局部变量 | 全局变量 | 动态分配 |
|------------|----------|----------|----------|
| 默认初始化 | 不确定值 | 零初始化 | 不确定值 |
| 值初始化 | 零初始化 | 零初始化 | 零初始化 |
| 零初始化 | 不适用 | 自动发生 | 需要 `()` 或 `{}` |

### 学习建议

1. **区分存储期**：理解静态存储期、线程存储期、自动存储期的区别
2. **掌握初始化时机**：零初始化发生在常量初始化之前（或替代之）
3. **使用现代语法**：优先使用 `{}` 列表初始化确保零初始化
4. **注意平台差异**：空指针值在不同平台可能不同，但零初始化保证可移植性

### 相关概念

| 概念 | 关系 |
|------|------|
| 默认初始化 (default initialization) | 不调用构造函数时的初始化行为 |
| 值初始化 (value initialization) | 包含零初始化的初始化序列 |
| 常量初始化 (constant initialization) | 优先于零初始化 |
| 聚合初始化 (aggregate initialization) | 未指定的成员进行值初始化 |
| 列表初始化 (list initialization) | 空列表 `{}` 触发值初始化 |
