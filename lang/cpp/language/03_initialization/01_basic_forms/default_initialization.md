# 默认初始化 (Default-initialization)

## 1. 概述 (Overview)

默认初始化（Default-initialization）是 C++ 中当对象在构造时未提供初始化器（initializer）时所执行的初始化行为。这是 C++ 初始化体系中最基础的初始化形式之一，其行为取决于被初始化对象的类型。

默认初始化的语义因类型而异：
- **类类型**：调用默认构造函数
- **数组类型**：对每个元素进行默认初始化
- **其他类型（标量类型等）**：不执行任何初始化，对象持有不确定值（indeterminate value）

理解默认初始化对于避免未定义行为（Undefined Behavior）和编写健壮的 C++ 代码至关重要。

## 2. 来源与演变 (Origin and Evolution)

### 首次引入

默认初始化的概念自 C++98 标准起就存在，但当时的行为与现代 C++ 有所不同。

### C++98 标准

在 C++98 中：
- 只有具有自动存储期的（可能带有 cv 限定符的）非 POD 类类型被认为会进行默认初始化
- 标量类型和 POD 类型在动态存储期下被认为"未初始化"
- 当时没有"值初始化"（value-initialization）的概念，空初始化器调用的是默认初始化

### C++11 标准

C++11 引入了重要变更：
- 将标量类型和 POD 类型的动态存储期情况重新归类为默认初始化的一种形式
- 引入了"值初始化"概念，空初始化器现在调用值初始化而非默认初始化
- 对于 const 对象的默认初始化规则进行了细化

### C++17 标准

- 引入了 `std::byte` 类型，作为"未初始化友好"（uninitialized-friendly）类型

### C++20 标准

- 在 constexpr 函数中允许平凡默认初始化和 asm 声明（通过 `__cpp_constexpr` 特性测试宏，值 `201907L`）

### C++26 标准（草案）

对不确定值的语义进行了重大修订：
- 引入了"错误值"（erroneous value）概念
- 区分了自动存储期和动态存储期对象的不确定值行为
- 对自动存储期对象的未初始化读取现在构成"错误行为"（erroneous behavior）而非未定义行为

### 缺陷报告历史

| DR 编号 | 应用版本 | 原始行为 | 修正行为 |
|---------|----------|----------|----------|
| CWG 178 | C++98 | 没有值初始化；空初始化器调用默认初始化 | 空初始化器调用值初始化 |
| CWG 253 | C++98 | const 对象的默认初始化不能调用隐式声明的默认构造函数 | 如果所有子对象都被初始化，则允许 |
| CWG 616 | C++98 | 任何未初始化对象的左值到右值转换都是 UB | 不确定的无符号字符值允许读取 |
| CWG 1787 | C++98 | 从寄存器缓存的不确定 unsigned char 读取是 UB | 明确定义为合法行为 |

## 3. 语法与参数 (Syntax and Parameters)

### 语法形式

默认初始化在以下两种语法形式中发生：

| 语法 | 形式 | 说明 |
|------|------|------|
| `T object;` | (1) | 变量声明无初始化器 |
| `new T` | (2) | new 表达式无初始化器 |

### 发生场景

默认初始化在以下三种情况下发生：

1. **无初始化器的变量声明**：声明具有自动、静态或线程存储期的变量时未提供初始化器
2. **无初始化器的 new 表达式**：通过 new 表达式创建动态存储期对象时未提供初始化器
3. **构造函数初始化列表遗漏**：基类或非静态数据成员未在构造函数初始化列表中提及

### 默认初始化的效果

根据类型 `T` 的不同，默认初始化的效果如下：

| 类型 | 效果 |
|------|------|
| （可能 cv 限定的）类类型 | 调用默认构造函数（通过空参数列表的重载决议选择） |
| 数组类型 | 对每个元素进行默认初始化 |
| 其他类型（标量类型等） | **不执行任何初始化**，对象保持不确定值 |

### const 对象的默认初始化

如果程序需要对 const 限定类型 `T` 的对象进行默认初始化，则 `T` 必须是**const-默认可构造**（const-default-constructible）的类类型或其数组。

一个类类型 `T` 是 const-默认可构造的，当且仅当：

**C++11 之前**：
- `T` 的每个直接非静态数据成员 `M` 都是类类型 `X`（或其数组），且 `X` 是 const-默认可构造的
- `T` 没有直接的可变成员（variant members）

**C++11 起**：
- 默认初始化 `T` 会调用用户提供的构造函数（不从基类继承），或
- `T` 的每个直接非变体非静态数据成员 `M` 有默认成员初始化器，或如果 `M` 是类类型 `X`（或其数组），`X` 是 const-默认可构造的
- 如果 `T` 是有至少一个非静态数据成员的联合体，恰好一个可变成员有默认成员初始化器
- 如果 `T` 不是联合体，对于每个有非静态数据成员的匿名联合体成员，恰好一个非静态数据成员有默认成员初始化器
- 每个潜在构造的基类都是 const-默认可构造的

## 4. 底层原理 (Underlying Principles)

### 不确定值与错误值

当获取具有自动或动态存储期的对象存储空间时，对象的初始状态取决于 C++ 版本：

**C++26 之前**：
- 对象具有**不确定值**（indeterminate value）
- 如果未执行初始化，对象保持不确定值直到被替换

**C++26 起**：
- **动态存储期对象**或标记为 `[[indeterminate]]` 的变量：字节具有**不确定值**
- **自动存储期对象**：字节具有**错误值**（erroneous value）

值的分类规则：
- 如果值表示中的任何位具有不确定值，则对象具有不确定值
- 否则，如果值表示中的任何位具有错误值，则对象具有错误值

### 行为后果

| 值类型 | 使用后果 |
|--------|----------|
| 不确定值（任何类型） | **未定义行为**（Undefined Behavior） |
| 错误值（C++26 起） | **错误行为**（Erroneous Behavior） |

### 未初始化友好类型

以下类型被称为**未初始化友好**（uninitialized-friendly）类型：
- `unsigned char`
- `char`（当其底层类型为 `unsigned char` 时）
- `std::byte`（C++17 起）

对于未初始化友好类型，在特定操作中使用不确定值或错误值是**明确定义**的：

1. **条件表达式的第二或第三操作数**
2. **逗号表达式的右操作数**
3. **整型转换、显式转换或 `static_cast` 到未初始化友好类型**
4. **弃值表达式**
5. **简单赋值运算符的右操作数**（左操作数是未初始化友好类型的左值）
6. **初始化未初始化友好类型的对象**

## 5. 使用场景 (Use Cases)

### 适用场景

| 场景 | 说明 |
|------|------|
| 类类型对象 | 需要调用默认构造函数完成初始化 |
| 性能敏感代码 | 故意延迟初始化，后续通过其他方式赋值 |
| 容器预分配 | 使用 `reserve()` 后延迟构造元素 |

### 不适用场景

| 场景 | 原因 | 推荐 |
|------|------|------|
| 标量类型变量 | 未初始化导致不确定值，可能引发 UB | 显式初始化为 0 或其他适当值 |
| const 标量对象 | 编译错误：const 标量必须初始化 | 必须提供初始化器 |
| 引用 | 编译错误：引用必须初始化 | 必须绑定到对象 |

### 最佳实践

1. **总是初始化标量类型**：避免使用不确定值
2. **使用类类型封装资源**：通过默认构造函数确保资源正确初始化
3. **谨慎使用未初始化变量**：仅在性能关键路径且确保后续赋值的情况下使用

### 常见陷阱

#### 陷阱 1：未初始化的局部变量

```cpp
int main() {
    int x;           // 不确定值！
    if (x > 0) { ... }  // 未定义行为
}
```

#### 陷阱 2：const 对象默认初始化失败

```cpp
struct T1 { int mem; };  // 隐式默认构造函数

int main() {
    const T1 t1;  // 编译错误！const 对象需要用户提供的构造函数或默认成员初始化器
}
```

#### 陷阱 3：数组元素未初始化

```cpp
int main() {
    int arr[10];  // 所有元素具有不确定值
    // 必须在使用前显式初始化每个元素
}
```

## 6. 代码示例 (Examples)

### 基础用法

```cpp
#include <string>
#include <iostream>

struct T1 {
    int mem;
};

struct T2 {
    int mem;
    T2() {}  // "mem" 不在初始化列表中
};

int n;  // 静态非类类型
       // 1) 零初始化将 n 初始化为零
       // 2) 默认初始化不做任何事，n 保持为零

int main() {
    int n;              // 非类类型，值是不确定的
    std::string s;      // 类类型，调用默认构造函数，值为 ""
    std::string a[2];   // 数组，默认初始化每个元素，值为 {"", ""}

    // int& r;          // 错误：引用必须初始化
    // const int n;     // 错误：const 非类类型必须初始化
    // const T1 t1;     // 错误：const 类类型且为隐式默认构造函数

    T1 t1;              // 类类型，调用隐式默认构造函数
    const T2 t2;        // const 类类型，调用用户提供的默认构造函数
                        // t2.mem 被默认初始化（不确定值）

    std::cout << "::n = " << ::n << std::endl;  // 输出: 0
    return 0;
}
```

### 不确定值的危险行为

```cpp
#include <iostream>

// 情况 1: 动态存储期的未初始化对象
int f(bool b) {
    unsigned char* c = new unsigned char;
    unsigned char d = *c;  // OK，"d" 具有不确定值
    int e = d;             // 未定义行为！
    return b ? d : 0;      // 如果 b 为 true，未定义行为
}

// 情况 2: 自动存储期的未初始化对象（C++26 前）
int g(bool b) {
    unsigned char c;       // "c" 具有不确定/错误值

    unsigned char d = c;   // 无 UB/错误行为，但 "d" 具有不确定/错误值

    // assert(c == d);     // 成立，但两个整型提升都有 UB/错误行为

    int e = d;             // UB/错误行为
    return b ? d : 0;      // 如果 b 为 true，UB/错误行为
}

// 情况 3: 使用未初始化友好类型的安全操作
void safe_operations() {
    unsigned char c;       // 不确定值

    // 安全：转换为另一个未初始化友好类型
    unsigned char d = static_cast<unsigned char>(c);

    // 安全：赋值给另一个未初始化友好类型
    unsigned char e;
    e = c;

    // 安全：在条件表达式中使用
    unsigned char f = true ? c : 0;
}

int main() {
    std::cout << "f(true) = " << f(true) << std::endl;  // 可能输出任何值
    return 0;
}
```

### 常见错误及修正

#### 错误 1：未初始化的局部变量

```cpp
// 错误：使用未初始化的变量
void bad_example() {
    int x;           // 不确定值
    if (x > 0) {     // 未定义行为
        // ...
    }
}

// 正确：显式初始化
void good_example() {
    int x = 0;       // 明确初始化为零
    if (x > 0) {     // 定义明确的行为
        // ...
    }
}
```

#### 错误 2：const 对象默认初始化失败

```cpp
// 错误：const 类类型对象无法默认初始化
struct Bad {
    int value;
};

void bad_example() {
    // const Bad b;  // 编译错误
}

// 正确方案 1：提供默认构造函数
struct Good1 {
    int value = 0;   // 默认成员初始化器
};

void good_example1() {
    const Good1 g;   // OK
}

// 正确方案 2：用户提供的构造函数
struct Good2 {
    int value;
    Good2() : value(0) {}  // 用户提供的默认构造函数
};

void good_example2() {
    const Good2 g;   // OK
}
```

#### 错误 3：new 表达式未初始化

```cpp
// 错误：动态分配的对象未初始化
void bad_example() {
    int* p = new int;      // 不确定值
    std::cout << *p;       // 未定义行为
    delete p;
}

// 正确：使用值初始化
void good_example() {
    int* p = new int();    // 值初始化，*p == 0
    // 或
    int* p2 = new int{};   // C++11 列表初始化，*p2 == 0
    std::cout << *p;       // 安全
    delete p;
    delete p2;
}
```

## 7. 总结 (Summary)

### 核心要点

| 要点 | 说明 |
|------|------|
| 定义 | 无初始化器时执行的初始化行为 |
| 类类型 | 调用默认构造函数 |
| 数组类型 | 递归默认初始化每个元素 |
| 标量类型 | **不执行初始化**，保持不确定值 |
| 引用/const 标量 | **编译错误**，必须提供初始化器 |

### 类型行为对比

| 类型 | 默认初始化行为 | 结果 |
|------|---------------|------|
| 类类型 | 调用默认构造函数 | 取决于构造函数 |
| 数组类型 | 元素逐个默认初始化 | 取决于元素类型 |
| 整型/浮点型 | 无初始化 | 不确定值 |
| 指针 | 无初始化 | 不确定值 |
| 引用 | **编译错误** | 必须初始化 |
| const 标量 | **编译错误** | 必须初始化 |

### 学习建议

1. **始终初始化变量**：养成声明时即初始化的习惯，避免不确定值
2. **理解类类型与标量类型的区别**：类类型会调用构造函数，标量类型不执行任何操作
3. **注意 const 对象的特殊要求**：const 对象的默认初始化需要满足 const-默认可构造条件
4. **避免读取不确定值**：除未初始化友好类型外，读取不确定值会导致 UB
5. **区分默认初始化与值初始化**：`T obj;`（默认初始化）vs `T obj{};`（值初始化）

### 相关概念

| 概念 | 关系 |
|------|------|
| 值初始化 (Value-initialization) | 空初始化器调用，通常进行零初始化后再默认初始化 |
| 零初始化 (Zero-initialization) | 将存储设置为全零 |
| 直接初始化 (Direct-initialization) | 使用显式构造函数调用或括号语法 |
| 复制初始化 (Copy-initialization) | 使用等号语法 |
| 列表初始化 (List-initialization) | 使用花括号语法 |
