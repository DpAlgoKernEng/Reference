# 冲突声明 (Conflicting Declarations)

## 1. 概述

**冲突声明 (Conflicting Declarations)** 是 C++ 语言中关于声明实体唯一性的重要规则。除非另有规定，两个声明不能（重新）引入同一个实体（entity）。如果存在此类声明，程序是**非良构 (ill-formed)** 的。

这一机制确保了程序中每个实体在特定作用域内的唯一性，防止命名冲突和语义歧义。理解冲突声明规则对于正确组织代码结构、避免编译错误至关重要。这些规则涉及：
- **对应声明 (Corresponding Declarations)**：判断两个声明是否对应
- **同一实体的多重声明**：判断两个声明是否声明同一实体
- **潜在冲突声明 (Potentially-conflicting Declarations)**：判断两个声明是否潜在冲突

## 2. 来源与演变

### 历史背景

C++ 语言的声明系统设计需要在支持重载（overloading）和防止命名冲突之间取得平衡。冲突声明规则明确了何时两个声明被视为声明同一实体，何时被视为冲突。

### 版本演进

| 版本 | 变更内容 |
|------|----------|
| **C++98** | 基础规则建立，定义了声明对应和实体声明的判断标准 |
| **C++11** | 新增别名模板（alias template）的重声明规则 |
| **C++14** | 新增变量模板（variable template）的重声明规则 |
| **C++20** | 新增 concept 的重声明规则；新增 requires 子句的等效性判断；新增模块链接支持 |
| **C++23** | 新增显式对象参数（explicit object parameter，Deducing this）的相关规则 |
| **C++26** | 新增名称无关声明（name-independent declaration）概念，允许使用 `_` 的特殊规则 |

### 缺陷报告

| DR | 应用版本 | 原有行为 | 修正行为 |
|----|----------|----------|----------|
| CWG 279 | C++98 | 不清楚具有链接用途 typedef 名称的无名类是否可以重声明 | 可以重声明 |
| CWG 338 | C++98 | 不清楚具有枚举器作为链接名称的无名枚举是否可以重声明 | 可以重声明 |
| CWG 1884 | C++98 | 同一实体的多重声明限制不明确 | 明确化 |

## 3. 语法与参数

### 对应声明 (Corresponding Declarations)

两个声明**对应 (correspond)** 的条件：它们（重新）引入相同的名称，或都声明构造函数，或都声明析构函数，**除非满足以下例外条件之一**：

| 例外条件 | 说明 |
|----------|------|
| using 声明 | 任一声明是 using 声明 |
| 类型与变量冲突 | 一个声明类型（非 typedef 名称），另一个声明变量、非静态数据成员（匿名联合除外）、枚举器、函数或函数模板 |
| 非对应重载 | 各自声明函数或函数模板，但不声明对应重载 |

### 对应函数重载 (Corresponding Function Overloads)

两个函数声明声明**对应重载**需满足以下所有条件：

| 条件 | 版本要求 |
|------|----------|
| 相同的参数类型列表（省略显式对象参数的类型） | 基础要求，C++23 起省略显式对象参数 |
| 等效的尾随 requires 子句（如果有，友元声明除外） | C++20 起 |

**非静态成员函数的额外条件**，需满足以下条件之一：

| 条件 | 版本 |
|------|------|
| 恰好一个是无 ref-qualifier 的隐式对象成员函数，且去除顶层引用后对象参数类型相同 | C++23 起 |
| 它们的对象参数具有相同类型 | 备选条件 |

### 对应函数模板重载 (Corresponding Function Template Overloads)

两个函数模板声明声明**对应重载**需满足以下所有条件：

| 条件 | 版本要求 |
|------|----------|
| 模板参数列表长度相同 | 基础要求 |
| 对应模板参数等效 | 基础要求 |
| 等效的参数类型列表（省略显式对象参数的类型） | 基础要求，C++23 起省略显式对象参数 |
| 等效的返回类型 | 基础要求 |
| 对应模板参数都无约束或都有等效约束 | C++20 起 |
| 等效的尾随 requires 子句（如果有） | C++20 起 |

**非静态成员函数模板的额外条件**，需满足以下条件之一：

| 条件 | 版本 |
|------|------|
| 恰好一个是无 ref-qualifier 的隐式对象成员函数模板，且去除所有引用后对象参数类型等效 | C++23 起 |
| 它们的对象参数具有等效类型 | 备选条件 |

### 名称无关声明 (Name-Independent Declaration) [C++26]

一个声明是**名称无关**的，如果其名称为 `_` 且声明以下内容之一：

- 具有自动存储期的变量
- 无存储类说明符的结构化绑定且不在命名空间作用域
- lambda 捕获引入的带初始化器的变量
- 非匿名联合的非静态数据成员

## 4. 底层原理

### 声明实体的同一性判断

两个声明声明**同一实体**，当满足以下所有条件时（考虑未命名类型的 typedef 名称和枚举名称用于链接目的）：

```
条件 1: 它们对应 (correspond)
    AND
条件 2: 具有相同的目标作用域（非函数参数作用域或模板参数作用域）
    AND
条件 3 [C++26]: 都不是名称无关声明
    AND
条件 4: 以下任一条件成立
    - 它们出现在同一翻译单元
    - 它们都声明具有模块链接的名称且附加到同一模块 [C++20]
    - 它们都声明具有外部链接的名称
```

### 重声明定义

实体或 typedef 名称 `X` 的声明是 `X` 的**重声明 (redeclaration)**，如果另一个 `X` 的声明可以从它到达。

### 重声明的限制

如果实体 `E` 的任意两个声明违反以下对应限制，程序是非良构的：

| 实体类型 | 重声明限制 |
|----------|------------|
| 变量 | 必须声明为相同类型的变量 |
| 函数 | 必须声明为相同类型的函数 |
| 枚举器 | 必须声明为枚举器 |
| 命名空间 | 必须声明为命名空间 |
| 类类型 | 必须声明为类类型 |
| 枚举类型 | 必须声明为枚举类型 |
| 类模板 | 必须声明为具有等效模板参数列表的类模板 |
| 函数模板 | 必须声明为具有等效模板参数列表和类型的函数模板 |
| 别名模板 [C++11] | 必须声明为具有等效模板参数列表和 type-id 的别名模板 |
| 变量模板 [C++14] | 必须声明为具有等效模板参数列表和类型的变量模板（或其部分特化） |
| concept [C++20] | 必须声明为 concept |

### 类型比较规则

类型在所有类型调整之后进行比较（期间 typedef 被其定义替换）。

对于数组对象的声明，可以指定仅在主数组界限存在或缺失上不同的数组类型。

**注意**：如果两个声明互不可达，则不需要诊断。

### 跨翻译单元规则

如果一个声明 `H` 声明了一个具有内部链接的名称，并且在另一个翻译单元 `U` 中的声明 `D` 之前出现，且如果 `H` 出现在 `U` 中会声明与 `D` 相同的实体，则程序是非良构的。

## 5. 使用场景

### 允许的重声明场景

| 场景 | 示例 |
|------|------|
| 函数重声明（相同签名） | `void f(); void f() {}` |
| 友元声明定义 | 在不同类中声明同一友元函数 |
| typedef 别名 | `typedef int Int; void f(int); void f(Int) {}` |
| 命名空间别名重复 | `namespace B = A; namespace B = A;` |

### 禁止的冲突声明场景

| 场景 | 错误原因 |
|------|----------|
| 同名不同类型 | `void g(); int g();` - 返回类型不同 |
| 同名不同实体类型 | `void h(); namespace h {}` - 函数与命名空间冲突 |
| 成员函数重复声明 | 静态与非静态成员函数签名相同时 |
| 潜在冲突 | 两个声明对应但声明不同实体 |

### 最佳实践

1. **避免同名不同实体**：确保同一作用域内同名声明声明同一实体
2. **注意重载条件**：理解函数重载和函数模板重载的对应规则
3. **利用 `_` 变量 (C++26)**：使用下划线作为名称无关声明的特殊名称
4. **模块化设计**：合理使用命名空间避免命名冲突

## 6. 代码示例

### 基础用法：友元声明对应

```cpp
// 友元声明可以在不同类中声明同一函数
struct A {
    friend void c();   // #1 声明
};

struct B {
    friend void c() {} // 对应 #1，并定义它
};

// typedef 别名不创建新类型
typedef int Int;

enum E : int { a };

void f(int);   // #2
void f(Int) {} // 定义 #2（Int 是 int 的别名，声明同一函数）
void f(E) {}   // OK，另一个重载（参数类型不同）

int main() {
    f(42);  // 调用 f(int)
    f(a);   // 调用 f(E)
}
```

### 成员函数重声明规则

```cpp
struct X {
    static void f();
    void f() const;   // 错误：重声明，签名冲突
    // 静态成员函数和 const 成员函数对应但类型不同

    void g();
    void g() const;   // OK，不对应（const 使其不对应）
    void g() &;       // 错误：重声明，与 void g() const 对应

    // C++23: 显式对象参数
    void h(this X&, int);
    void h(int) &&;   // OK，另一个重载

    void j(this const X&);
    void j() const &; // 错误：重声明

    void k();
    void k(this X&);  // 错误：重声明
};
```

### 常见错误：冲突声明

```cpp
// 错误示例 1：同名不同类型
void g();      // #1
void g(int);   // OK，与 #1 是不同实体（不对应）
int g();       // 错误：与 #1 同一实体但类型不同

// 错误示例 2：同名不同实体类型
void h();      // #2
namespace h {} // 错误：与 #2 同一实体但不是函数

// 错误示例 3：潜在冲突
void f() {
    int x, y;
    void x(); // 错误：x 已声明为 int，现在声明为函数
    int y;    // 错误：y 重定义
}
```

### 名称无关声明 (C++26)

```cpp
// C++26 起：使用 _ 的名称无关声明
void g() {
    int _;
    _ = 0; // OK
    int _; // C++26 起 OK，名称无关声明
    _ = 0; // 错误：查找集中有两个非函数声明
}

void h() {
    int _;        // #1
    _++;          // OK
    static int _; // 错误：与 #1 冲突
                  // 静态变量不是名称无关声明
}

// 命名空间别名可以重复
namespace A {}
namespace B = A;
namespace B = A; // OK，无效果
namespace B = B; // OK，无效果
namespace A = B; // OK，无效果
namespace B {};  // 错误：B 已绑定到 A，不同实体
```

### 枚举器与全局命名冲突

```cpp
void f();  // 全局函数 f

enum { f }; // 错误：f 是不同实体
// 枚举器 f 与全局函数 f 冲突
```

### 正确用法与错误对比

```cpp
// 正确：函数重载
void process(int);
void process(double);
void process(const std::string&);

// 错误：仅返回类型不同
int compute(int);
double compute(int);  // 错误！

// 正确：命名空间别名
namespace MyNS = std;
namespace MyNS = std;  // OK，重复的别名定义

// 错误：别名指向不同命名空间
namespace OtherNS = std;
namespace OtherNS = boost;  // 错误！

// 正确：模板声明
template<typename T>
void func(T);

template<typename T>
void func(T*);  // OK，不同重载

// 错误：模板重声明类型不同
template<typename T>
int func2(T);

template<typename T>
void func2(T);  // 错误！返回类型不同
```

## 7. 总结

### 核心要点

冲突声明规则是 C++ 类型系统和链接系统的基石，确保了：

| 要点 | 说明 |
|------|------|
| **实体唯一性** | 每个实体在特定作用域内有唯一标识 |
| **重载区分** | 明确区分重载（不同实体）与重声明（同一实体） |
| **链接一致性** | 外部链接和模块链接的声明合并规则 |

### 关键概念对照表

| 概念 | 英文 | 说明 |
|------|------|------|
| 对应声明 | Corresponding declarations | 声明同一名称的声明 |
| 对应重载 | Corresponding overloads | 具有相同签名的函数/函数模板 |
| 重声明 | Redeclaration | 对同一实体的再次声明 |
| 潜在冲突 | Potentially conflict | 对应但声明不同实体 |
| 名称无关声明 | Name-independent declaration | C++26 起允许使用 `_` 的特殊声明 |

### 技术对比

| 场景 | 是否对应 | 是否同一实体 | 结果 |
|------|----------|--------------|------|
| 相同签名的两个函数声明 | 是 | 是 | 允许（重声明） |
| 不同签名的两个函数声明 | 否 | 否 | 允许（重载） |
| 函数与命名空间同名 | 是 | 是 | 禁止（类型冲突） |
| 静态成员与非静态成员同名 | 取决于签名 | 取决于签名 | 可能禁止 |

### 学习建议

1. **理解"对应"概念**：掌握声明对应的判断规则是理解冲突声明的基础
2. **区分重载与重声明**：重载是不同实体，重声明是同一实体
3. **关注版本更新**：C++20/23/26 的变化影响显式对象参数和名称无关声明
4. **实践调试**：编译器错误信息通常能帮助定位冲突声明问题
