# 数组声明 (Array Declaration)

## 1. 概述

数组声明用于声明数组类型的对象。数组是一种连续存储多个相同类型元素的数据结构，是 C++ 中最基础的数据类型之一。

数组类型定义了包含 N 个连续分配的 T 类型对象，元素编号从 0 到 N-1，可通过下标运算符 `[]` 进行访问。数组可由任意基本类型（除 void 外）、指针、成员指针、类、枚举或其他已知边界的数组构造而成。

关键特性：
- **连续内存布局**：元素在内存中连续存储
- **编译时大小确定**：数组大小必须是编译时常量表达式（C++14 起为 `std::size_t` 类型的转换常量表达式）
- **不可整体赋值**：数组对象虽为左值，但不能直接赋值
- **自动退化**：数组可隐式转换为指向首元素的指针

## 2. 来源与演变

### 历史背景

数组概念源于 C 语言，在 C++ 中得到了继承和扩展。C 语言中的数组是构建其他数据结构的基础，C++ 延续了这一设计理念，同时增加了类型安全性和一些新特性。

### C++ 标准演变

| 版本 | 变化内容 |
|------|----------|
| **C++98** | 基础数组声明语法，继承自 C 语言 |
| **C++11** | 新增属性（attribute）支持；`auto` 可用于数组元素类型的推导 |
| **C++14** | 数组大小表达式要求变更为 `std::size_t` 类型的转换常量表达式 |
| **C++17** | 数组纯右值（prvalue）可通过临时量具体化转换为亡值（xvalue） |
| **C++20** | 允许从已知边界数组初始化/赋值给未知边界数组的引用和指针 |

### 缺陷报告修正

| DR | 适用版本 | 原行为 | 修正后行为 |
|------|----------|--------|------------|
| CWG 393 | C++98 | 未知边界数组的指针或引用不能作为函数参数 | 允许使用 |
| CWG 619 | C++98 | 省略的数组边界不能从先前声明推断 | 允许推断 |
| CWG 2099 | C++98 | 即使有初始化器，静态数据成员的数组边界也不能省略 | 允许省略 |
| CWG 2397 | C++11 | `auto` 不能用作元素类型 | 允许使用 |

## 3. 语法与参数

### 基本语法

```
noptr-declarator `[` expr(可选) `]` attr(可选)
```

### 参数说明

| 参数 | 说明 |
|------|------|
| `noptr-declarator` | 任何有效的声明器。如果以 `*`、`&` 或 `&&` 开头，必须用括号括起，否则整个声明器会被视为指针或引用声明器 |
| `expr` | 整型常量表达式（C++14 前）/ `std::size_t` 类型的转换常量表达式（C++14 起），求值结果必须大于零 |
| `attr` | 属性列表（C++11 起） |

### 声明形式

```cpp
T a[N];    // 声明包含 N 个 T 类型元素的数组
T a[];     // 声明未知边界的数组（不完整类型）
T a[] = {...};  // 使用初始化器推断边界
```

### cv-限定符传播

对数组类型应用 cv-限定符时，限定符会应用到元素类型上：

```cpp
typedef const char CC;
CC a[5] = {};       // 类型为 "5 个 const char 的数组"

typedef char CA[5];
const CA b = {};     // 类型同样为 "5 个 const char 的数组"
```

### 禁止的类型

- **引用数组**：不存在引用的数组
- **函数数组**：不存在函数的数组
- **未知边界数组元素**：多维数组除第一维外不能有未知边界

## 4. 底层原理

### 内存布局

数组在内存中采用连续存储：

```
声明：int a[5];
内存：[a[0]][a[1]][a[2]][a[3]][a[4]]
地址：  +0    +4    +8    +12   +16  (相对于数组首地址)
```

### 数组到指针的退化

存在从数组类型的左值/右值到指针类型右值的隐式转换，生成指向数组首元素的指针：

```cpp
int a[3] = {1, 2, 3};
int* p = a;          // a 退化为 &a[0]

sizeof(a);           // 返回数组大小（12 字节）
sizeof(p);           // 返回指针大小（4 或 8 字节）
```

退化发生时机：
- 数组出现在期望指针而非数组的上下文中
- 退化只发生一次（对多维数组）

### 多维数组原理

多维数组本质上是"数组的数组"：

```cpp
int a[2][3];        // 包含 2 个元素的数组，每个元素是 int[3]
// 内存布局：[a[0][0]][a[0][1]][a[0][2]][a[1][0]][a[1][1]][a[1][2]]
// 按行主序（row-major）存储

int (*p)[3] = a;    // a 退化为指向首行（第一个 int[3]）的指针
// 而非 int** 类型！
```

### 性能特征

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 元素访问 `a[i]` | O(1) | 直接地址计算 |
| 遍历 | O(n) | 连续内存，缓存友好 |
| 复制 | O(n) | 需要逐元素复制 |
| 取地址 | O(1) | 数组本身是左值 |

## 5. 使用场景

### 适用场景

| 场景 | 说明 |
|------|------|
| 固定大小数据集 | 元素数量编译时已知 |
| 高性能计算 | 连续内存布局，缓存友好 |
| 与 C 接口交互 | 原生数组与 C 完全兼容 |
| 嵌入式开发 | 无动态内存分配开销 |
| 常量表 | 查找表、配置数据等 |

### 不适用场景

| 场景 | 推荐替代 |
|------|---------|
| 大小动态变化 | `std::vector` |
| 需要边界检查 | `std::array` + `at()` 或 `std::span` |
| 频繁插入删除 | `std::vector` 或 `std::list` |
| 需要复制语义 | `std::array` |

### 注意事项

1. **数组不可整体赋值**
   ```cpp
   int a[3] = {1, 2, 3}, b[3] = {4, 5, 6};
   // a = b;  // 错误！数组不能整体赋值
   ```

2. **数组退化为指针时丢失大小信息**
   ```cpp
   void f(int arr[]) {  // 实际上等价于 void f(int* arr)
       sizeof(arr);      // 返回指针大小，而非数组大小
   }
   ```

3. **new[] 可创建零大小数组**
   ```cpp
   int* p = new int[0];  // 合法，但访问 p[0] 是未定义行为
   delete[] p;           // 仍需释放
   ```

4. **结构体中的数组可以复制**
   ```cpp
   struct S { int arr[3]; };
   S s1 = {{1, 2, 3}}, s2;
   s2 = s1;  // 正确：隐式定义的复制赋值运算符可处理数组成员
   ```

### 常见陷阱

1. **指针与多维数组混淆**
   ```cpp
   int a[2][3];
   // int** p = a;        // 错误！
   int (*p)[3] = a;       // 正确
   ```

2. **数组参数声明误解**
   ```cpp
   void f(int a[10]);     // 看似接收 10 个元素的数组
   // 实际上等价于 void f(int* a);
   ```

3. **未知边界数组限制**
   ```cpp
   extern int a[][2];     // 正确
   // extern int b[2][];  // 错误：只有第一维可以未知
   ```

## 6. 代码示例

### 基础用法

```cpp
#include <iostream>

int main() {
    // 一维数组
    int a[5] = {1, 2, 3, 4, 5};

    // 访问元素
    std::cout << "First: " << a[0] << std::endl;     // 1
    std::cout << "Last: " << a[4] << std::endl;       // 5

    // 使用下标遍历
    for (int i = 0; i < 5; ++i) {
        std::cout << a[i] << " ";
    }

    // 使用范围 for 循环（C++11）
    for (int x : a) {
        std::cout << x << " ";
    }

    return 0;
}
```

### 多维数组

```cpp
#include <iostream>

int main() {
    // 二维数组（矩阵）
    int matrix[2][3] = {
        {1, 2, 3},  // 第一行
        {4, 5, 6}   // 第二行
    };

    // 访问元素
    std::cout << matrix[0][1] << std::endl;  // 2
    std::cout << matrix[1][2] << std::endl;  // 6

    // 遍历
    for (int i = 0; i < 2; ++i) {
        for (int j = 0; j < 3; ++j) {
            std::cout << matrix[i][j] << " ";
        }
        std::cout << std::endl;
    }

    return 0;
}
```

### 数组与指针

```cpp
#include <iostream>
#include <iterator>
#include <numeric>

void g(int (&a)[3]) {
    std::cout << a[0] << std::endl;
}

void f(int* p) {
    std::cout << *p << std::endl;
}

int main() {
    int a[3] = {1, 2, 3};
    int* p = a;  // 数组退化为指针

    // sizeof 区别
    std::cout << sizeof a << std::endl   // 数组大小：12
              << sizeof p << std::endl;  // 指针大小：8（64位）

    // 需要数组类型时，只有数组可用
    g(a);     // 正确：函数接受数组引用
    // g(p);  // 错误：指针不能绑定到数组引用

    // 范围 for 需要数组类型
    for (int n : a) {
        std::cout << n << " ";
    }
    // for (int n : p) { }  // 错误

    // 需要指针时，数组和指针都可用
    f(a);  // 正确：数组退化为指针
    f(p);  // 正确

    return 0;
}
```

### 未知边界数组

```cpp
// 声明（不完整类型）
extern int x[];            // 未知边界的 int 数组

// 定义时推断边界
int a[] = {1, 2, 3};      // 类型为 int[3]

// 多维数组：只有第一维可以未知
extern int matrix[][3];   // 正确：未知行数，每行 3 个 int
// extern int matrix2[3][];  // 错误

// 从先前声明继承边界
extern int arr[10];
int arr[];                // 正确：边界为 10

// 类静态成员
struct S {
    static int data[10];
};
int S::data[];            // 正确：边界为 10

// C++20 起：未知边界数组的引用和指针
int known[] = {1, 2, 3};
int (&r)[] = known;       // C++20 起正确
int (*p)[] = &known;      // C++20 起正确
```

### 数组右值（C++17）

```cpp
#include <iostream>
#include <type_traits>
#include <utility>

void f(int (&&x)[2][3]) {
    std::cout << sizeof x << std::endl;  // 24
}

struct X {
    int i[2][3];
};

template<typename T>
using identity = T;

int main() {
    // 通过类型别名创建数组临时对象
    using arr_t = int[2][3];
    f(arr_t{});  // 数组纯右值绑定到右值引用

    // 通过模板别名
    f(identity<int[][3]>{{1, 2, 3}, {4, 5, 6}});

    // 访问类右值的数组成员（产生亡值）
    std::cout << sizeof X().i << std::endl;  // 24
    f(X().i);  // 正确：绑定到亡值

    // 使用 std::move
    int a[2][3];
    f(std::move(a));  // 正确：绑定到亡值

    return 0;
}
```

### 常见错误及修正

#### 错误 1：数组整体赋值

```cpp
// 错误
int a[3] = {1, 2, 3}, b[3] = {4, 5, 6};
// a = b;  // 错误：数组不能整体赋值

// 修正 1：使用 std::copy 或循环
#include <algorithm>
std::copy(std::begin(b), std::end(b), std::begin(a));

// 修正 2：使用 std::array
#include <array>
std::array<int, 3> a = {1, 2, 3}, b = {4, 5, 6};
a = b;  // 正确

// 修正 3：包装在结构体中
struct IntArray3 { int data[3]; };
IntArray3 c = {{1, 2, 3}}, d = {{4, 5, 6}};
c = d;  // 正确
```

#### 错误 2：多维数组指针类型错误

```cpp
int a[2][3];

// 错误
// int** p = a;  // 错误：二维数组不退化为 int**

// 正确
int (*p)[3] = a;  // 指向包含 3 个 int 的数组的指针

// 错误理解
// a[0] 是 int* 类型？不，a[0] 是 int[3] 类型
// a[0] 退化后是 int* 类型
```

#### 错误 3：数组参数声明误解

```cpp
// 以下三种声明完全等价！
void f(int arr[10]);
void f(int arr[]);
void f(int* arr);

// 如果需要保留大小信息
template<size_t N>
void f(int (&arr)[N]) {
    std::cout << "Size: " << N << std::endl;
}

// 或使用 std::array
void f(std::array<int, 10>& arr);
```

## 7. 总结

数组是 C++ 中最基础的数据结构，具有以下核心特性：

| 特性 | 说明 |
|------|------|
| 内存布局 | 连续存储，缓存友好 |
| 大小 | 编译时确定（静态数组） |
| 访问效率 | O(1) 随机访问 |
| 复制 | 不支持整体复制（除非包装在结构体中） |
| 退化 | 可隐式转换为指向首元素的指针 |

### 与 std::array 对比

| 特性 | 原生数组 | std::array |
|------|---------|------------|
| 整体赋值 | 不支持 | 支持 |
| 获取大小 | `sizeof(a)/sizeof(a[0])` | `a.size()` |
| 边界检查 | 无 | `at()` 方法 |
| 迭代器 | 无 | 有 |
| 与 C 兼容 | 完全 | `data()` 方法 |

### 学习建议

1. **理解数组退化**：这是理解 C/C++ 指针与数组关系的关键
2. **优先使用 std::array**：在 C++ 中，除非需要与 C 接口交互，否则优先使用 `std::array`
3. **注意多维数组的指针类型**：`int a[2][3]` 退化为 `int (*)[3]`，而非 `int**`
4. **掌握未知边界数组**：理解不完整类型的用途和限制
