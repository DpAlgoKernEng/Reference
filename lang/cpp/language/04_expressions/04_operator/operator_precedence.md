# C++ 运算符优先级 (Operator Precedence)

## 1. 概述 (Overview)

运算符优先级（Operator Precedence）是 C++ 语言中定义运算符在表达式求值时绑定顺序的规则。当一个表达式中包含多个运算符时，优先级决定了哪些运算符先与其操作数结合，从而影响表达式的计算顺序和结果。

C++ 共定义了 17 个优先级级别，从最高优先级（第 1 级）到最低优先级（第 17 级）。优先级越高，运算符越先与操作数绑定。同一优先级的运算符按照结合性（Associativity）规定的方向进行绑定。

**关键概念**：
- **优先级（Precedence）**：决定不同运算符之间的绑定顺序
- **结合性（Associativity）**：决定同优先级运算符的绑定方向
- **操作数（Operands）**：参与运算的值或表达式

## 2. 来源与演变 (Origin and Evolution)

### 历史背景

C++ 的运算符优先级规则继承自 C 语言，并在此基础上进行了扩展：

| 版本 | 变更内容 |
|------|----------|
| C++98 | 继承 C 语言优先级规则，添加 `::` 作用域解析运算符 |
| C++11 | 引入 `alignof`、`noexcept`、`sizeof...` 运算符 |
| C++20 | 引入 `<=>` 三向比较运算符（太空船运算符）、`co_await`、`co_yield` |

### 设计动机

运算符优先级规则的设计目的是：
1. 使表达式书写更接近数学直觉（如乘法优先于加法）
2. 减少括号使用，提高代码可读性
3. 与 C 语言保持兼容，便于代码迁移

### C 与 C++ 的差异

在 C 语言中，三目条件运算符的优先级高于赋值运算符；而在 C++ 中，三目条件运算符的优先级低于赋值运算符。这导致某些表达式在两种语言中解析方式不同。

**示例**：
```cpp
// 在 C++ 中解析为：e = ((a < d) ? (a++) : (a = d))
// 在 C 中会因语法或语义约束而编译失败
e = a < d ? a++ : a = d;
```

- **a, b, c**：表示操作数，可以是变量、字面量或表达式
- **type**：表示类型名称
- **结合性**：指示同优先级运算符的计算方向
  - 从左到右（Left-to-right）：先计算左边的运算符
  - 从右到左（Right-to-left）：先计算右边的运算符

## 4. 底层原理 (Underlying Principles)

### 编译时概念

运算符优先级和结合性是**编译时概念**，与**求值顺序（Order of Evaluation）**这一运行时概念无关。编译器根据优先级规则确定表达式的语法结构，但不一定按照该顺序执行计算。

**关键区别**：
- **优先级**：决定运算符如何与操作数绑定（语法层面）
- **求值顺序**：决定操作数在运行时的计算顺序（语义层面）

### 解析规则

当解析表达式时，优先级高的运算符会先与操作数绑定，就像被括号括起来一样。

**示例 1**：
```cpp
std::cout << a & b
// 解析为：(std::cout << a) & b
// 而不是：std::cout << (a & b)
// 因为 << 优先级（第 7 级）高于 &（第 11 级）
```

**示例 2**：
```cpp
*p++
// 解析为：*(p++)
// 而不是：(*p)++
// 因为后缀 ++（第 2 级）优先级高于 *（第 3 级）
```

### 结合性规则

同一优先级的运算符按照结合性方向依次绑定：

**从右到左结合性示例**：
```cpp
a = b = c
// 解析为：a = (b = c)
// 赋值运算符从右到左结合，先执行 b = c
```

**从左到右结合性示例**：
```cpp
a + b - c
// 解析为：(a + b) - c
// 加减运算符从左到右结合，先执行 a + b
```

### 一元运算符的结合性

一元运算符的结合性说明是冗余的（仅为完整性列出）：
- 一元前缀运算符总是从右到左结合：`delete ++*p` 解析为 `delete(++(*p))`
- 一元后缀运算符总是从左到右结合：`a[1][2]++` 解析为 `((a[1])[2])++`

**特殊情况**：成员访问运算符虽然归类为后缀运算符，但其结合性有意义：`a.b++` 解析为 `(a.b)++`，而不是 `a.(b++)`。

### 运算符重载的影响

运算符重载**不影响**优先级和结合性规则。无论运算符是否被重载，其优先级保持不变。

**示例**：
```cpp
std::cout << a ? b : c;
// 解析为：(std::cout << a) ? b : c
// 因为 << 优先级高于 ?:
// 即使 << 被重载为流插入运算符，优先级仍然不变
```

### 特殊说明

1. **sizeof 的类型转换限制**：`sizeof` 的操作数不能是 C 风格类型转换。表达式 `sizeof(int) * p` 被解析为 `(sizeof(int)) * p`，而不是 `sizeof((int)*p)`。

2. **三目条件运算符的中置表达式**：条件运算符中间的表达式（`?` 和 `:` 之间）被视为括号括起来处理，其相对于 `?:` 的优先级被忽略。

## 5. 使用场景 (Use Cases)

### 适用场景

**1. 编写复杂表达式**

理解优先级可以避免不必要的括号，使代码更简洁：

```cpp
// 已知 * 优先级高于 +，无需括号
int result = a + b * c;

// 已知 && 优先级高于 ||，可简化逻辑表达式
if (a || b && c) { /* ... */ }
// 等价于：if (a || (b && c)) { /* ... */ }
```

**2. 解析代码行为**

理解他人代码或调试时，需要准确判断表达式的求值结果：

```cpp
int arr[] = {1, 2, 3};
int *p = arr;
int val = *p++;  // val = 1, p 指向 arr[1]
```

**3. 代码审查**

检查代码中是否存在因优先级理解错误导致的潜在 bug：

```cpp
// 危险：可能期望先计算 & 运算
if (flags & MASK != 0) { /* ... */ }
// 实际解析为：if (flags & (MASK != 0)) { /* ... */ }
// 应改为：if ((flags & MASK) != 0) { /* ... */ }
```

### 最佳实践

1. **有疑问时使用括号**：当优先级不明确时，使用括号明确意图
2. **保持表达式简单**：避免过于复杂的单行表达式
3. **使用代码格式化工具**：现代 IDE 和格式化工具可以帮助识别问题

### 常见陷阱

| 陷阱 | 错误理解 | 正确解析 | 建议写法 |
|------|----------|----------|----------|
| 位运算与比较 | `flags & MASK != 0` | `flags & (MASK != 0)` | `(flags & MASK) != 0` |
| 移位与算术 | `a << 2 + 1` | `a << (2 + 1)` | `(a << 2) + 1` 或 `a << 3` |
| 指针与自增 | `*p++` | `*(p++)` | 明确意图后选择写法 |
| 逗号在函数调用中 | `func(a, b)` | 不是逗号运算符 | 参数分隔符 |
| 赋值与比较 | `a = b == c` | `a = (b == c)` | `(a = b) == c` 若意图如此 |

## 6. 代码示例 (Examples)

### 基础用法：算术运算符优先级

```cpp
#include <iostream>

int main() {
    int a = 2, b = 3, c = 4;

    // 乘法优先级高于加法
    int result1 = a + b * c;  // 解析为 a + (b * c) = 2 + 12 = 14
    std::cout << "a + b * c = " << result1 << std::endl;  // 输出: 14

    // 使用括号改变计算顺序
    int result2 = (a + b) * c;  // (2 + 3) * 4 = 20
    std::cout << "(a + b) * c = " << result2 << std::endl;  // 输出: 20

    return 0;
}
```

### 逻辑运算符优先级

```cpp
#include <iostream>
#include <iostream>

int main() {
    bool a = true, b = false, c = true;

    // && 优先级高于 ||
    bool result1 = a || b && c;  // 解析为 a || (b && c) = true || false = true
    std::cout << std::boolalpha;
    std::cout << "a || b && c = " << result1 << std::endl;  // 输出: true

    // 改变结合顺序
    bool result2 = (a || b) && c;  // (true || false) && true = true
    std::cout << "(a || b) && c = " << result2 << std::endl;  // 输出: true

    return 0;
}
```

### 指针与自增运算符

```cpp
#include <iostream>

int main() {
    int arr[] = {10, 20, 30};
    int *p = arr;

    // 后缀 ++ 优先级高于 *
    int val1 = *p++;  // 解析为 *(p++)，取当前值后 p 自增
    std::cout << "*p++ = " << val1 << std::endl;  // 输出: 10
    std::cout << "*p = " << *p << std::endl;       // 输出: 20

    // 前缀 ++ 与 * 同优先级，从右到左结合
    p = arr;  // 重置指针
    int val2 = *++p;  // 解析为 *(++p)，先自增后取值
    std::cout << "*++p = " << val2 << std::endl;  // 输出: 20

    // 如果想先取值后自增值
    p = arr;  // 重置指针
    int val3 = (*p)++;  // 取 arr[0] 的值后，arr[0] 自增
    std::cout << "(*p)++ = " << val3 << std::endl;  // 输出: 10
    std::cout << "arr[0] = " << arr[0] << std::endl; // 输出: 11

    return 0;
}
```

### 位运算优先级陷阱

```cpp
#include <iostream>

int main() {
    int flags = 0x0F;  // 二进制: 0000 1111
    int mask = 0x05;   // 二进制: 0000 0101

    // 错误写法：比较运算符优先级高于位运算
    // bool result = flags & mask != 0;  // 解析为 flags & (mask != 0)
    // 即 flags & 1 = 0x0F & 0x01 = 1 (非零为 true)

    // 正确写法：使用括号明确意图
    bool result = (flags & mask) != 0;  // (0x0F & 0x05) != 0 -> 5 != 0 -> true
    std::cout << std::boolalpha;
    std::cout << "(flags & mask) != 0 = " << result << std::endl;  // 输出: true

    // 验证中间结果
    std::cout << "flags & mask = " << (flags & mask) << std::endl;  // 输出: 5

    return 0;
}
```

### 赋值运算符结合性

```cpp
#include <iostream>

int main() {
    int a, b, c;

    // 赋值运算符从右到左结合
    a = b = c = 10;  // 解析为 a = (b = (c = 10))
    std::cout << "a = " << a << ", b = " << b << ", c = " << c << std::endl;
    // 输出: a = 10, b = 10, c = 10

    // 链式赋值常用于初始化多个变量
    int x = 0, y = 0, z = 0;
    x = y = z = 100;
    std::cout << "x = " << x << ", y = " << y << ", z = " << z << std::endl;
    // 输出: x = 100, y = 100, z = 100

    return 0;
}
```

### 三目条件运算符优先级

```cpp
#include <iostream>

int main() {
    int a = 5, b = 10, c = 15;

    // 条件运算符优先级较低
    int result1 = a + b > 10 ? c : 0;  // 解析为 ((a + b) > 10) ? c : 0
    std::cout << "result1 = " << result1 << std::endl;  // 输出: 15

    // 赋值与条件运算符
    int result2;
    result2 = a < b ? a : b;  // 解析为 result2 = ((a < b) ? a : b)
    std::cout << "result2 = " << result2 << std::endl;  // 输出: 5

    // 条件运算符中间表达式被视为括号括起
    int x = 1, y = 2, z = 3;
    int result3 = x ? y : z = 4;  // 解析为 x ? y : (z = 4)
    // 因为 x 为 true，返回 y，z 的赋值不执行
    std::cout << "result3 = " << result3 << ", z = " << z << std::endl;
    // 输出: result3 = 2, z = 3

    return 0;
}
```

### 常见错误：移位与算术运算

```cpp
#include <iostream>

int main() {
    int a = 1;

    // 错误理解：期望先移位再加
    // int wrong = a << 2 + 1;  // 解析为 a << (2 + 1) = 1 << 3 = 8

    // 正确写法：
    int right1 = (a << 2) + 1;  // (1 << 2) + 1 = 4 + 1 = 5
    int right2 = a << 2 + 1;    // 1 << 3 = 8（如果确实需要这个结果）

    std::cout << "(a << 2) + 1 = " << right1 << std::endl;  // 输出: 5
    std::cout << "a << 2 + 1 = " << right2 << std::endl;    // 输出: 8

    // 建议：始终使用括号明确意图
    int clear = (a << 2) + 1;  // 清晰表达：先移位再加
    std::cout << "Clear intent: " << clear << std::endl;    // 输出: 5

    return 0;
}
```

### C++20 三向比较运算符

```cpp
#include <iostream>
#include <compare>

int main() {
    int a = 5, b = 10;

    // 三向比较运算符（太空船运算符）优先级为第 8 级
    // 高于相等性运算符（第 10 级）
    auto result = a <=> b;

    if (result < 0) {
        std::cout << "a < b" << std::endl;
    } else if (result > 0) {
        std::cout << "a > b" << std::endl;
    } else {
        std::cout << "a == b" << std::endl;
    }
    // 输出: a < b

    return 0;
}
```

## 7. 总结 (Summary)

### 核心要点

| 要点 | 说明 |
|------|------|
| **优先级层次** | C++ 定义了 17 个优先级级别，优先级数字越小，绑定越紧密 |
| **结合性规则** | 同优先级运算符按结合性方向绑定（从左到右或从右到左） |
| **编译时概念** | 优先级和结合性在编译时确定表达式结构，与运行时求值顺序无关 |
| **重载不影响** | 运算符重载不会改变优先级和结合性规则 |
| **括号优先** | 括号可以覆盖默认优先级，明确表达式意图 |

### 记忆技巧

1. **最高优先级**：作用域解析 `::` 和后缀运算符（`++` `--` `()` `[]` `.` `->`）
2. **最低优先级**：逗号运算符 `,`
3. **一元运算符**：前缀运算符（`++` `--` `+` `-` `!` `~` `*` `&`）优先级较高
4. **算术运算**：乘除模 > 加减
5. **移位运算**：低于算术，高于比较
6. **比较运算**：关系运算符 > 相等性运算符
7. **位运算**：按位与 > 按位异或 > 按位或
8. **逻辑运算**：逻辑与 > 逻辑或
9. **条件与赋值**：三目条件 > 赋值运算符

### 技术对比

| 特性 | C 语言 | C++ |
|------|--------|-----|
| 作用域解析 | 无 | `::`（最高优先级） |
| 三目条件与赋值 | 条件 > 赋值 | 条件 < 赋值 |
| C++20 新增 | - | `<=>`、`co_await`、`co_yield` |
| 成员指针 | 无 | `.*` `->*` |

### 学习建议

1. **不必死记**：理解常见运算符的相对优先级即可，复杂表达式使用括号
2. **使用工具辅助**：现代 IDE 会提示优先级问题，静态分析工具可检测潜在错误
3. **阅读标准文档**：C++ 标准中优先级是从语法规则推导而来，而非直接规定
4. **关注版本更新**：C++20 引入了新的运算符，需要更新优先级知识
