# 自增/自减运算符 (Increment/Decrement Operators)

## 1. 概述 (Overview)

自增/自减运算符（Increment/Decrement Operators）是 C++ 中的一元运算符，用于对操作数的值进行加一或减一操作。这两个运算符分别表示为 `++`（自增）和 `--`（自减），根据运算符相对于操作数的位置，分为前缀形式（prefix）和后缀形式（postfix）两种。

| 运算符名称 | 语法 | 可重载 | 类内定义原型 | 类外定义原型 |
|-----------|------|--------|-------------|-------------|
| 前缀自增 (pre-increment) | `++a` | 是 | `T& T::operator++();` | `T& operator++(T& a);` |
| 前缀自减 (pre-decrement) | `--a` | 是 | `T& T::operator--();` | `T& operator--(T& a);` |
| 后缀自增 (post-increment) | `a++` | 是 | `T T::operator++(int);` | `T operator++(T& a, int);` |
| 后缀自减 (post-decrement) | `a--` | 是 | `T T::operator--(int);` | `T operator--(T& a, int);` |

**核心特性**：
- 前缀版本返回引用，后缀版本返回值
- 后缀版本使用 `int` 类型哑元参数（dummy parameter）区分前缀和后缀
- 内置版本支持算术类型和指针类型

## 2. 来源与演变 (Origin and Evolution)

### 历史背景

自增/自减运算符源自 C 语言，最早出现在 B 语言中。Ken Thompson 在设计 B 语言时引入了 `++` 和 `--` 运算符，原因是 PDP-11 计算机有专门的递增/递减指令，可以直接映射到硬件指令以提高效率。Dennis Ritchie 在设计 C 语言时保留了这些运算符，C++ 继承了这一特性。

### 设计动机

1. **代码简洁性**：提供简洁的语法进行加一/减一操作
2. **硬件映射**：早期计算机有专门的递增/递减硬件指令
3. **指针操作**：在遍历数组时，指针递增非常常见
4. **表达式嵌入**：允许在更大表达式中进行递增/递减操作

### 版本变更

| C++ 版本 | 变更内容 |
|---------|---------|
| C++98 | 初始定义，`bool` 类型支持自增（递增后为 `true`） |
| C++17 | 废弃并移除 `bool` 类型的自增/自减操作；前缀自增对 `bool` 类型变为非法 |
| C++20 | 对 `volatile` 限定的操作数执行自增/自减操作标记为废弃 |

### 缺陷报告

| 缺陷编号 | 应用版本 | 原行为 | 修正行为 |
|---------|---------|--------|---------|
| CWG 2855 | C++98 | 前缀自增/自减应用常规算术转换，但后缀版本未应用 | 后缀版本也应用常规算术转换 |
| CWG 2901 | C++98 | 内置后缀自增/自减未应用左值到右值转换 | 应用左值到右值转换 |

## 3. 语法与参数 (Syntax and Parameters)

### 前缀运算符语法

```
++ expression    // 前缀自增 (pre-increment)
-- expression    // 前缀自减 (pre-decrement)
```

### 后缀运算符语法

```
expression ++    // 后缀自增 (post-increment)
expression --    // 后缀自减 (post-decrement)
```

### 参数说明

**内置运算符适用类型**：
- 算术类型（`bool` 类型有特殊限制）
- 指针类型（指向完整对象类型）

**重载函数参数**：

| 参数位置 | 前缀版本 | 后缀版本 | 说明 |
|---------|---------|---------|------|
| 第一个参数 | `T&` 或 `A&` 或 `P&` | `T&` 或 `A&` 或 `P&` | 要操作的对象 |
| 第二个参数 | 无 | `int`（哑元参数） | 用于区分前缀和后缀版本 |

> **哑元参数说明**：后缀版本的 `int` 参数是哑元参数，用于在函数重载时区分前缀和后缀版本。当用户定义的后缀运算符被调用时，该参数值始终为 0，但可以通过函数调用语法显式指定其他值（如 `a.operator++(2)` 或 `operator++(a, 2)`）。

## 4. 底层原理 (Underlying Principles)

### 前缀运算符行为

**内置前缀自增**：
- 表达式 `++x` 等价于 `x += 1`
- 返回操作数的左值引用
- 对 `bool` 类型（直至 C++17）设置为 `true`

**内置前缀自减**：
- 表达式 `--x` 等价于 `x -= 1`
- 返回操作数的左值引用
- 对 `bool` 类型程序非法

### 后缀运算符行为

**内置后缀自增/自减**：
1. 先对表达式应用左值到右值转换，得到修改前的值
2. 修改操作数的值（如同执行前缀版本）
3. 返回修改前的值（纯右值 prvalue）
4. 结果类型为操作数类型的 cv 无限定版本

**求值顺序**：
- 后缀自增/自减的值计算在操作数修改之前排序
- 相对于不确定顺序的函数调用，后缀自增/自减操作是单一求值

### 重载解析参与的函数签名

**前缀版本**：
```cpp
A& operator++(A&);     // 算术类型（非 bool）
P& operator++(P&);     // 指针类型
A& operator--(A&);     // 算术类型
P& operator--(P&);     // 指针类型
```

**后缀版本**：
```cpp
A operator++(A&, int);  // 算术类型（非 bool）
P operator++(P&, int);  // 指针类型
A operator--(A&, int);  // 算术类型
P operator--(P&, int);  // 指针类型
```

### 性能特征

| 运算符类型 | 时间复杂度 | 空间复杂度 | 备注 |
|-----------|-----------|-----------|------|
| 前缀自增/自减 | O(1) | O(1) | 直接修改并返回引用 |
| 后缀自增/自减 | O(1) | O(1) | 需要创建临时副本保存原值 |

> **性能提示**：由于后缀版本需要构造临时副本，在不使用返回值的上下文中，前缀版本通常更高效。

## 5. 使用场景 (Use Cases)

### 适用场景

1. **循环计数器**：在 `for`、`while` 循环中递增/递减计数器
2. **指针遍历**：遍历数组或容器元素
3. **迭代器操作**：STL 迭代器的移动
4. **状态机**：状态编号的递增/递减
5. **索引操作**：数组索引的计算和更新

### 最佳实践

```cpp
// 推荐：前缀版本用于不使用返回值的场景
for (auto it = vec.begin(); it != vec.end(); ++it) {
    // 使用 *it
}

// 推荐：后缀版本用于需要保存原值的场景
int old_value = counter++;

// 推荐：清晰的代码风格
if (++count == limit) {  // 先递增再比较
    // 达到限制
}
```

### 注意事项

1. **避免未定义行为**：不要在同一表达式中对同一变量多次修改
2. **前缀优先**：不使用返回值时优先使用前缀版本
3. **bool 类型限制**：C++17 起禁止对 `bool` 类型使用自增/自减
4. **volatile 废弃**：C++20 起对 `volatile` 类型使用自增/自减已废弃

### 常见陷阱

| 陷阱 | 说明 | 正确做法 |
|-----|------|---------|
| 多次修改 | `a = a++ + 1;` 导致未定义行为 | 分开操作：`a = a + 1; a++;` |
| 混合使用 | `int n6 = n1 + ++n1;` 未定义行为 | 使用中间变量 |
| bool 自增 | C++17 起非法 | 使用 `b = true;` 代替 |
| 后缀效率 | 循环中使用 `i++` 效率较低 | 循环中使用 `++i` |

## 6. 代码示例 (Examples)

### 基础用法

```cpp
#include <iostream>

int main() {
    int n1 = 1;
    int n2 = ++n1;      // 前缀自增：n1 变为 2，n2 = 2
    int n3 = ++ ++n1;   // 前缀自增可链式：n1 变为 4，n3 = 4
    int n4 = n1++;      // 后缀自增：n4 = 4，然后 n1 变为 5

    std::cout << "n1 = " << n1 << '\n'
              << "n2 = " << n2 << '\n'
              << "n3 = " << n3 << '\n'
              << "n4 = " << n4 << '\n';

    return 0;
}

// 输出：
// n1 = 5
// n2 = 2
// n3 = 4
// n4 = 4
```

### 指针操作示例

```cpp
#include <iostream>

int main() {
    int arr[] = {10, 20, 30, 40, 50};
    int* p = arr;

    std::cout << "*p = " << *p << '\n';        // 10
    std::cout << "*++p = " << *++p << '\n';    // 20 (先移动指针，再解引用)
    std::cout << "(*p)++ = " << (*p)++ << '\n'; // 20 (先取值，再递增值)
    std::cout << "*p = " << *p << '\n';        // 21

    return 0;
}
```

### 迭代器示例

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> vec = {1, 2, 3, 4, 5};

    // 使用前缀自增遍历（推荐）
    for (auto it = vec.begin(); it != vec.end(); ++it) {
        std::cout << *it << ' ';
    }
    std::cout << '\n';

    // 反向遍历使用自减
    for (auto rit = vec.rbegin(); rit != vec.rend(); ++rit) {
        std::cout << *rit << ' ';
    }
    std::cout << '\n';

    return 0;
}

// 输出：
// 1 2 3 4 5
// 5 4 3 2 1
```

### 自定义类型重载示例

```cpp
#include <iostream>

class Counter {
private:
    int value;

public:
    Counter(int v = 0) : value(v) {}

    // 前缀自增：返回引用
    Counter& operator++() {
        ++value;
        return *this;
    }

    // 后缀自增：返回原值
    Counter operator++(int) {
        Counter temp = *this;  // 保存原值
        ++value;               // 修改
        return temp;           // 返回原值
    }

    int get() const { return value; }
};

int main() {
    Counter c(5);

    Counter c1 = ++c;  // c1 = 6, c = 6
    Counter c2 = c++;  // c2 = 6, c = 7

    std::cout << "c = " << c.get() << '\n';   // 7
    std::cout << "c1 = " << c1.get() << '\n'; // 6
    std::cout << "c2 = " << c2.get() << '\n'; // 6

    return 0;
}
```

### 常见错误及修正

```cpp
#include <iostream>

int main() {
    int a = 1;

    // 错误：同一表达式中多次修改同一变量（未定义行为）
    // int b = a + a++;  // 错误！

    // 正确：分开操作
    int temp = a;
    int b = temp + a;  // 或者先完成计算，再递增

    // 错误：后缀自增不能链式
    // int c = a++ ++;  // 编译错误

    // 正确：前缀自增可以链式
    int d = ++ ++a;    // 正确：a = 3, d = 3

    // 错误示例：bool 自增（C++17 起非法）
    bool flag = false;
    // flag++;  // C++17 起编译错误

    // 正确替代
    flag = true;

    std::cout << "a = " << a << ", b = " << b << ", d = " << d << '\n';

    return 0;
}
```

## 7. 总结 (Summary)

### 核心要点

| 特性 | 前缀版本 | 后缀版本 |
|-----|---------|---------|
| 语法 | `++expr` / `--expr` | `expr++` / `expr--` |
| 返回值 | 操作数的引用 | 修改前的值（副本） |
| 执行顺序 | 先修改，再返回 | 先保存，再修改，返回保存值 |
| 效率 | 高（无临时对象） | 较低（需创建副本） |
| 链式调用 | 支持 | 不支持 |

### 运算符对比

| 运算符 | 等价表达式 | 结果值 | 副作用 |
|-------|-----------|--------|--------|
| `++x` | `x += 1` | 修改后的 `x` | `x` 递增 |
| `x++` | `tmp = x, x += 1, tmp` | 修改前的 `x` | `x` 递增 |
| `--x` | `x -= 1` | 修改后的 `x` | `x` 递减 |
| `x--` | `tmp = x, x -= 1, tmp` | 修改前的 `x` | `x` 递减 |

### 学习建议

1. **理解求值顺序**：前缀先修改再返回，后缀先返回原值再修改
2. **性能意识**：在循环等场景优先使用前缀版本
3. **避免未定义行为**：不要在同一表达式中对同一变量多次修改
4. **版本差异**：注意 C++17 对 `bool` 类型的限制，C++20 对 `volatile` 的废弃
5. **迭代器约定**：STL 迭代器遵循前缀/后缀返回值约定，自定义类型也应遵循

### 标准库支持

自增/自减运算符在标准库中被广泛重载：
- **原子类型**：`std::atomic<T>` 支持自增/自减操作
- **时间类型**：`std::chrono::duration` 支持自增/自减
- **迭代器**：所有迭代器类型都重载了 `operator++`，双向迭代器和随机访问迭代器还支持 `operator--`
