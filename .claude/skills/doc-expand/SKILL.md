# 技术文档扩展器

将单个技术文档扩展到指定深度。

## 输入检测

根据输入类型判断处理方式：

| 输入类型 | 检测方式 | 处理方法 |
|---------|---------|---------|
| 文件路径 | 存在对应文件 | Read 工具读取 |
| URL | 以 http/https 开头 | WebFetch 获取内容 |
| 文本内容 | 直接提供 | 直接处理 |

**检测逻辑：**

```
IF 输入是有效文件路径 → Read 工具加载内容
ELSE IF 输入以 http/https 开头 → WebFetch 获取内容
ELSE → 作为文本内容直接处理
```

## 调用方式

两种调用方式：

**1. 直接调用：**

```
Skill 工具：
  skill: "doc-expand"
  args: "输入=<路径或URL或文本> 模式=<模式>"
```

**2. 批量 Agent 调用：**

```
输入：<路径>
模式：<模式>
```

模式未指定时默认为 `detailed`。

## 步骤 1：解析参数

从 args 或 prompt 中提取：

| 来源 | 模式 | 提取内容 |
|------|------|---------|
| args | `输入=<值>` | 值 |
| args | `模式=<模式>` | 模式 |
| prompt | `输入：<值>` | 值 |
| prompt | `模式：<模式>` | 模式 |

默认值：
- `模式` → `detailed`
- 输出 → 返回内容（不写入文件）

## 步骤 2：检测输入类型

检查输入值：
1. 尝试文件存在性检查（Read 工具内部验证）
2. 检查 URL 模式（http/https）
3. 否则作为文本内容处理

## 步骤 3：获取内容

| 输入类型 | 操作 |
|---------|------|
| 文件路径 | Read 工具加载文件 |
| URL | WebFetch 工具获取 |
| 文本内容 | 直接使用 |

记录信息：
- 来源类型（file/url/text）
- 原始行数
- 文档类型（工具/语言/概念）

## 步骤 4：选择模板

根据模式应用对应模板结构：

| 模式 | 模板文件 | 章节数 |
|------|---------|--------|
| `brief` | 4 章节 | 3-4 节 |
| `standard` | 5-6 章节 | 5-6 节 |
| `detailed` | 7-8 章节 | 7-8 节 |
| `deep` | 8+ 章节 | 8+ 节 |

模板参考文件位于 `references/` 目录：
- `brief-template.md`
- `standard-template.md`
- `detailed-template.md`
- `deep-template.md`

## 步骤 5：扩展内容

按模板结构扩展：

| 优先级 | 操作 |
|--------|------|
| 1 | 补充模板中缺失的章节 |
| 2 | 用表格、代码、细节扩展简短章节 |
| 3 | 添加编号标题（## 1., ### 1.1） |
| 4 | 添加实用代码示例 |
| 5 | 添加实际案例（detailed/deep） |
| 6 | 添加原理分析（仅 deep） |

## 步骤 6：质量检查

返回前验证：

| 检查项 | 要求 |
|--------|------|
| 行数 | 在模式目标范围内 |
| 章节 | 所有模板章节存在 |
| 标题 | 正确编号 |
| 代码块 | 符合模式要求 |
| 语言 | 中文，技术术语保留英文 |

## 步骤 7：写入文件

使用 Write 工具将生成内容写入文件。

**成功输出：**

```
✅ 扩展完成
输入：{来源类型} ({来源})
输出：{输出路径}
模式：{模式}
行数：{行数}
```

**失败输出：**

```
❌ 扩展失败
输入：{来源}
错误：{错误原因}
```

Agent 通过检查输出是否以 `✅` 开头来判断成功/失败。

## 输出路径决策

| 条件 | 行为 |
|------|------|
| Agent 指定输出路径 | 使用指定路径 |
| 未指定路径 | 输出到当前工作目录 |
| 输入是文件 | 同目录，原名扩展 |
| 输入是 URL/文本 | 当前目录，document.md |

### 从参数提取输出路径

| 来源 | 模式 | 提取内容 |
|------|------|---------|
| args | `输出=<路径>` | 路径 |
| prompt | `输出：<路径>` | 路径 |

### 使用 Write 工具

```
Write 工具参数：
  file_path: ${output_path}
  content: ${generated_content}
```

## 扩展模式

| 模式 | 目标行数 | 章节 | 代码块 |
|------|---------|------|--------|
| `brief` | 100-200 | 4 | 3-5 |
| `standard` | 200-300 | 5-6 | 5-8 |
| `detailed` | 300-400 | 7-8 | 8-12 |
| `deep` | 400+ | 8+ | 12+ |

## 参考文件

`references/` 目录下的模板提供章节结构指导。

## 注意事项

- Skill 使用 Write 工具写入文件
- 批量处理：使用 `/batch-expand` 命令
- 技术准确性优先
- 尽可能保留原有风格
