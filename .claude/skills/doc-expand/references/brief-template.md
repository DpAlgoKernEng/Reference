# Brief 模式模板

目标 100-200 行，3-4 章节，3-5 代码块

## 结构

```markdown
# 工具名称 - 一句话描述

## 1. 概述

简要介绍工具定位和核心功能。

## 2. 基础使用

### 2.1 快速入门

最小可用示例。

### 2.2 常用命令

| 命令 | 说明 |
|------|------|
| cmd1 | 描述 |
| cmd2 | 描述 |

## 3. 配置选项

| 选项 | 说明 |
|------|------|
| opt1 | 描述 |

## 4. 参考资源

- [官方文档](url)
- [社区资源](url)
```

## 内容要点

| 章节 | 内容 |
|------|------|
| 概述 | 1-2 段，定位 + 核心功能 |
| 基础使用 | 1-2 个代码示例，命令表格 |
| 配置选项 | 简单表格 |
| 参考资源 | 2-3 个链接 |

## 精简原则

- 不添加历史、对比
- 不添加进阶特性
- 不添加实战案例
- 保持"快速查阅"定位
