# 错误处理指南

本文档定义路线图生成过程中各种异常场景的处理方式。

## 信息不足时的处理

| 缺失信息 | 处理方式 |
|---------|---------|
| 目标角色不明确 | 使用 AskUserQuestion 提供选项：开发工程师/高级开发/架构师/专家 |
| 当前水平未知 | 使用 AskUserQuestion 提供选项：零基础/初级/中级/高级 |
| 时间投入不明 | 默认兼职学习（每日3小时），在路线图中说明可调整 |
| 技术方向模糊 | 根据关键词检测，多方向接近时让用户确认 |

## 约束冲突时的处理

| 冲突场景 | 处理方式 |
|---------|---------|
| 目标过高 + 时间不足 | 建议延长学习时间或降低目标级别，提供两个方案 |
| 当前水平 + 目标不匹配 | 说明差距，提供补强阶段的建议 |
| 特殊兴趣与主方向偏离 | 建议分阶段：先掌握主方向，再深入兴趣领域 |
| 资源不可获取 | 替换为同级别可获取资源，标注原推荐 |

## 常见问题处理

### Q: 用户想要学习所有方向？

**A:** 建议分阶段学习，先选择一个主方向完成路线图，其他方向作为后续阶段。

### Q: 用户时间非常有限（每天<1小时）？

**A:** 延长学习周期2-3倍，聚焦核心内容，减少项目复杂度。

### Q: 用户目标过于激进（零基础1年成为架构师）？

**A:** 说明现实情况，提供分阶段目标：Year1-2 高级开发，Year3-4 架构师。

### Q: 用户已有相关经验想跳过基础？

**A:** 确认具体已掌握内容，调整起点到合适阶段，保留复习资源供参考。

### Q: 用户希望同时学习多个语言？

**A:** 建议先精通一门语言达到中级水平，再扩展第二语言，避免分散精力。

## 信息收集策略

### 必获信息（必须确认）

- **目标角色**: 开发工程师、高级开发、架构师、专家
- **当前水平**: 零基础、初级、中级、高级
- **每日可投入时间**: 全职学习 vs 兼职（具体小时数）

### 可选信息（按需收集）

- **语言偏好**: Java/Go/Python vs C++/Rust，中文 vs 英文资源
- **特殊兴趣**: 云原生、AI、数据库、操作系统内核等
- **现有技术栈**: 已掌握的语言/框架

### 信息收集方式

| 信息状态 | 处理方式 |
|---------|---------|
| 信息完整 | 直接进入路线图生成 |
| 必获信息缺失 | 使用 `AskUserQuestion` 逐个提问，优先必获信息 |
| 可选信息缺失 | 根据方向检测结果推断，必要时提问确认 |
