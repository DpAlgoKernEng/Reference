# 路线图质量检查清单

生成路线图后，执行以下检查确保质量达标。

## 完整性检查

- [ ] 所有年份/阶段都有时间标注
- [ ] 每个阶段有至少1个项目产出
- [ ] 每个主题有明确的学习资源
- [ ] 里程碑有可量化的标准
- [ ] 资源汇总表完整

## 合理性检查

- [ ] 时间估计与用户投入匹配
- [ ] 难度递进符合逻辑（基础→进阶→高级）
- [ ] 资源优先级标注清晰（高/中/低）
- [ ] 项目规模与阶段匹配
- [ ] 开源贡献路径现实可行

## 可执行性检查

- [ ] 书籍/课程资源可获取（已出版/在线）
- [ ] 项目在给定时间内可完成
- [ ] 验证标准可衡量（性能指标/代码量/认证）
- [ ] Quick Start Guide 前3个月任务详细

## 个性化检查

- [ ] 根据当前水平调整了起点
- [ ] 根据时间投入调整了进度
- [ ] 根据语言偏好选择了资源
- [ ] 特殊兴趣领域得到强调

## 质量标准定义

### 完整性
- 覆盖时间线中所有年份/阶段
- 每个阶段有具体主题和资源
- 每个主题有具体产出
- 项目有明确定义和要求

### 具体性
- 适用的周度分解
- 具名书籍/课程/文档作为资源
- 量化的产出（代码行数、博客篇数）
- 可衡量的性能目标

### 可执行性
- 任务在给定资源下可完成
- 项目范围适当
- 里程碑可验证
- 时间估计现实

### 递进性
- 逻辑技能构建顺序
- 主题间依赖关系清晰
- 难度适当递增
- 前置条件明确说明
