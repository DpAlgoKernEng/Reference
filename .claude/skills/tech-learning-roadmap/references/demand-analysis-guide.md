# 需求分析指南

收集和分析用户学习需求的详细指南。

## 关键信息维度

### 1. 方向偏好

确定学习的大方向：

| 方向 | 特点 | 典型技术栈 |
|------|------|-----------|
| 应用开发 | 业务逻辑、用户体验 | Java/Spring, Node.js, Python/Django |
| 系统编程 | 性能、底层、基础设施 | C/C++, Rust, Go |
| 前端开发 | 用户界面、交互 | React, Vue, TypeScript |
| 数据工程 | 数据处理、分析 | Spark, Flink, SQL |
| AI/ML | 模型、算法 | Python, PyTorch, TensorFlow |
| 云原生 | 容器、编排、DevOps | Kubernetes, Docker, Terraform |

### 2. 当前水平

评估起点：

| 水平 | 特征描述 | 建议起点 |
|------|----------|----------|
| 零基础 | 无编程经验 | 语言基础 + 计算机基础 |
| 初级 | 能写简单程序 | 数据结构 + 项目实践 |
| 中级 | 有项目经验 | 深入原理 + 系统设计 |
| 高级 | 有架构经验 | 垂直领域 + 技术影响力 |
| 专家 | 有深度贡献 | 创新突破 + 社区建设 |

### 3. 目标角色

明确职业目标：

| 角色 | 核心能力 | 典型职责 |
|------|----------|----------|
| 开发工程师 | 编码、调试 | 实现功能模块 |
| 高级开发 | 设计、优化 | 技术方案、难题攻关 |
| 技术专家 | 深度、专精 | 技术决策、难点攻克 |
| 架构师 | 广度、权衡 | 架构设计、技术选型 |
| 技术总监 | 战略、管理 | 团队建设、技术规划 |

### 4. 时间投入

了解可投入时间：

| 模式 | 每日时间 | 年进度估算 |
|------|----------|-----------|
| 全职学习 | 8+ 小时 | 完整 1 年计划 |
| 半脱产 | 4-6 小时 | 1.5-2 年计划 |
| 兼职学习 | 2-3 小时 | 2-3 年计划 |
| 碎片学习 | 1 小时 | 3-5 年计划 |

### 5. 语言偏好

确定主攻语言：

| 语言类型 | 语言选择 | 适用场景 |
|---------|----------|----------|
| 企业应用 | Java, Go | 后端服务、微服务 |
| 系统编程 | C++, Rust | 基础设施、性能敏感 |
| 快速开发 | Python, JavaScript | 原型、数据处理 |
| 函数式 | Scala, Haskell | 数据处理、金融 |

### 6. 特殊兴趣

识别垂直领域兴趣：

| 领域 | 核心技术 | 学习深度 |
|------|----------|----------|
| 数据库内核 | 存储引擎、查询优化 | 2-3 年专精 |
| AI 系统 | 框架开发、推理优化 | 2-3 年专精 |
| 操作系统 | 内核、驱动 | 3-5 年专精 |
| 云原生 | K8s、Service Mesh | 1-2 年深入 |
| 安全 | 密码学、渗透 | 2-3 年专精 |

## 澄清问题清单

当信息不完整时，按优先级提问：

### 必问问题

1. "你的目标是什么？想成为什么角色？"
2. "目前的基础如何？有编程经验吗？"
3. "每天能投入多少时间学习？"

### 次要问题

4. "更偏向哪个方向？应用开发还是系统编程？"
5. "有偏好的编程语言吗？"
6. "有特别感兴趣的领域吗？比如 AI、数据库、云原生？"

### 补充问题

7. "有具体的项目想法吗？"
8. "希望多久达到目标？"
9. "有必须使用的技术栈吗？（工作要求）"

## 需求分析模板

```markdown
## 用户学习需求

### 基本信息
- 方向：[应用开发 / 系统编程 / 其他]
- 当前水平：[零基础 / 初级 / 中级 / 高级]
- 目标角色：[开发工程师 / 高级开发 / 架构师 / 专家]
- 时间投入：[全职 / 兼职]（每日 X 小时）
- 语言偏好：[Java/Go/Python / C++/Rust / 其他]
- 特殊兴趣：[云原生 / AI / 数据库 / 操作系统 / 无]

### 约束条件
- 必须使用的技术：
- 时间限制：
- 其他：

### 需求优先级
1. [最优先]
2. [次优先]
3. [可选]
```

## 隐含需求推断

根据显性需求推断隐含需求：

| 显性需求 | 隐含需求 |
|---------|----------|
| 想做架构师 | 需要学习系统设计、分布式系统 |
| 想做系统编程 | 需要扎实的计算机基础 |
| 零基础起步 | 需要从编程语言和基础概念开始 |
| 时间有限 | 需要聚焦核心内容，减少冗余 |
| 有工作经验 | 可以跳过基础部分，直接进阶 |
