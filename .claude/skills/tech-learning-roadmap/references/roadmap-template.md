## 自定义变量清单

生成路线图时，需要根据用户输入填充以下变量：

### 必填变量

| Variable | Description | Example Values |
|----------|-------------|----------------|
| `direction_name` | 技术方向名称 | 全栈架构师, 系统底层专家 |
| `total_duration` | 总学习时长 | 4年, 5年 |
| `core_languages` | 主要编程语言 | Java/Go/Python, C++/Rust |
| `target_role` | 目标职业角色 | 架构师, 专家, 高级开发 |
| `daily_hours` | 每日学习时长 | 3-4小时, 全职 |
| `current_level` | 当前技术水平 | 零基础, 中级, 高级 |

### 可选变量

| Variable | Description | Default |
|----------|-------------|---------|
| `focus_area` | 重点深入领域 | 基础覆盖全部 |
| `language_pref` | 学习资源语言偏好 | 中文优先 |
| `output_format` | 输出格式偏好 | Markdown |
| `certification` | 是否包含认证 | 是 |

---

## 调整规则

### 根据当前水平调整

| Current Level | Adjustment Rule |
|---------------|-----------------|
| 零基础 | 保留全部内容，适当延长基础阶段 |
| 初级 | 基础阶段压缩20%，增加实践比重 |
| 中级 | 跳过基础阶段，直接进入进阶 |
| 高级 | 聚焦高级主题和专家能力 |

### 根据学习时间调整

| Daily Hours | Adjustment Rule |
|-------------|-----------------|
| 全职(8h+) | 标准4年路线压缩为2-3年 |
| 半职(4-6h) | 标准路线，可能延长 |
| 业余(2-3h) | 延长50%，分阶段完成 |

### 根据重点领域调整

| Focus Area | Adjustment Rule |
|------------|-----------------|
| 云原生 | 增加K8s/Istio比重，减少AI部分 |
| AI工程化 | 增加Python/PyTorch比重 |
| 数据库 | 增加存储引擎/事务比重 |
| 性能优化 | 增加调优/压测比重 |

## 输出格式选项

### 标准Markdown（默认）
- 适合GitHub/GitLab存储
- 支持表格、代码块
- 易于版本管理

### 可执行Markdown
- 包含进度追踪Checklist
- 可用工具检查完成度
- 适合项目管理

### JSON格式
- 适合程序解析
- 可导入学习管理系统
- 支持进度API
