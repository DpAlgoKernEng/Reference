# 任务生成方法论

将学习主题转化为可执行任务的详细指南。

## 任务结构模板

```
任务格式：
├── 主题：[学科]
├── 时长：[周/天]
├── 学习行动：
│   ├── 阅读：[具体章节/页数]
│   ├── 观看：[课程章节]
│   ├── 实践：[编码练习]
│   └── 分析：[源码章节]
├── 产出要求：
│   ├── 代码：[行数/文件数]
│   ├── 文档：[博客文章/页数]
│   ├── 项目：[功能/模块]
├── 验证：
│   ├── 自测：[具体问题]
│   ├── 评审：[标准]
│   ├── 基准：[性能目标]
```

## 里程碑结构模板

```
里程碑结构：
├── 时间：[月/季度/年末]
├── 知识检查：
│   ├── 掌握概念：[列表]
│   ├── 获得技能：[列表]
├── 项目完成：
│   ├── 交付物：[具体产物]
│   ├── 质量：[指标]
├── 外部验证：
│   ├── 认证：[如适用]
│   ├── 开源：[PR数量/类型]
│   ├── 面试：[能回答X级别问题]
```

## 学习行动类型

### 阅读类任务

| 资源类型 | 描述方式 | 示例 |
|---------|----------|------|
| 书籍 | 书名 + 章节/页数 | 《C++ Primer》第1-5章 |
| 文档 | URL/路径 + 小节 | 官方文档 Tutorial 部分 |
| 论文 | 论文标题 + 页数 | Raft 论文 Section 3-5 |
| 博客 | 标题 + URL | Martin Fowler 文章 |

### 观看类任务

| 资源类型 | 描述方式 | 示例 |
|---------|----------|------|
| 视频课程 | 平台 + 课程名 + 章节 | Coursera ML 课程 Week 3 |
| 技术演讲 | 会议 + 主题 | QCon 2023: Microservices |
| 直播回放 | 主题 + 时长 | 2小时内核开发直播 |

### 实践类任务

| 任务类型 | 描述方式 | 产出标准 |
|---------|----------|----------|
| 编码练习 | 练习平台 + 题目数 | LeetCode 50题 |
| 项目开发 | 项目描述 + 功能点 | 实现用户认证模块 |
| 源码分析 | 项目 + 模块 | Spring IOC 源码分析 |
| 性能优化 | 目标 + 提升幅度 | API 响应时间降低50% |

### 分析类任务

| 任务类型 | 描述方式 | 产出标准 |
|---------|----------|----------|
| 代码审查 | 项目 + 关注点 | 安全漏洞审计报告 |
| 架构分析 | 系统 + 视角 | 微服务拆分方案文档 |
| 性能分析 | 系统 + 指标 | 性能瓶颈定位报告 |

## 产出量化标准

### 代码产出

| 级别 | 行数/周 | 复杂度 |
|------|---------|--------|
| 入门 | 200-500 | 简单脚本/工具 |
| 进阶 | 500-1000 | 功能模块 |
| 高级 | 1000+ | 完整项目 |

### 文档产出

| 类型 | 频率 | 标准 |
|------|------|------|
| 学习笔记 | 每周 | 1000+ 字 |
| 技术博客 | 每月 | 3000+ 字 |
| 项目文档 | 每项目 | 完整 README + 设计文档 |

### 项目产出

| 级别 | 项目类型 | 完成标准 |
|------|----------|----------|
| 入门 | Demo/玩具项目 | 功能可用 |
| 进阶 | 实用工具/组件 | 测试覆盖 80%+ |
| 高级 | 生产级项目 | 性能 + 可观测性 + 文档完整 |

## 时间估算指南

### 学习时间分配

| 活动类型 | 占比 | 说明 |
|---------|------|------|
| 阅读/观看 | 30% | 理论学习 |
| 实践编码 | 40% | 动手练习 |
| 项目开发 | 20% | 综合应用 |
| 总结复盘 | 10% | 巩固提升 |

### 任务时长估算

| 任务复杂度 | 估算方法 | 示例 |
|-----------|----------|------|
| 简单 | 直接估算 | 学习一个API：1天 |
| 中等 | 分解估算 | 学习一个框架：2-4周 |
| 复杂 | WBS分解 | 掌握一个技术栈：按模块估算 |

## 验证标准设计

### 自测问题

设计原则：
- 覆盖核心概念
- 包含应用场景
- 有明确的正确答案

示例格式：
```
Q1: 什么是 X？它解决什么问题？
Q2: X 和 Y 有什么区别？如何选择？
Q3: 在场景 Z 中如何使用 X？
```

### 评审标准

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 正确性 | 40% | 功能正确，无 bug |
| 代码质量 | 30% | 可读性、可维护性 |
| 性能 | 20% | 满足性能目标 |
| 文档 | 10% | 注释、README 完整 |

### 性能基准

| 类型 | 基准示例 |
|------|----------|
| API 响应 | P99 < 100ms |
| 吞吐量 | QPS > 10000 |
| 资源占用 | 内存 < 512MB |
| 测试覆盖 | 覆盖率 > 80% |
