## 检测规则

### 匹配算法

1. **遍历所有方向的关键词表**
   - 统计核心关键词（权重2）和辅助关键词（权重1）匹配分数

2. **选择得分最高的方向作为主方向**

3. **多方向接近时需用户确认**
   - 如果多个方向得分接近（差距<3分），使用 `AskUserQuestion` 提问
   - 问题: "检测到您对多个方向有兴趣。请确认您的主要学习方向："
   - 选项: 各方向的详细描述（如："数据库内核开发 - 存储引擎/SQL解析/事务处理"）

### 口语化触发检测

用户可能以非正式方式表达，需要进行语义解析：

| 用户表达示例 | 检测方向 |
|-------------|----------|
| "我想学 Rust，怎么规划？" | Rust 系统编程 |
| "从零开始学数据库内核" | 数据库内核开发 |
| "怎么系统学习大模型开发？" | LLM 大模型应用 |
| "Go 后端开发学习路线" | Go 后端开发 |
| "前端转全栈怎么学" | Node.js 全栈开发 |

### 方向扩展处理

如果用户输入的关键词无法匹配现有方向：

1. 根据关键词组合推断新方向
2. 必要时使用 `AskUserQuestion` 确认用户的具体学习目标
3. 生成路线图时参考相似方向的模板结构
